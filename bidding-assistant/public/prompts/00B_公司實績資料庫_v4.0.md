## ✏️ 填寫須知（給維護人員）

**這份資料庫是「素材倉庫」**——你的工作是把每個案子的工作內容和成果盡量完整地記下來，不需要判斷「這個對未來投標有沒有用」。AI 會自己挑。

**怎麼新增一筆實績：**
1. 複製最下面的「空白模板」區塊
2. 編號格式：`P-[西元年]-[序號]`，如 `P-2025-003`
3. 逐欄填寫，填不出來的寫 `[待補]`
4. 填完後更新最上面的「最後更新」日期

**什麼案子要收錄：**
- ✅ 以公司名義得標的案子
- ✅ 以公司名義當協力廠商配合別人執行的案子
- ✅ 前身社團時期執行的計畫
- ✅ 沒得標但投過標的案子（標註「未得標」，記錄投標經驗本身也有價值）
- ❌ 純商業交易（賣貨、接零散設計稿）——除非規模夠大且與政府標案能力相關

**工作內容細項怎麼寫（最重要的欄位）：**

每一項工作內容，回答三個問題：**做了什麼事、規模多大、怎麼做的**。一到兩句話。

✅ 好的寫法：
```
| 開幕式統籌 | 規劃並執行 300 人規模開幕典禮，含流程設計、
|           | 主持稿撰寫、貴賓接待動線安排、音響燈光廠商協調 |
```

❌ 壞的寫法：
```
| 開幕式統籌 | 負責開幕式相關工作 |        ← 沒有規模、沒有具體內容
| 開幕式 | 開幕式 |                       ← 等於沒寫
```

❌ 也不要寫太細：
```
| 開幕式統籌 | 民國114年3月15日09:00報到   ← 這是結案報告不是素材庫
|           | 09:30長官致詞、09:45剪綵…… |
```

**成果數據怎麼寫：**

所有量化數字都寫進來。參與人次、場次、滿意度、媒體露出、觸及人數、印刷數量、影片觀看次數……能量化的全部收。AI 使用時會自己挑與本案相關的數字。

沒有量化數據的案子，寫質性成果也可以：「獲機關承辦人書面肯定」「機關隔年續約」。

## 🤖 AI 使用指引

<!-- 以下內容是給 AI 在各階段（L1/L2/L4 等）讀取本資料庫時的行為規範 -->

### 實績篩選

閱讀每筆實績的完整工作內容細項與成果數據，挑選與本案 RFP 需求最相關的 3-5 筆。

篩選時注意：
- 工作內容的相似度比案名的相似度重要——案名叫「行銷推廣計畫」但工作內容有展覽佈置，對展覽類標案就是相關實績
- `承接主體` 為「大員洛川股份有限公司」的實績優先於前身社團時期的實績
- `本公司角色` 為「得標廠商」的實績優先於「協力廠商」的實績
- 年度越近的實績越優先
- `結案狀態` 為「未得標」的案件不可作為實績引用，但可供內部能力評估參考

### 實績描述生成

為每筆入選實績生成 100-180 字客製化描述，規則如下：

1. **先全貌，再重點**——第一句交代全案規模與本公司整體角色（避免被誤認為分包商），第二句起切入與本案最相關的工作面向
2. 同一筆實績在不同標案中可強調完全不同的面向
3. 成果數據同樣先全貌再重點——先放總體數據，再突出與本案相關的指標
4. 若 `本公司角色` 為「協力廠商」，描述中必須誠實呈現，使用「本公司協力執行」而非「本公司承辦」
5. 語體：正式政府採購建議書用語，符合 GC-01 零容忍模糊

### 示範：同一筆實績兩種描述

**P-2025-003 → 用於展覽策劃類標案**

> 民國 114 年承辦對外貿易發展協會「TAIWAN SELECT 年度行銷推廣計畫」（決標金額新臺幣 XXX 萬元），統籌全年 11 場展覽之開幕式規劃、禮賓接待區設計與行銷素材製作。其中在展覽策劃面向，本公司負責每場展區空間配置、參觀動線規劃與主視覺設計落地執行，累計完成 11 場次展覽佈置。

**P-2025-003 → 用於行銷傳播類標案**

> 民國 114 年承辦對外貿易發展協會「TAIWAN SELECT 年度行銷推廣計畫」（決標金額新臺幣 XXX 萬元），統籌全年 11 場展覽之開幕式規劃、禮賓接待區設計與行銷素材製作。其中在行銷傳播面向，本公司負責品牌年度行銷素材設計製作、開幕式媒體聯繫安排與品牌形象一致性維護，為期 12 個月之整合行銷實績。

→ 第一句相同（全貌），第二句切入不同面向（重點）。評委看到的是「這家公司承辦了一個大案子，其中跟我們這個案子相關的部分是……」，不是「這家公司只做了某一小塊」。

### L1 即時統計

在 L1 戰略分析階段進行能力準備度評估時，AI 應先掃描本資料庫全部實績，即時統計以下數字作為評估基礎（不需要人工預先整理）：

- 總實績案件數（排除「未得標」）
- 以公司身份承接的案件數
- 以社團身份執行的案件數
- 累計決標金額
- 與本案 RFP 需求相關的實績筆數與案名
