## 一、按工作階段拆解

### Layer 1：戰略分析（/分析）

| 分析項目 | 需要的公司資訊 | 具體內容 |
|---|---|---|
| 案件基本資訊快掃 | ❌ 不需要 | 純粹從招標文件提取 |
| 機關需求與潛台詞解讀 | ❌ 不需要 | 分析機關意圖，跟公司無關 |
| 地理情報分析 | ❌ 不需要 | 查場地、交通、周邊資源 |
| 評分標準拆解 | ❌ 不需要 | 分析配分邏輯和得分策略 |
| **我方適配度評估** | ✅ 需要 | 三大業務主軸、核心競爭優勢、團隊規模 |
| **相關實績比對** | ✅ 需要 | 過往案件清單（案名、機關、金額、類型）用來判斷「我們做過類似的嗎」 |
| **假想敵分析** | ⚠️ 輕度需要 | 公司定位（規模、資本額、在地性）用來跟假想敵做對比 |
| Go/No-Go 決策建議 | ⚠️ 輕度需要 | 公司目前產能、同期是否有其他案子在跑 |

**結論**：Layer 1 的前 60% 工作（需求分析、潛台詞、地理情報、評分拆解）完全不需要公司資訊。只有後段的「適配度」和「競爭分析」才需要。

---

### Layer 2：備標規劃（/規劃）

| 規劃項目 | 需要的公司資訊 | 具體內容 |
|---|---|---|
| 章節架構設計 | ❌ 不需要 | 依評分標準反推，跟公司無關 |
| 各章寫作策略 | ⚠️ 輕度需要 | 需知道公司強項在哪，才能決定「哪章重點展示、哪章守住不失分」 |
| 創意方向發想 | ❌ 不需要 | 依需求書和目標受眾發想，是方案層面的事 |
| **差異化策略** | ✅ 需要 | 需知道公司獨特優勢（跨領域整合、漢服文化、嘻哈音樂）才能定位差異化 |
| 經費框架邏輯 | ❌ 不需要 | 依需求書規格和市場行情推算 |
| **團隊分工規劃** | ✅ 需要 | 誰負責寫哪章、設計誰做、誰去簡報——需要知道團隊成員和專長 |

### Layer 3：企劃稿（/企劃）

| 企劃項目 | 需要的公司資訊 | 具體內容 |
|---|---|---|
| 核心企劃理念 | ❌ 不需要 | 為機關設計的方案概念 |
| 各工作項目執行方案 | ❌ 不需要 | 活動怎麼辦、講座怎麼設計、流程怎麼排 |
| 交付項目規格 | ❌ 不需要 | 依需求書規格展開 |
| 時程甘特圖 | ❌ 不需要 | 依需求書期程排列 |
| 品質管控機制 | ⚠️ 輕度需要 | 通用管控流程可直接寫，但如能帶入「本公司既有 QC 流程」更有說服力 |
| 風險管理與應變 | ❌ 不需要 | 依案件特性推導風險，應變 SOP 是通用的 |
| **經費概算** | ⚠️ 輕度需要 | 人事費需參照公司實際薪資結構；業務費依市場行情 |

## 二、按建議書章節拆解（Layer 4 細分）

### 完全不需要公司資訊的章節

| 章節 | 內容來源 | 說明 |
|---|---|---|
| **前言（第1-2段）** | 需求書 + 分析結論 | 展示「我們懂機關的需求」，內容是對本案的理解，不涉及公司 |
| **計畫緣起與目標理解** | 需求書 + 政策背景 | 回應機關為何要辦這個案子、要達成什麼 |
| **服務企劃內容（全部）** | 需求書 + 企劃創意 | 建議書最核心的章節：活動怎麼辦、講座主題設計、文宣規劃、流程設計——這些全部是「方案本身」，跟公司是誰無關 |
| **時程規劃** | 需求書期程 | 甘特圖、里程碑，依合約期間排定 |
| **風險管理與應變** | 案件特性 | 天候風險、人流管控、設備故障等，依案件類型推導 |
| **創意加值服務** | 企劃發想 | 超出需求書要求的附加價值提案 |
| **CSR / 永續作為** | 通用 | 環保餐具、無障礙服務、碳中和等，不需要公司資料 |

## 三、實際操作建議

### 需要準備的公司資料檔（純文字，不要 PDF）

你需要整理 **一份文件**，包含以下五個區塊：

```
【A】公司基本資料
    - 公司名稱、統編、設立日期、負責人、地址、資本額
    - 三大業務主軸（各一句話）
    - 核心競爭優勢（3-4 點）
    - 「年輕公司 vs 資深團隊」的標準敘事段落

【B】實績資料表
    | 年度 | 案名 | 機關 | 金額 | 類型標籤 | 關鍵成果 |
    （5-10 筆，每筆一行）

【C】核心成員資料
    每人：姓名 / 最高學歷 / 核心經歷 3-5 條 / 可擔任角色
    （3-5 人）

【D】組織架構與人力配置範本
    標準組織圖 + 各職位說明
    （投標時依案件規模微調）

【E】費率參考表
    | 職位 | 日薪/月薪 | 備註 |
    + 常用外聘費率（攝影、設計、講師等）
```

### 什麼時候載入公司資料

| 階段 | 載入方式 |
|---|---|
| Layer 1 前半段（需求分析） | ❌ 不載入，讓 Claude 專注分析需求書 |
| Layer 1 後半段（適配度） | ⚠️ 只貼入【A】和【B】，約 300 字 |
| Layer 2（備標規劃） | ⚠️ 貼入【A】【B】【C】概要 |
| Layer 3（企劃稿） | ❌ 大部分不需要；經費章節才貼入【E】 |
| Layer 4 — 企劃內容章節 | ❌ 不載入 |
| Layer 4 — 公司簡介章節 | ✅ 載入【A】【B】完整版 |
| Layer 4 — 團隊章節 | ✅ 載入【C】【D】完整版 |
| Layer 4 — 經費章節 | ✅ 載入【E】 |

### 最終效果

- 約 70% 的對話完全不需要公司資訊 → 更輕量、更穩定
- 需要時才載入對應區塊 → 精準、不浪費空間
- 公司資料只有一份來源 → 不會出現前後不一致
