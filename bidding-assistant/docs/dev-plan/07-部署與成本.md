# 07 部署與成本

## 一、部署架構

### Phase 1（自用開發階段）

```
bidding-assistant/（Next.js 16 App Router）
  ├── 前端：React 19 + Tailwind CSS 4
  ├── API Routes：/api/*（Notion proxy、PCC proxy）
  ├── 開發：npm run dev（localhost:3000）
  └── 部署：Vercel Hobby（免費）
```

- **主機**：Vercel Hobby plan（$0）
  - 100GB/月頻寬（自用綽綽有餘）
  - Serverless Function timeout：10 秒
  - 每次部署上限 250MB
  - 自動 HTTPS + CDN
- **域名**：暫用 Vercel 提供的 `.vercel.app` 子域名
- **資料庫**：不自建，用 Notion API + localStorage

### Phase 2（團隊使用階段）

| 變更 | 原因 | 成本影響 |
|------|------|---------|
| Vercel Hobby → Pro | 多人使用可能超出免費額度 | +$20/月 |
| 自訂域名 | 正式對外使用 | ~$15/年 |
| Serverless timeout 60s | L1 分析等長時間操作 | 含在 Pro |
| 檔案上傳方案 | 招標文件 PDF 5-30MB 超過 4.5MB body limit | Vercel Blob 或 Presigned URL |

### 已知限制與對策

| 限制 | 影響 | 對策 |
|------|------|------|
| Serverless 10s timeout（Hobby） | PCC 批次搜尋、Notion 全量抓取跑不完 | 前端分段請求，每次查一頁 |
| 4.5MB body size（硬限制） | 招標 PDF 上傳 | Phase 1 手動處理，Phase 2 Presigned URL |
| 冷啟動 500ms-2s | 首次請求延遲 | 可接受（非即時性應用） |
| Hobby→Pro 費用跳變 | $0→$20/月 | Phase 1 在免費額度內，Phase 2 前評估 |
