## 一、功能概述

### 目標
讓使用者在標案送件後，一鍵自動生成評選簡報初版（含講稿），無需手動組織投影片內容。

### 核心使用場景
1. 案件進入「評選簡報」階段 → 點擊「生成簡報」
2. 系統自動根據案件類型選擇結構模板（工程 / 顧問 / 軟體）
3. Claude AI 自動生成 6-8 張投影片內容 + 每張講稿
4. 使用者可在 UI 中預覽、修改、匯出 PPTX

### 時間節點
- 送件檔案成熟度：已完成評估篩選（M03+M04）、建議書初稿（M02）、資格文件（M07）
- 評選簡報時機：送件前 1-2 天準備（為面試做預演）

## 三、功能清單

### 3.1 投影片結構模板（3 種）

| 模板類型 | 投影片結構 | 適用案件 |
|---------|----------|--------|
| **工程類** | 1. 封面 2. 公司簡介 3. 工程資質 4. 技術方案 5. 實績案例 6. 時程計畫 7. 團隊組成 8. Q&A 預備 | 土木、機械、電氣 |
| **顧問類** | 1. 封面 2. 公司簡介 3. 顧問經驗 4. 服務方法論 5. 本案工作計畫 6. 團隊背景 7. 預期成果 8. Q&A 預備 | 策略、管理、規劃 |
| **軟體類** | 1. 封面 2. 公司簡介 3. 技術棧 4. 產品功能 5. 安全性 / 效能 6. 開發團隊 7. 支援與維護 8. Q&A 預備 | 資訊系統、SaaS、App |

### 3.2 內容生成邏輯

**第 1 步**：辨識案件類型
- 輸入：案件資料（案件名稱、類別、預算、主要需求）
- 輸出：匹配的模板類型 → 標準結構

**第 2 步**：Slide Content 生成（Claude AI）
- 呼叫 `/api/ai/generate-slides`
- 輸入：
  ```json
  {
    "case_id": "...",
    "template_type": "engineering|consulting|software",
    "case_data": { "name": "...", "budget": "...", "requirements": "..." },
    "slide_count": 8
  }
  ```
- 輸出：
  ```json
  {
    "slides": [
      {
        "slide_index": 1,
        "title": "封面",
        "key_points": ["公司名", "標案名稱", "提案日期"],
        "speaker_notes": "各位評審，感謝給予我們...（100-150 字）"
      },
      { ... }
    ]
  }
  ```

**第 3 步**：匯出 PPTX
- 調用 M06 `docgen` 的新增 `.pptx` 輸出類型
- 套用決策 #5（通用專業樣式）
- 返回下載連結

## 四、技術規格

### 4.1 API 路由

#### 新增端點

```
POST /api/cases/[id]/presentation/generate
  Request:
  {
    "case_id": string,
    "template_type": "engineering" | "consulting" | "software",
    "force_regenerate": boolean (optional, default false)
  }
  Response:
  {
    "presentation_id": string,
    "slides": Slide[],
    "created_at": ISO 8601 timestamp
  }

GET /api/cases/[id]/presentation
  Response: Presentation object (已生成過的簡報資料)

POST /api/cases/[id]/presentation/export
  Request: { format: "pptx" }
  Response: { download_url: string, expires_in: 3600 }

PUT /api/cases/[id]/presentation/slides/[index]
  Request: { title: string, key_points: string[], speaker_notes: string }
  Response: updated Slide object
```

#### AI 路由（M06 呼叫）

```
POST /api/ai/generate-slides
  Request:
  {
    "template_type": string,
    "case_data": { name, budget, requirements, ... },
    "slide_count": number
  }
  Response: Slide content 陣列
```

### 4.3 Supabase 資料表

```sql
CREATE TABLE presentations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  case_id UUID NOT NULL REFERENCES cases(id),
  template_type TEXT CHECK (template_type IN ('engineering', 'consulting', 'software')),
  status TEXT DEFAULT 'draft',
  pptx_url TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  created_by UUID REFERENCES auth.users(id)
);

CREATE TABLE presentation_slides (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  presentation_id UUID NOT NULL REFERENCES presentations(id) ON DELETE CASCADE,
  slide_index INT NOT NULL,
  title TEXT NOT NULL,
  key_points TEXT[] NOT NULL,
  speaker_notes TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- RLS Policy
ALTER TABLE presentations ENABLE ROW LEVEL SECURITY;
ALTER TABLE presentation_slides ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users_can_view_own_case_presentations"
  ON presentations FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM cases c
      WHERE c.id = presentations.case_id
      AND c.created_by = auth.uid()
    )
  );
```

### 4.5 Claude API 整合

```typescript
// src/lib/presentation/ai-generation.ts

export async function generatePresentationContent(
  templateType: TemplateType,
  caseData: CaseDataForPresentation
): Promise<PresentationSlide[]> {
  const systemPrompt = `
    你是商務簡報寫手。根據案件資料和模板類型，生成簡報內容。
    要求：
    - 每張投影片 3-4 個要點，簡潔有力
    - 講稿 100-150 字，避免冗詞，適合現場演講
    - 格調專業、客觀、有說服力
  `;

  const userMessage = `
    模板類型：${templateType}
    案件名稱：${caseData.name}
    案件類別：${caseData.category}
    預算：${caseData.budget}
    主要需求：${caseData.requirements}

    產出 JSON 格式簡報內容（8 張投影片，含標題、要點、講稿）。
  `;

  const response = await anthropic.messages.create({
    model: "claude-opus-4",
    max_tokens: 4000,
    system: systemPrompt,
    messages: [{ role: "user", content: userMessage }]
  });

  // 解析 JSON，回傳 Slide[]
  return parseSlides(response.content[0].text);
}
```
