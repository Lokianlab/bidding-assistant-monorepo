## API 基本資訊

- URL: `https://pcc-api.openfun.app/api/`
- 資料量：1,422 萬筆公告（1999/01 ~ 2026/02/13）
- 費用：免費，無需認證
- 限制：有 rate limit（短時間密集呼叫會被 429）

## 端點清單

| 端點 | 說明 | 參數 |
|------|------|------|
| `/api/getinfo` | 系統資訊（資料時間範圍、總數） | 無 |
| `/api/` | API 列表 | 無 |
| `/api/searchbycompanyname` | 依公司名稱搜尋 | `query`, `page` |
| `/api/searchbytitle` | 依標案名稱搜尋 | `query`, `page` |
| `/api/listbydate` | 列出特定日期標案 | `date`（YYYYMMDD） |
| `/api/listbyunit` | 列出特定機關標案 | `unit_id` |
| `/api/unit` | 列出機關列表 | 無 |
| `/api/tender` | 標案詳情 | `unit_id`, `job_number` |
| `/api/searchallspecialbudget` | 列出所有特別預算 | 無 |
| `/api/searchbyspecialbudget` | 搜尋特別預算標案 | `query` |

## 評委資料（重大發現）

**API 有評委資料！** 但僅限「公開評選/最有利標」的決標公告。

### 實測案例：新北閱讀節 NTCLS114-002

```json
{
  "evaluation_committee": [
    {
      "name": "陳雪菱",
      "status": "已退休",
      "sequence": "1",
      "attendance": "是",
      "experience": "國立台灣大學圖書資訊系教授"
    },
    {
      "name": "林昱美",
      "status": "已退休",
      "sequence": "2",
      "attendance": "是",
      "experience": "台大圖書館閱覽組主任兼副館長"
    }
  ]
}
```

### 評委欄位

| 欄位 | 說明 |
|------|------|
| name | 姓名 |
| status | 身份（已退休/公務人員/學者等） |
| sequence | 委員序號 |
| attendance | 是否出席 |
| experience | 經歷（服務機關、職稱、所任工作） |

### 條件

- 只有「經公開評選或公開徵求之限制性招標」的**決標公告**才有
- 招標中的案件沒有（尚未評選）
- 最低標的案件沒有（沒有評選委員會）

## 分析能力矩陣

| 分析類型 | 可行性 | 端點 | 備註 |
|---------|--------|------|------|
| 競爭對手得標率 | ✅ 直接可做 | searchbycompanyname | 大員洛川 139 筆 |
| 機關-廠商偏好 | ✅ 直接可做 | listbyunit + tender | |
| 價格競爭力 | ✅ 直接可做 | tender detail | 含每家投標金額 |
| 類別市場分析 | ✅ 直接可做 | searchbytitle | 食農教育 847 筆 |
| **評委交叉比對** | ✅ 可做 | searchbytitle + tender | 需逐筆爬詳情 |
| **評委-廠商相關性** | ✅ 可做 | 同上 | 需建三維矩陣 |
| 評委個別評分 | ❌ 無此資料 | — | 只有最終結果 |
| 非公開評選的評委 | ❌ 無此資料 | — | 最低標無評委 |

## 評委交叉比對實作流程

```
1. searchbytitle("食農教育") → 847 筆
2. 篩選「決標公告」→ ~300-400 筆
3. 逐筆 tender detail → 擷取評委名單（需節流，每秒 1-2 次）
4. 建立 評委-機關-得標廠商 三維矩陣
5. 分析維度：
   a. 某評委出現在哪些機關（機關偏好）
   b. 某評委在場時哪家公司得標率高（廠商相關性）
   c. 某機關是否固定用同一批評委（評委班底）
```

預估時間：300 筆 × 0.5 秒 = ~3 分鐘/類別

### PCC MCP 建議工具

1. `search_company` — 搜尋公司投標記錄 + 自動計算得標率
2. `search_tender` — 按標題搜尋標案
3. `list_agency_tenders` — 列出機關標案
4. `get_tender_detail` — 取得標案詳情（含評委）
5. `analyze_competitor` — 分析競爭對手（得標率、常見機關、價格模式）
6. `analyze_evaluators` — 分析評委模式（需批量爬取）
7. `list_agencies` — 列出機關代碼
