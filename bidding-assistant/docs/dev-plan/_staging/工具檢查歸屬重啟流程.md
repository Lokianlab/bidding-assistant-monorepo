## 背景

安裝 Gemini CLI 和 Codex CLI 後，需要一個機制確保每台機器都有這些工具。最初把工具檢查加進 `/更新` 指令（Step 2），但用戶指出：「檢查和安裝工具，只有在啟動或重啟的時候才做，其他的更新指令流程都不做。」

原因：`/更新` 是輕量同步（pull + 碰撞偵測 + 恢復上下文），每次對話中可能執行多次，不應該每次都跑工具檢查。工具不會憑空消失，只有重啟（新 session）時才需要確認一次。

## 結論

- **工具檢查只放在 CLAUDE.md 的重啟流程 Step 3**，不放 `/更新`
- `/更新` 維持原本的 4 步驟不動（pull → 碰撞偵測 → 恢復上下文 → 完成）
- `/安裝` 指令同步補上 Gemini CLI 和 Codex CLI 的安裝命令

## 寫入內容

已寫入 CLAUDE.md 重啟流程 Step 3，內容如下：

```
工具檢查（每次重啟都執行）：
a. CLI：gemini --version、codex --version → 缺的自動 npm install -g
b. MCP：讀 .mcp.json，逐個檢查
   - command 是 npx → 跳過（自動拉）
   - command 是 node + dist/index.js → 檢查檔案存在
   - 不存在 → 告知用戶需要 npm install && npm run build
```
