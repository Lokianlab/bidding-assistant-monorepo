### 史多禮案例教訓（首次驗證，統編 42666702）

#### 分析錯誤

1. **定性判斷錯誤**：沒有 Perplexity 時，把史多禮定性為「工作室」，實際上是「故事 StoryStudio」媒體品牌公司（33.9 萬粉絲）
2. **營收低估**：只算標案收入（7,271 萬），忽略媒體/出版/教育等非標案收入（推估佔 30-50%）
3. **品質用猜的**：從「回頭客多」推論品質好，實際上有金鼎獎兩屆得獎的硬證據
4. **遺漏關聯企業**：完全沒發現臺灣吧（臺灣各種吧股份有限公司）的關聯，同一代表人蕭宇辰
5. **遺漏政策風險**：114 年文化預算大砍（刪 11 億凍 34 億），主要客戶機關全部受影響

#### 方法論教訓

- 洞見分析不能年度小切片（2-5 筆/格）→ 結論自相矛盾 → 全樣本才有意義
- 禁用戲劇化用語（被批「打選戰」）
- 人稱統一用公司名或「他們」，不要你/他交替
- SWOT 表格不能丟，「文字變多資訊變少」= 退步

### PCC 數據技術備忘

- `brief.companies.name_key` 判斷得標/未得標
- 預算欄位用 `:預算金額` 結尾匹配（欄位名不固定）
- openfun 記錄數 ≠ mlwmlw.org 記錄數 → 金額用 mlwmlw，行為分析用 openfun
- `/searchbyaward` 和 `/searchbyunit` 偶爾回 HTML → 不穩定，需做錯誤處理
- 大量數據處理寫 .mjs 腳本跑，不用 `node -e`（escape 問題）
