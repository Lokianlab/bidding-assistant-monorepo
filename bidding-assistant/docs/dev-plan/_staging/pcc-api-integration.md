## API 為什麼是核心

這個 API 是系統情報基礎設施，因為它能**自動取代 P 偵察報告中 80% 的人工搜尋**。

| P 偵察報告要查的 | 以前靠 Perplexity 盲搜 | 現在 PCC API 直接給 |
|---|---|---|
| 機關近3年同類標案 | ⚠️ 不完整、可能遺漏 | ✅ searchbytitle + 篩機關 |
| 得標者/金額/投標家數 | ⚠️ 只查得到部分 | ✅ tender 決標資料，精確到元 |
| 未得標者名單 | ❌ Perplexity 幾乎查不到 | ✅ API 直接提供（含金額、排名、分數、原因） |
| 評委名單 | ❌ Perplexity 查不到 | ✅ API 有（如案件有公佈） |
| 競爭對手全部投標紀錄 | ⚠️ 只能一家一家搜 | ✅ searchbycompanyname 一次拉完 |
| 大員洛川自身紀錄 | ⚠️ 手動拼湊 | ✅ searchbycompanyname → 139 筆 |
| 機關政策脈絡/新聞 | ✅ Perplexity 擅長 | ❌ API 無此資料 |
| 競爭者公司背景 | ✅ Perplexity 擅長 | ❌ API 只有投標紀錄 |
| 地方政治/產業脈絡 | ✅ Perplexity 擅長 | ❌ API 無此資料 |

**結論**：PCC API 負責硬數據（數字、名單、紀錄），Perplexity 負責軟情報（脈絡、新聞、分析）。

## 工作流程

```
Step 0: 使用者指定案件（案名/案號/機關名）
    ↓
Step 1: 系統查 PCC API（自動，1-2 分鐘）
    ├── searchbytitle → 同類型標案一覽
    ├── tender → 每案決標詳情（得標者、未得標者、評委）
    ├── searchbycompanyname("大員洛川") → 我方投標歷史
    └── searchbycompanyname(各競爭者) → 對手投標歷史
    ↓ 產出：預填數據包
Step 2: 系統組裝 P 偵察 prompt（自動，幾秒）
    ↓ 硬數據已填，Perplexity 只需補軟情報
Step 3: 使用者貼進 Perplexity（手動，等 3-5 分鐘）
    ↓ 產出：P_偵察報告.md
Step 4: 使用者把 P 報告 + 招標文件貼回系統
    ↓
Step 5: 系統跑 L1 戰略分析（Claude，10-15 分鐘）
    ↓ 產出：L1 戰略分析報告 + 投標決策
```

使用者全程只動手 3 次：拿 prompt 出去、把 P 報告帶回來、餵招標文件。

## 系統能自動做到的事

### 1. 競爭情報自動化
- **一鍵查任何廠商的全部投/得標紀錄**（searchbycompanyname）
- **自動識別常見對手**：從大員洛川 139 筆紀錄中，提取每案的所有競爭者，統計「撞案頻率」
- **對手得標率計算**：每個對手出現幾次、得標幾次、勝率多少
- **對手擅長領域分析**：從對手得標案件的案名/機關，推斷其強項

### 2. 機關情報自動化
- **機關歷史標案一覽**：同機關的同類型案件（searchbytitle + 篩 unit_id）
- **預算趨勢**：同類案件歷年預算金額變化
- **得標金額 vs 預算比**：判斷是否有壓價趨勢
- **在位者識別**：誰連續得標、護城河多深
- **投標家數趨勢**：紅海 vs 藍海判斷

### 3. 自我分析自動化
- **大員洛川完整投標史**：139 筆（2 頁）
- **按機關/案件類型分類**：投過哪些機關、哪些領域
- **勝率統計**：目前 0%（確認痛點分析中的數據）
- **常見落選原因分析**：從未得標原因欄位統計

### 4. L1 戰略分析增強
- Step 5.1 在位者分析 → API 提供真實得標者 + 金額 + 連續得標次數
- Step 5.2 挑戰者分析 → API 提供真實投標者 + 排名 + 分數
- Step 5.3 投標家數趨勢 → API 提供精確數字
- Step 5.4 競爭矩陣 → 用真實公司名，不是假想敵
- Step 5.5 我方歷史 → API 直接提供

## API 技術備註

- **免費、無認證、開放 CORS**
- 每頁 100 筆，用 `page` 參數翻頁
- `total_records` / `total_pages` 提供分頁資訊
- 端點命名注意：正確是 `searchbycompanyname`（不是 searchbycompany）
- HTTP 200 回 HTML 錯誤頁 = 端點不存在（伺服器不回 404 status code）
- 資料來自政府電子採購網（web.pcc.gov.tw），有時間差但覆蓋面廣
- 大員洛川在 API 中的名稱：`大員洛川股份有限公司 (Tailok-Coop Co., Ltd.)` 或 `大員洛川股份有限公司`
- 大員洛川統編：`89170941`
