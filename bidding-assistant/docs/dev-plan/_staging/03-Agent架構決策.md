## 內容

### 雙層 Agent 策略

| 層 | 用途 | 技術 | 誰用 |
|----|------|------|------|
| **施工層** | 開發系統時多 agent 並行寫程式 | Claude Code Agent Team | 開發者 |
| **產品層** | Discord Bot 背後的 AI 編制 | Claude Code SDK | 企劃團隊（透過 Discord） |

### 產品層 Agent 編制

| Agent | 角色 | 負責 | 使用的 prompt |
|-------|------|------|-------------|
| 戰略官 | 分析與決策 | L1 戰略分析、投標決策、案件拆解 | L1 v2.3 + P 報告 |
| 企劃官 | 規劃與撰寫 | L2 備標規劃、L3 章節撰寫 | L2 + L3 + 撰寫規範 |
| 品管官 | 品質把關 | 三道閘門、L4 審查 | L4 + 品質規則 |
| 知識官 | 知識管理 | 知識庫檢索、注入、回流 | T1 + 00A-00E |
| 簡報官 | 表達與演練 | L5-L8 | L5-L8 |

### 成本控制

- 產品層用 Claude Code SDK（走 Max 訂閱，不走 API 計費）
- 必須實作用量監控（token 數 + 回應時間 + 限速事件）
- 限速主動報告（偵測到 rate limit 時 Discord 通知管理者）
- 輕量查詢不啟動 Agent，直接查資料庫
