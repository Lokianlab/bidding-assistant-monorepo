## 收入公式衝擊

```
收入 = 得標案件數 × 平均案件金額 × 毛利率
```

| 公式變數 | PCC API 能做的事 | 影響程度 |
|---------|-----------------|---------|
| 發現機會數 ↑ | `listbydate` + `searchbytitle` 自動監控新標案 | 中（替代人工刷 pcc） |
| 投標決策通過率 ↑ | 競爭對手分析、機關偏好、歷年預算行情、評委模式 | **高**（從盲投到有情報） |
| 得標率 ↑ | 評委背景 → 調整企劃寫作策略；價格分析 → 報價策略 | **高** |
| 平均案件金額 ↑ | 篩選高價值案件，避免浪費精力在小案 | 中 |

## 戰略官自動產出的情報（L1 備標評估）

| 分析項 | API 端點 | 決策影響 |
|--------|---------|---------|
| 同類案競爭激烈度 | `searchbytitle` 歷年同類案 | 判斷值不值得投 |
| 常見對手名單+得標率 | `searchbycompanyname` 交叉 | 知道要打贏誰 |
| 機關是否有默契廠商 | `listbyunit` + `tender` | 避免白做工 |
| 預算行情/底價比例 | `tender` detail 歷年金額 | 報價策略 |
| 評委可能人選+背景 | 同機關/同類案歷年評委 | **最高價值情報** |
| 評委-廠商相關性 | 三維矩陣分析 | 判斷勝算 |

## 對企劃撰寫的影響（L3）

知道評委背景後企劃官可調整寫作策略：
- 學術背景評委 → 加強理論架構、文獻引用
- 實務背景評委 → 加強執行細節、過往成果
- 特定議題專家 → 摘要直接回應該議題

## 開發計畫優先序調整建議

### PCC MCP 應併入 Layer 0

理由：
1. 開發成本低（API 免費、結構化、不需認證，預估 1-2 天）
2. 價值密度高（直接推動收入公式最前端）
3. 戰略官的基礎（沒有 PCC 資料，戰略官半盲）

```
原計畫：Layer 0（知識庫 00A-00F）→ Layer 1（Bot + 戰略官）
建議：  Layer 0（知識庫 + PCC MCP）→ Layer 1（Bot + 戰略官立刻有武器）
```

### Batch H 可提前且簡化

原計畫 Batch H「pcc-api 標案監控」排 Phase 3，但 API 直接可用不需爬蟲：
- `/case monitor 食農教育` → 每日 `searchbytitle` 檢查 + Discord 通知
- 可併入 P0 Bot 基礎

### 戰略官能力升級

```
原本：戰略官 = Notion 案件庫（自己的歷史）
現在：戰略官 = Notion 案件庫 + 全國 1,422 萬筆標案（整個市場）
```

從「看後照鏡開車」→「有 GPS 導航」

## 一句話結論

PCC MCP 併入 Layer 0 同步建設，讓戰略官從第一天就有全市場情報，直接推動「投標決策通過率」和「得標率」。
