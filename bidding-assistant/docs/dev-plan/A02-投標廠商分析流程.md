# A02 投標廠商分析流程

## 分析流程

1. **使用者指定分析對象**（公司名稱 + 統編）
2. **Claude 產出 Perplexity 提示詞模板**
3. **使用者跑 Perplexity 深度搜尋**，回傳結果
4. **Claude 拉 PCC 數據**（searchbycompanyname）+ 整合產出 11 章報告

詳細執行指示見：`bidding-assistant/docs/投標廠商分析-指示.md`

## 報告架構（11 章）

| 章 | 名稱 | 核心產出 |
|----|------|---------|
| 1 | 公司定性 | 一句話定性 + 維度表（本質、品牌、商業模式、護城河） |
| 2 | 團隊結構 | 創辦人背景、核心成員、運作模式、關鍵人物依賴度 |
| 3 | 營收與商業模式 | 年度標案營收表、非標案收入盤點、營收結構 |
| 4 | 品質實證 | 獎項、出版品、展覽成果、品牌影響力、外部評價 |
| 5 | 利基市場定位 | 核心市場表、產業結構、政府預算趨勢 |
| 6 | 競爭力評估 | SWOT（含數據佐證，必須用表格呈現） |
| 7 | 競爭者比較 | 競爭者名冊、能力對比、標案交叉比對、威脅排序 |
| 8 | 關聯企業分析 | 關聯圖譜、資源共享、聯合投標紀錄（無關聯企業則省略） |
| 9 | 競爭策略研判 | 差異化切入點、老客戶滲透路徑 |
| 10 | 風險訊號 | 履約爭議、訴訟、政策風險、AI 衝擊等 |
| 11 | 關鍵數字速查 | 單一彙總表（供快速決策） |

各章完整子項目和格式要求見指示文件。

## 與 M01 情報模組的關係

| | A02 深度分析 | M01 對手側寫（規劃中） |
|---|---|---|
| 觸發方式 | 人工指定 + Perplexity 介入 | 全自動（純 PCC 數據） |
| 耗時 | 30-60 分鐘 | 秒級回應 |
| 產出 | 11 章完整報告 | 撞單次數、機關勝率等速查 |
| 適用時機 | 決定投標前的深度研究 | 瀏覽案件時快速了解對手 |

對手側寫的數據是深度分析的子集（僅 PCC 部分）。兩者互補，不重疊。

## 史多禮案例教訓（首次驗證，統編 42666702）

> **適用範圍**：以下教訓來自文化/媒體類廠商（網上資料豐富）。分析資料稀少的廠商（如工程、IT 外包）時，可能遇到不同類型的盲區——Perplexity 找不到資料時，以下第 1-4 條教訓無法以同樣方式驗證。

### 分析錯誤

1. **定性判斷錯誤**：沒有 Perplexity 時，把史多禮定性為「工作室」，實際上是「故事 StoryStudio」媒體品牌公司（33.9 萬粉絲）
2. **營收低估**：只算標案收入（7,271 萬），忽略媒體/出版/教育等非標案收入（推估佔 30-50%）
3. **品質用猜的**：從「回頭客多」推論品質好，實際上有金鼎獎兩屆得獎的硬證據
4. **遺漏關聯企業**：完全沒發現臺灣吧（臺灣各種吧股份有限公司）的關聯，同一代表人蕭宇辰
5. **遺漏政策風險**：114 年文化預算大砍（刪 11 億凍 34 億），主要客戶機關全部受影響

### 方法論準則

- 洞見分析不能年度小切片（2-5 筆/格）→ 結論自相矛盾 → 全樣本才有意義
- 禁用戲劇化用語（被批「打選戰」）
- 人稱統一用公司名或「他們」，不要你/他交替
- SWOT 表格不能丟，「文字變多資訊變少」= 退步
