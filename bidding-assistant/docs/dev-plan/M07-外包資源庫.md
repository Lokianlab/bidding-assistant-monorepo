## 一、功能定位

### 核心用途
提案評估時，推薦外部合作夥伴（如顧問、子承商、技術服務商等），以補強內部能力缺口。

### 使用流程
```
M03 戰略分析頁面
  ↓
[推薦合作夥伴] 側邊欄
  ↓
自動搜尋相關夥伴（by 專業類別）
  ↓
顯示評分、合作歷史、聯絡方式
  ↓
使用者可新增、編輯、標記「已洽詢」
```

## 二、資料結構設計

### Supabase 表：`partner_contacts`

| 欄位 | 型別 | 說明 | 例值 |
|------|------|------|------|
| `id` | UUID | 主鍵 | — |
| `tenant_id` | TEXT | 多租戶隔離（外鍵 → auth.users.id） | — |
| `name` | TEXT | 合作夥伴名稱 | 「ABC 建築設計顧問」 |
| `category` | TEXT[] | 專業類別（多選） | `['建築設計', '工程評估']` |
| `contact_name` | TEXT | 聯絡人 | 「王小明」 |
| `phone` | TEXT | 電話 | `09XX-XXX-XXX` |
| `email` | TEXT | 電子郵件 | `contact@abc.com` |
| `url` | TEXT | 公司網址 | `https://abc.com` |
| `rating` | INTEGER | 評分（1-5） | 4 |
| `notes` | TEXT | 備註 | 「過去 2 個案合作，效率佳」 |
| `cooperation_count` | INTEGER | 合作次數 | 2 |
| `last_used` | TIMESTAMP | 最後使用日期 | 2026-02-15 |
| `tags` | TEXT[] | 自訂標籤 | `['推薦', '設計']` |
| `status` | TEXT | 狀態（active / archived） | `'active'` |
| `created_at` | TIMESTAMP | 建立時間 | — |
| `updated_at` | TIMESTAMP | 更新時間 | — |

### RLS 策略

```sql
-- 每個租戶只能讀寫自己的夥伴記錄
CREATE POLICY "tenant_isolation" ON partner_contacts
  USING (tenant_id = auth.jwt() ->> 'sub')
  WITH CHECK (tenant_id = auth.jwt() ->> 'sub');
```

## 三、API 設計

### Routes：`/api/partners/`

| 方法 | 端點 | 說明 | 認證 |
|------|------|------|------|
| GET | `/` | 列出該租戶的所有夥伴 | 需要 |
| GET | `/?category=abc&search=xyz` | 搜尋（by 類別、名稱） | 需要 |
| POST | `/` | 建立新夥伴記錄 | 需要 |
| PATCH | `/:id` | 編輯夥伴資訊 | 需要 |
| DELETE | `/:id` | 刪除夥伴（軟刪除 to archived） | 需要 |
| POST | `/:id/usage` | 標記已洽詢（+cooperation_count、更新 last_used） | 需要 |

### 實作位置

- **邏輯層**：`src/lib/partners/`
  - `types.ts` — 介面定義
  - `helpers.ts` — 搜尋、篩選邏輯
  - `usePartners.ts` — React Hook（CRUD + 搜尋）

- **API 層**：`src/app/api/partners/`
  - `route.ts` — GET / POST 實作
  - `[id]/route.ts` — PATCH / DELETE / POST usage 實作
  - `[id]/helpers.ts` — 個別端點的邏輯（如驗證）

## 五、集成設計

### 與 M03 Strategy 的關聯

```
Strategy 頁面
  ├─ 左：分析結果
  ├─ 中：決策框架
  └─ 右：[推薦合作夥伴] ← PartnerSidebar
         ├─ 自動篩選：若分析結果標註「需要 XX 專業」
         │  → 自動查詢相關類別的夥伴
         └─ 使用者手動操作：搜尋、新增、標記洽詢
```

### 實裝邏輯

1. M03 Strategy 頁面掛載 PartnerSidebar
2. PartnerSidebar 讀取 Strategy 的「需要的技能/專業」（prop）
3. `usePartners` Hook 自動過濾匹配的夥伴
4. 使用者點選 → 記錄洽詢
