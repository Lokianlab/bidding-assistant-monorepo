# 05 工作流程

## 案件發現漏斗

```
pcc 瀏覽（多帳號 200*N）
    ↓ 行政初篩（標準：案名 vs 公司業務範圍）
600-1200 件/月 通過初篩
    ↓ 行政第一輪（每件都做）：貼表 + 貼檔
    ↓ 企劃初選（看 Notion 摘要）
~200 件/月 選入
    ↓ 行政第二輪（開 Drive 資料夾 + 貼備標評估文件）
    ↓ 企劃 + AI 深度分析（L1 快速分析 + A/B/C/D 分級，詳見 08）
~4 件/月 決定投標 → L2-L8
```

## 行政每日四動作（600-1200 件/月）

1. **貼表** — 案件資訊填入 Notion
2. **貼檔** — 下載→清理→轉 PDF→嵌入 Notion
3. **摘要** — Notion AI 生成（格式不穩定）
4. **情蒐**
   - 決標記錄查詢（PCC API 可自動化）
   - 投標地址行車資訊（手動，僅限進入決策流程的案件）

人力：約 5 位助理分工，每人每月 120-240 件

## 真實工作流程（Jin 描述 2026-02-22）

### 角色

- **行政人員**：每天上午巡標（上 PCC 抓新公告進 Notion）
- **企劃人員（Jin/鹿淵）**：篩選、評估、寫企劃
- **設計師**：接設計需求單做視覺

### 每日節奏

| 時間 | 誰 | 做什麼 | 工具 |
|------|-----|--------|------|
| 上午 | 行政 | 上 PCC 抓新公告，貼進 Notion | PCC 網站 → Notion |
| 上午 | 企劃 | 打開「企劃協作及備標」表，處理已篩選的案子 | Notion |
| 下午 | 企劃 | 跑「評估工作流」，篩選當天新案子 | Notion |
| 需要時 | 企劃 | 開設計需求單給設計師 | Notion |
| 隨時 | 企劃 | 監看「投標進度監控處」 | Notion |

### Notion 企劃作業區（四個區塊）

1. **評估工作流**
   - 關鍵字初篩，分四類：
     - 絕對可以：食農教育/藝術/服務採購/100萬以下/影片製作/行銷計畫/春聯
     - 需要讀細節：主燈設計/各種節慶（燈節、藝術節等）/舞台/布置/晚會演唱會
     - 先不要：課後服務
     - 剩下的留給其他有經驗的人
   - 註：此關鍵字清單是「代理人簡化版」，Jin 本人判斷更細
   - 兩關篩選：初級（消極汰除）→ 二級（積極挑選）

2. **企劃協作及備標**（主工作區）
   - 欄位：優先序（第一/二/三順位）、完成進度、負責人、截止日期、案號、機關、預算、工作說明

3. **投標進度監控處**
   - 競標中控臺，按狀態分類：著手領/備標、競標階段、已出席簡報、逾期未參與、領標後未參與、資格不符、未獲青睞
   - 投標排程

4. **設計需求單**
   - 欄位：需求名稱（綁案號）、需求人、初稿交件時間、完稿期限、狀態

## PCC API → P 偵察 → L1 整合（核心工作流）

### Step 0：使用者指定案件
輸入：案名 / 案號 / 機關名

### Step 1：PCC API 自動擷取（系統，1-2 分鐘）

```
searchbytitle(案名關鍵詞)
  → 同類型標案一覽
  → 每案用 tender() 取決標詳情
    → 得標者、未得標者、評委、金額

searchbycompanyname("大員洛川")
  → 我方投標歷史（139 筆）

searchbycompanyname(各競爭者名稱)
  → 對手投標歷史
```

產出：**預填數據包**（結構化表格）

### Step 2：組裝 P 偵察 prompt（系統，幾秒）

將預填數據包塞入 P 偵察報告模板：
- 「機關歷史標案一覽」已填 → Perplexity 只補政策脈絡
- 「競爭者名單」已填 → Perplexity 精準搜各家公司背景
- 「評委名單」已填 → Perplexity 搜各評委背景
- 「我方紀錄」已填 → Perplexity 只查相關新聞

產出：一份可直接貼的 prompt

### Step 3：Perplexity Deep Research（人，3-5 分鐘）

使用者複製 prompt → 貼入 Perplexity → 等待 → 匯出 .md

### Step 4：L1 戰略分析（Claude，10-15 分鐘）

輸入：招標文件 + P 偵察報告.md
使用 prompt：`01_戰略分析_v2.3.md`

L1 中依賴 P 偵察報告的步驟：
- Step 3 潛台詞分析 → 交叉比對政策脈絡
- Step 5 競爭格局 → 用真實得標者/投標者（非假想敵）
- Step 6 SWOT → 至少各一項引用 P 報告外部數據
- Step 7 投標決策 → 至少一個理由基於外部情報

### Step 5：團隊決策會議（人，30 分鐘）

基於 L1 報告做出：投 / 謹慎評估 / 放棄

## 使用者全程操作次數

| 步驟 | 操作 | 時間 |
|------|------|------|
| 指定案件 | 輸入關鍵詞 | 10 秒 |
| 貼 Perplexity | 複製 → 貼上 | 30 秒 |
| 等 Perplexity | 等待 | 3-5 分鐘 |
| 帶回 P 報告 | 複製 → 貼回 | 30 秒 |
| 餵招標文件 | 上傳/貼上 | 1 分鐘 |

**人只動手 3 次**：拿 prompt 出去、把結果帶回來、餵招標文件。
