# M08 評選簡報生成器 — 規格文件

**版本**：0.1（規格確認中）
**分派**：JDNE 規格 → Z1FV 實裝（或 A44T）
**階段**：第 9 階段（評選簡報）
**依賴**：M06 docgen 管線、Claude AI endpoint
**預期測試數**：≥ 50 tests

---

## 1. 功能概述

目標：送件後一鍵生成評選簡報初版，附建議講稿。讓非專業報告人員也能上場。

**核心使用場景**：
- 案件進入「評選簡報」分頁後，點「生成簡報」
- AI 根據標案類型、提案內容自動生成 6-8 張投影片
- 每張投影片配對應講稿（3-5 句話）
- 可預覽、編輯、匯出 PPTX

---

## 2. 功能清單

### 2.1 簡報結構建議
- 根據標案類型（工程建築、顧問、軟體等）選擇模板
- 模板清單：
  - 工程類：封面 → 公司介紹 → 技術方案 → 團隊 → 工程計畫 → 成本控制 → Q&A
  - 顧問類：封面 → 問題理解 → 解決思路 → 案例 → 時程 → 成本 → Q&A
  - 軟體類：封面 → 業務背景 → 技術架構 → 功能說明 → 交付計畫 → 支持承諾 → Q&A

### 2.2 投影片內容草稿（AI 生成）
- 輸入：標案 ID + 用戶編輯過的提案初稿
- 輸出：每張投影片標題 + 3-5 個要點（bullets）
- AI prompt：在 `/src/lib/presentation/` 下設計 templates
  - `getPresentationOutline(caseType, proposalContent)` → 結構 JSON
  - 格式：`{ slides: [ { title, bullets: [...] } ] }`

### 2.3 講稿生成（AI 生成）
- 根據投影片內容自動生成 3-5 句講稿
- 語調：專業但親近（非念稿風格）
- API 呼叫：`POST /api/ai/generate-speech`
  - 輸入：投影片標題 + 要點
  - 輸出：講稿文字

### 2.4 匯出為 PPTX
- 擴充 M06 docgen 的 `.pptx` 輸出類型
- 整合 `docx-assembler` 或獨立 `pptx-lib`（待決）
- 包含：
  - 投影片布局（標題、要點、講稿備註欄）
  - 公司品牌色（從 KB 或 case 資料取）
  - 頁碼 + 日期（自動填充）

### 2.5 在 case-work 頁面加入入口
- case-work 分頁結構延伸：
  - 「基本資訊」、「提案內容」、「資格文件」、**「評選簡報」**（新）
  - 簡報分頁按鈕：「生成簡報初版」→ 進度條 → 預覽 → 編輯 → 匯出

## 3. 技術規格

### 3.1 API 設計

```
POST /api/cases/[id]/presentation/generate
Request body:
{
  "caseType": "engineering" | "consulting" | "software",
  "proposalContent": { ... },
  "options": {
    "slideCount": 6-10,  // 用戶可自訂
    "includeNotes": true
  }
}

Response:
{
  "slides": [
    {
      "id": "slide-1",
      "title": "公司概況",
      "bullets": ["成立 20XX 年", "服務 XXX 客戶", ...],
      "speech": "各位評委，感謝機會。我們公司成立於...",
      "notes": "（內部備註：這邊要強調...)
    },
    ...
  ],
  "metadata": { "caseId", "generatedAt", "version" }
}
```

### 3.2 檔案結構

```
bidding-assistant/
  src/
    lib/
      presentation/
        templates.ts          # 簡報模板定義
        helpers.ts            # 邏輯輔助函式
        usePresentations.ts   # React Hook
    app/
      tools/
        presentation/
          page.tsx            # 主頁面
          [caseId]/
            preview.tsx       # 預覽組件
            editor.tsx        # 編輯組件
      api/
        ai/
          generate-speech/    # 講稿生成 endpoint
        cases/
          [id]/
            presentation/     # 簡報相關 routes
              route.ts        # 生成、獲取、更新
```

### 3.3 元件清單

| 元件 | 用途 | 測試重點 |
|-----|------|---------|
| `PresentationPreview` | 投影片預覽（幻燈片控件） | 導航、切頁、講稿展開 |
| `PresentationEditor` | 投影片編輯（inline 編輯） | 編輯保存、校驗、同步 |
| `SpeechNotes` | 講稿編輯面板 | 文字輸入、字數限制、預覽 |
| `ExportButton` | PPTX 匯出 | 匯出成功、檔案完整性 |

## 5. 決策點（待 Jin / JDNE 確認）

1. **簡報模板來源**
   - 硬編碼 3 種模板？或從 KB 動態拉取？
   - 建議：硬編碼 3 種（v1 簡單），后期升級為可配置

2. **講稿 AI endpoint**
   - 用 Claude `/messages` 還是 `/completions`？
   - 建議：`/messages` 新 API 品質更好

3. **PPTX 庫選擇**
   - `docx-assembler`（已用於 M06，擴充 PPTX 類型）
   - 還是獨立 `pptx` npm 套件（如 `pptxjs`, `slide-deck`）？
   - 建議：先用 `docx-assembler` 試試，不行再換

4. **講稿字數限制**
   - 每張 3-5 句話還是 100-150 字？
   - 建議：100-150 字（對應 3-5 句自然表達）

5. **簡報預設樣式**
   - 用公司品牌色還是通用專業樣式？
   - 建議：通用優先（v1），后期加品牌色自訂
