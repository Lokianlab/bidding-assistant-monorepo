# W01 行政巡標自動化（Daily Bid Scanning Automation）

## 定位

P0 優先——Jin 親定的第一件事（0222 確認，見 `docs/records/stash-product-direction-0222.md`）。

每天上午，行政人員上 PCC 抓新公告 → 貼進 Notion。這是目前完全手動的流程。
自動化後：系統用關鍵字搜 PCC → 呈現候選清單 → 巡標人點「要」→ 全自動建案。

## 用戶工作流程（Jin 定義）

```
每日排程（或手動觸發）
→ PCC API 用預設關鍵字搜尋
→ 擷取公告清單（標題、預算、機關、截標日）
→ 關鍵字初篩（四類分類）
→ 巡標人看清單，點「要」或「不要」
→ 點「要」後全自動：
  → 從公告 URL 擷取所有欄位
  → Notion 建檔（進「企劃協作及備標」表）
  → Google Drive 建資料夾
  → 產生案件摘要
  → 跑情報蒐集
  → 結果回寫 Notion
```

## 關鍵字篩選引擎

### 四類清單（來源：Jin 0222 定義）

| 類別 | 關鍵字 | 動作 |
|------|--------|------|
| **絕對可以** | 食農教育、藝術、服務採購、100萬以下、影片製作、行銷計畫、春聯 | 自動推薦 ⭐ |
| **需要讀細節** | 主燈設計、各種節慶（燈節/藝術節等）、舞台、布置、晚會、演唱會 | 標記 🔍 |
| **先不要** | 課後服務 | 自動排除 ❌ |
| **其他** | 不符合以上任何類別 | 標記 ❓（留給有經驗的人） |

### 篩選邏輯

```typescript
interface KeywordRule {
  category: "must" | "review" | "exclude" | "other";
  keywords: string[];        // 比對標案名稱
  budgetMax?: number;        // 預算上限（如「100萬以下」）
  label: string;             // UI 顯示標籤
}

// 篩選優先序：排除 → 絕對可以 → 需要讀 → 其他
function classifyTender(title: string, budget: number, rules: KeywordRule[]): Classification {
  // 1. 先檢查排除
  if (matchesAny(title, excludeRules)) return { category: "exclude", reason: "..." };
  // 2. 絕對可以（含預算條件）
  if (matchesAny(title, mustRules) || budget <= 1_000_000) return { category: "must", reason: "..." };
  // 3. 需要讀細節
  if (matchesAny(title, reviewRules)) return { category: "review", reason: "..." };
  // 4. 其他
  return { category: "other", reason: "不符合已知類別" };
}
```

### 關鍵字管理

- 關鍵字清單存在設定中（Settings），用戶可在 UI 編輯
- Jin 說此清單是「代理人簡化版」，Jin 本人判斷更細
- 支援正則表達式匹配（如「.*節」匹配各種節慶）
- 兩關篩選：初級（消極汰除）→ 二級（積極挑選）

## Notion 整合

### 寫入 API

```typescript
// /api/notion/create-case/route.ts
POST /api/notion/create-case
{
  token: string;                    // Notion Integration Token
  databaseId: string;               // 目標資料庫 ID
  fields: {
    標案名稱: string;               // title property
    招標機關: string;               // text
    案號: string;                    // text
    預算金額: number;               // number
    截標時間: string;               // date (ISO format)
    公告日: string;                  // date
    標案類型: string[];              // multi_select
    評審方式: string;               // select
    決標方式: string;               // select
    // ... 更多欄位
  };
  autoFields?: {
    標案進程: "To-do";             // 預設狀態
    備標決策: "入選";              // 預設決策
  };
}
→ { pageId: string, url: string }
```

### PCC → Notion 欄位映射

| PCC API 欄位 | Notion 欄位 | 型別 | 說明 |
|-------------|-------------|------|------|
| `brief.title` | 標案名稱 | title | 直接對應 |
| `brief.unit_name` | 招標機關 | text | 直接對應 |
| `brief.job_number` | 案號 | text | 直接對應 |
| `brief.budget` | 預算金額 | number | 解析數字 |
| `detail.deadline` | 截標時間 | date | 解析日期 |
| `brief.publish_date` | 公告日 | date | 解析日期 |
| `brief.category` | 標案類型 | multi_select | 對應到預設選項 |
| `detail.award_method` | 評審方式 | select | 對應到預設選項 |
| `detail.tender_method` | 決標方式 | select | 對應到預設選項 |
| 自動產生 | 案件唯一碼 | text | `PCC-{案號}` |
| 預設值 | 標案進程 | status | `To-do` |
| 預設值 | 備標決策 | select | `入選` |

**注意**：Notion 的 89 個欄位中，建案時只填有資料的，其餘留空由企劃後續補齊。

### Notion API 呼叫

```typescript
// Notion REST API: Create a page
async function createNotionCase(token: string, databaseId: string, properties: object) {
  const res = await fetch("https://api.notion.com/v1/pages", {
    method: "POST",
    headers: {
      "Authorization": `Bearer ${token}`,
      "Notion-Version": "2022-06-28",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      parent: { database_id: databaseId },
      properties,
    }),
  });
  if (!res.ok) throw new Error(`Notion API ${res.status}`);
  return res.json();
}
```

## 實作分期

### Phase 1：關鍵字篩選引擎 + 巡標 API

**產出**：`/api/scan` + 關鍵字引擎 + 型別

- `src/lib/scan/types.ts`（ScanResult、Classification、KeywordRule）

- `/api/scan/route.ts`（呼叫 PCC API → 篩選 → 回傳分類結果）
- 測試：每個分類的 happy path + 邊界
- Feature Registry 註冊 `scan`

**前置條件**：無（PCC API 已就緒）

### Phase 2：Notion 建案 API

**產出**：`/api/notion/create-case` + 欄位映射

- `src/lib/notion/client.ts`（Notion API 封裝：create page）
- `src/lib/notion/types.ts`（Notion property 型別）
- `src/lib/scan/notion-mapper.ts`（PCC → Notion 欄位對應）
- `/api/notion/create-case/route.ts`
- 測試：建案 + 欄位映射 + 錯誤處理
- Settings 擴充：Notion token + DB ID

**前置條件**：確認 Notion 資料庫 ID + Integration 權限

### Phase 3：巡標 UI

**產出**：`/scan` 頁面 + 元件

- `src/app/scan/page.tsx`
- `src/components/scan/`（ScanDashboard、TenderCard、CreateCaseDialog）
- `src/lib/scan/useScanResults.ts`（結果管理 Hook）
- 設定面板：關鍵字管理 + Notion 設定
- 整合測試

**前置條件**：Phase 1 + Phase 2

### Phase 4：情報自動觸發 + 回寫

**產出**：建案後自動跑情報 → 回寫 Notion

- `/api/notion/update-case/route.ts`（回寫用）
- 建案後自動觸發：競爭分析、機關情報、市場趨勢
- 情報結果存入 Notion 頁面 blocks（或自定義欄位）
- 案件摘要自動產生

**前置條件**：Phase 3 + 確認情報結果存在 Notion 的哪裡

### Phase 5：排程 + Google Drive（可選）

**產出**：定時自動掃描 + Drive 整合

- 排程選項：
  - 方案 A：Vercel Cron Jobs（簡單，免費額度內）
  - 方案 B：用戶自行設定 cron（彈性）
  - 方案 C：先手動觸發（Phase 1 已支援），排程後做
- Google Drive 資料夾建立（需 Google API 金鑰）

**前置條件**：Phase 3 跑通，用戶確認排程方案
