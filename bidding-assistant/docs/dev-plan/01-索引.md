# 01 索引 — 開發計畫文件清單

## 文件管理規則

1. **每份文件開頭有 YAML frontmatter**（version, updated, status, depends, changelog）
2. **討論有結論 → 主動問使用者是否更新對應文件**
3. **修改時 show diff** — 列出「舊內容 → 新內容」對比，使用者確認才寫入
4. **版號規則**：`v1.0` → `v1.1`（小改）→ `v2.0`（結構性大改）
5. **整合校正**：修改任何一本後，檢查本索引是否需同步更新
6. **狀態值**：`定案` | `草案` | `待寫` | `修訂中`

## 暫存與寫入指令

### `/暫存`

將本次討論的結論整理成暫存檔，存到 `docs/dev-plan/_staging/`。

- **檔名格式**：`{目標文件編號}-{主題關鍵詞}.md`（如 `03-Agent成本方案.md`、`M01-快取改IndexedDB.md`）
- **每份暫存檔必須包含**：
  - 目標文件（對應哪份正式文件）
  - 操作類型（新增 / 修改 / 刪除）
  - 具體內容（要寫入的文字）
- **不修改任何正式文件**

### `/寫入計畫`

將暫存區的結論正式寫入開發計畫文件。

1. 掃描 `docs/dev-plan/_staging/` 所有暫存檔
2. 列出清單讓使用者確認範圍（可選擇只寫入其中幾份）
3. 若多份暫存檔指向同一目標文件 → 先整合，標出衝突點讓使用者裁決
4. 逐筆顯示新舊 diff，使用者確認一筆才寫一筆
5. 每筆寫入後更新該文件的 `version` / `updated` / `changelog`
6. 全部完成後同步更新 `01-索引.md`
7. 已寫入的暫存檔 → 移到 `_staging/_done/`；未寫入的 → 保留原位

### 暫存檔範例

```markdown
# 暫存：修改 Agent SDK 計費方案

- **目標文件**：03-技術選型.md
- **操作**：修改 ADR-04
- **來源對話**：Discord Bot 架構討論（2026-02-18）

## 內容

將 ADR-04 Agent 架構的「待決」改為方案 C：
- 單一 Agent + 不同 system prompt
- 月費估算：~$30/月
- 理由：...
```
