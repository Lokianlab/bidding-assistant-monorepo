## 一、功能概述

### 目標
自動產出工作計畫書，追蹤里程碑進度，生成定期報告，讓履約過程有文件支撐和進度可見性。

### 核心使用場景
1. 案件得標後進入「履約管理」分頁
2. 系統 AI 自動根據標案規格生成「工作計畫書初版」（含分週里程碑）
3. 用戶逐條標記里程碑完成狀態
4. 系統定期生成履約進度報告，並在 dashboard 顯示逾期警示

### 時間節點
- 觸發時機：案件狀態變更為「已得標」
- 計畫期限：通常 3-12 個月（取決於案件類型）
- 進度更新：每週或每月一次用戶手動標記 + AI 自動摘要

## 三、功能清單

### 3.1 工作計畫書生成（AI 輔助）

**觸發**：用戶點擊「生成計畫書」
**輸入**：案件 ID → 從 Notion 讀取案件規格（標案名、期限、業主要求等）
**處理**：
- 呼叫 `/api/ai/generate-workplan`
- Claude 根據案件規格產生 JSON 里程碑清單（8-12 項）
- 每項包括：里程碑名稱 / 預計開始 / 預計結束 / 負責人 / 預期成果描述

**輸出**：
```json
{
  "milestones": [
    {
      "id": "M1",
      "name": "需求訪談與規格確認",
      "start_date": "2026-03-05",
      "end_date": "2026-03-15",
      "owner": "PM",
      "description": "與業主完成 5 次訪談，確認系統需求文件"
    },
    // ... 更多里程碑
  ]
}
```

### 3.2 里程碑管理

**CRUD 操作**：
- **新增**：POST `/api/cases/[id]/milestones/`
- **列表**：GET `/api/cases/[id]/milestones/`
- **編輯**：PATCH `/api/cases/[id]/milestones/[milestone-id]/`
- **刪除**：DELETE `/api/cases/[id]/milestones/[milestone-id]/`
- **進度標記**：PATCH `/api/cases/[id]/milestones/[milestone-id]/progress/`

**進度狀態**：
- `planned`（計畫中）
- `in_progress`（進行中）
- `completed`（已完成）
- `at_risk`（風險）
- `delayed`（逾期）

### 3.3 進度追蹤 UI

**頁面**：`/app/case/[id]/fulfillment/`

**佈局**：
1. **進度面板**（頂部）
  - L9-L12 四層進度條
  - 整體完成率百分比
  - 最後更新時間

2. **里程碑清單**（中央）
  - 表格顯示：里程碑名、日期、負責人、狀態、進度條
  - 可拖拽排序
  - 支援批量標記

3. **警示面板**（右側）
  - 逾期里程碑列表（紅色警示）
  - 即將到期里程碑（黃色警示）
  - 快捷操作（標記為完成 / 延期）

### 3.4 定期報告生成（AI）

**觸發**：
- 用戶手動點擊「生成報告」
- 系統自動 cron 每月 1 日自動生成

**輸入參數**：
- 案件 ID
- 時間區間（起迄日期）
- 報告類型（月報 / 季報 / 最終）

**報告內容**（AI 生成）：
```
【履約進度報告】
案件：[案件名稱] | 期間：[時間區間]

1. 完成里程碑（3 項）
   - 需求訪談與規格確認 ✓
   - 系統設計與架構評估 ✓
   - ...

2. 進行中里程碑（2 項）
   - 開發環境建置 (40%)
   - ...

3. 風險與延期（1 項）
   - [風險說明]

4. 下月計畫（2 項）
   - [計畫項目]
```

**輸出**：PDF 檔案（可直接寄送客戶）

### 3.5 Dashboard 整合

**位置**：Case-work 儀表板「履約」分頁

**顯示內容**：
- 本案整體進度（環形進度條）
- 關鍵里程碑狀態（列表或卡片）
- 警示面板（逾期 / 風險項目）
- 快速操作按鈕（生成報告、編輯計畫、標記進度）

## 四、技術規格

### 4.1 後端結構

**新增資料夾**：`/src/lib/fulfillment/`

```
src/lib/fulfillment/
  types.ts           ← 型別定義（Milestone, WorkPlan, FulfillmentStatus）
  helpers.ts         ← 純函式（計算進度、判斷風險、生成報告摘要）
  useFulfillment.ts  ← React Hook（讀取 / 更新里程碑、監聽進度變化）
  __tests__/         ← 測試檔案（≥80 tests）
```

### 4.2 API 路由

**新增路由**：`/src/app/api/cases/[id]/milestones/`

```
GET    /api/cases/[id]/milestones/
       → 列表所有里程碑（支援分頁、篩選狀態）

POST   /api/cases/[id]/milestones/
       → 建立新里程碑（輸入：name, start_date, end_date, owner, description）

PATCH  /api/cases/[id]/milestones/[milestone-id]/
       → 編輯里程碑資訊

DELETE /api/cases/[id]/milestones/[milestone-id]/
       → 刪除里程碑

PATCH  /api/cases/[id]/milestones/[milestone-id]/progress/
       → 標記進度（輸入：status, completion_percent, notes）

GET    /api/cases/[id]/fulfillment/report/
       → 生成履約報告（參數：start_date, end_date, report_type）
```

### 4.3 Supabase 資料結構

**新增表**：`case_milestones`

```sql
CREATE TABLE case_milestones (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  case_id UUID NOT NULL REFERENCES cases(id),
  tenant_id UUID NOT NULL REFERENCES users(id),

  name TEXT NOT NULL,
  start_date DATE NOT NULL,
  end_date DATE NOT NULL,
  owner TEXT,
  description TEXT,

  status TEXT DEFAULT 'planned',  -- planned, in_progress, completed, at_risk, delayed
  completion_percent INT DEFAULT 0,  -- 0-100
  notes TEXT,

  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now(),

  CONSTRAINT no_duplicate_name UNIQUE(case_id, name),
  INDEX idx_case_milestones_tenant (tenant_id)
);

CREATE TABLE fulfillment_reports (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  case_id UUID NOT NULL REFERENCES cases(id),
  tenant_id UUID NOT NULL REFERENCES users(id),

  report_type TEXT,  -- 'monthly', 'quarterly', 'final'
  start_date DATE,
  end_date DATE,
  content TEXT,  -- AI 生成的報告內容（JSON）

  created_at TIMESTAMP DEFAULT now(),

  INDEX idx_fulfillment_reports_tenant (tenant_id)
);
```

### 4.4 RLS 策略

```sql
-- case_milestones 多租戶隔離
CREATE POLICY "Users can view own tenant's milestones"
  ON case_milestones
  USING (tenant_id = auth.uid());

-- fulfillment_reports 多租戶隔離
CREATE POLICY "Users can view own tenant's reports"
  ON fulfillment_reports
  USING (tenant_id = auth.uid());
```

```
PATCH /api/cases/[id]/milestones/[milestone-id]/progress

Response (200):
{ "success": true, "updated_at": "2026-02-23T10:00:00Z" }
```

```
GET /api/cases/[id]/fulfillment/report?start_date=2026-02-01&end_date=2026-02-28&type=monthly

{
```
