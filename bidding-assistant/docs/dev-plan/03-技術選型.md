## ADR-01：前端框架 — Next.js 16

- **決定**：Next.js 16 + App Router + React 19 + TypeScript + Tailwind CSS 4
- **理由**：已有完整專案基礎（560 tests, 10 features, 23 routes），無理由換框架
- **否決**：Remix（生態系小）、Nuxt（Vue 學習成本）、純 React SPA（失去 SSR 和 API Routes）

## ADR-03：向量資料庫 — 現階段不需要

- **決定**：暫不建向量資料庫
- **理由**：
  - 知識庫（00A-00E）預估 ~72,000 tokens = 1M context 的 7%，直接塞進 context 即可
  - Notion 資料庫有 10,000+ 筆，但查詢用 Notion API structured filter，不需要向量搜尋
  - 「找相似歷史案件」目前無意義（0 得標，無 00E 檢討回饋）
- **觸發條件**：知識庫超過 200K tokens 塞不進 context，或 00E 累積 20+ 案需要相似案件比對時，重新評估
- **否決（當下）**：Pinecone / pgvector / Chroma — 架構過度複雜，收益不明

## ADR-04：Agent 架構 — 雙層策略

| 層 | 用途 | 技術 | 誰用 |
|----|------|------|------|
| **施工層** | 開發系統時多 agent 並行寫程式 | Claude Code Agent Team | 開發者 |
| **產品層** | SaaS 網頁背後的 AI 編制 | Claude Code SDK (TypeScript) | 企劃團隊（透過 SaaS 網頁） |

> 目前以 SaaS 網頁為主介面。Discord Bot 未來可作為輕量操作入口擴充（如快速查詢、通知推播）。

### 產品層 Agent 編制

| Agent | 角色 | 負責 | 使用的 prompt |
|-------|------|------|-------------|
| 戰略官 | 分析與決策 | L1 戰略分析、投標決策、案件拆解 | L1 v2.3 + P 報告 |
| 企劃官 | 規劃與撰寫 | L2 備標規劃、L3 章節撰寫 | L2 + L3 + 撰寫規範 |
| 品管官 | 品質把關 | 三道閘門、L4 審查 | L4 + 品質規則 |
| 知識官 | 知識管理 | 知識庫檢索、注入、回流 | T1 + 00A-00E |
| 簡報官 | 表達與演練 | L5-L8 | L5-L8 |

### 成本控制

- **已決**：產品層用 Claude Code SDK，走 Max 訂閱（不走 API 計費），額外成本 = $0，與 ADR-02 一致
- 必須實作用量監控（token 數 + 回應時間 + 限速事件）
- 限速主動報告（偵測到 rate limit 時通知管理者）
- 輕量查詢不啟動 Agent，直接查資料庫

## ADR-05：公司識別方式 — 統編優先

- **決定**：所有交叉比對、統計、分組以統編（8位數字）為 key
- **理由**：PCC API 中公司名稱有多種寫法（如有/無英文名、空白差異），但統編永遠不變
- **否決**：模糊名稱比對（不可靠、需維護同義詞表）
- **名稱正規化**：僅用於 UI 顯示和搜尋，不用於身份判斷

## ADR-06：情報資料來源 — g0v PCC API

- **決定**：使用 g0v PCC API (`https://pcc-api.openfun.app`) 作為核心情報來源
- **理由**：免費、無認證、開放 CORS、資料涵蓋完整（得標者、未得標者、評委、金額）
- **否決**：直接爬政府電子採購網（違反 ToS、維護成本高、反爬機制）
- **補充**：API 可取代 P 偵察報告 ~80% 的人工搜尋，Perplexity 只需補軟情報

## ADR-07：操作介面定位 — SaaS 網頁

- **決定**：SaaS 網頁 = 主要產品介面（提案寫作駕駛艙），Web app = 管理後台
- **理由**：SaaS 網頁整合度更高、操作體驗更完整，能承載完整的 L1-L8 流程
- **暫不採用**：Discord Bot — 整合受限於 API 彈性不足（15 分鐘逾時、25MB 上傳限制），但保留未來作為輕量入口的擴充可能（快速查詢、通知推播）
- **否決**：LINE Bot（API 限制多）、Slack（額外訂閱費）

### 決策脈絡

原本評估時 Gemini CLI 和 Codex CLI 都標「不裝」，後來改為全部裝齊。決策原則：**既然都有現有訂閱，CLI 安裝零成本、零維護（npm install -g），沒理由不裝。** 多一個備援 AI 永遠比少一個好。

### 完整工具生態系

```
P（Perplexity 網頁版）→ 外部情報（P 偵察報告）
C（Claude Code, Opus 4.6, 1M）→ 文件分析 + 策略推理 + L1-L8
Gemini CLI（Ultra, 1M）→ 備援 AI（研究/長上下文）
Codex CLI（Pro）→ 備援 AI（寫作/創意）
Napkin AI（網頁版）→ 資訊圖表
Nano Banana（Gemini 網頁版）→ 視覺概念圖/插畫
Notion MCP（Claude.ai 內建）→ 案件資料庫 + 知識庫
Canva MCP（Claude.ai 內建）→ 設計/排版/簡報
NotebookLM MCP（.mcp.json，npx 自動拉）→ PDF 精準檢索
```

額外安裝的 CLI/MCP = 2（Gemini CLI + Codex CLI），額外費用 = $0（走現有訂閱）。

## ADR-10：自建 Notion MCP — 暫緩

- **決定**：暫時不自建 Notion MCP server
- **理由**：Claude.ai 內建的 Notion MCP 已涵蓋完整 CRUD（搜尋、讀寫頁面、建資料庫、查詢 view），目前只有 Claude 在用 Notion
- **觸發條件**：當 GPT / Gemini agent 也需要存取 Notion 時，再建本地版 Notion MCP
- **否決（當下）**：立即自建 — 功能重複、維護成本不值得
