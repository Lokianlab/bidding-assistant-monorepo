## Jin 的核心指示（永久有效）

1. 給判斷，不要丟選擇題
2. 不要套用爛模板，品質提升 > 速度提升
3. 每一步都要存，過程即產出
4. 我們互動的過程就是系統的原型
5. Google Drive 串接不能跳過（一鍵上新 = Notion + Drive 同時）
6. Notion 模板 Step 0-9 是舊版參考，正式體系用 L1-L8

## 技術發現（踩過的坑）

- **Notion formula/rollup 欄位**不能放進 API 的 filter_properties，會報錯。解法：resolvePropertyIds 自動排除 + 失敗降級重試。（s1 0224）
- **Notion token** 差一個字（82→86）導致長期連不上，排查時要逐字比對。（s1 0224）
- **Middleware** 在開發模式需要跳過 OAuth，否則本機測試全擋。（s1 0224）

## 架構決策

- **備標自動化系統**：3 個 Phase（Phase 1 自動偵察+L1 → Phase 2 備標規劃+企劃骨架 → Phase 3 報價+品質檢核）。（s1 0224）
- **12 個協作模組**：M-ENV, M-SCAN, M-FUNNEL, M-ASSET, M-COMPARE, M-PROBE, M-CASE, M-READ, M-INTEL, M-SAVE, M-ASSEMBLE, M-L1。詳見 `協作模組定義-20260224.md`。（s1 0224）
- **提示詞組裝引擎**：`src/data/config/prompt-assembly.ts`，已有 KB 引用矩陣，擴展為自動組裝。（s1 0224）
