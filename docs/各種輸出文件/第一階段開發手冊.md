## 一、核心需求（為什麼要做）

**一句話：做對投/不投的決定。**

就算後續不再擴充，光是這一點就能改善整體時間投入和命中率。

現狀：
- 46 案/年，0% 命中率
- 每案手動查情報花數小時（PCC 網站翻找、Google 搜尋、問人）
- 決定投不投靠直覺，沒有系統性情報支撐
- 決定「投」之後，手動建檔、開資料夾、填資料再花 30 分鐘
- 找過去的素材（簡報頁、企劃段落）每次翻找 20 分鐘

目標：
- 快速判斷該不該投，不再浪費時間在打不贏的案子上
- 情報品質決定企劃品質，企劃品質決定投標命中率（因果鏈，非順序問題）
- 這是情報部門的完整 MVP，企劃部門（寫企劃、做簡報）留在第二刀

|------|------|------|-------------|

## 三、使用者動線（用起來長什麼樣）

從使用者角度走一遍完整流程，每一步標出對應的工作項編號。

### 場景一：發現新案 → 決定投不投

```
1. 打開系統首頁
   → 看到案件清單、待決策、時限警示                          [D3]
   → 新案推薦區：系統自動巡標發現的案件

2. 看到一個有興趣的案件 → 點進案件情報頁
   → 案件基本資料已自動填入（PCC 拉的）                      [A1]
   → 機關歷史：該機關過去 3 年同類案件、常用廠商              [A1]
   → 競爭者分析：過去得標者、連續得標年數、累計金額           [A1]

3. 上傳 RFP（招標文件）
   → 拖曳上傳 PDF                                            [A4-補充]
   → AI 自動解析：評分項目、權重、重點需求、隱藏需求          [A3]

4. 看到 Perplexity 提示詞（假設驗證式）
   → 複製提示詞 → 去 Perplexity 跑 → 貼結果回來             [A5, A7-補充]
   → 系統解析結果，更新情報頁

5. 勝算五項檢核
   → 能自動填的已填（PCC 數據）、不能的標紅（待人工補充）     [A6]
   → 五項各有紅/黃/綠燈                                      [A6]
   → 整體投標建議                                            [A6]

6. Jin 看完情報摘要和勝算報告
   → 決定「投」→ 進場景二
   → 決定「不投」→ 記錄理由，歸檔                            [D5]
```

### 場景二：決定「投」→ 一鍵建案

```
7. Jin 按下「投」
   → 系統記錄決策（投/理由/時間）                             [D5]
   → Notion 自動建案，填入所有已知資料                        [B1, B2]
   → Drive 自動建資料夾，套用鷹架範本                         [B3, B4]
   → 範本預填已知資料                                        [B5]
   → 完成，打開案件資料夾，一切就位
```

### 場景三：找過去的素材

```
8. 企劃人員需要找過去做過的類似內容
   → 進入知識庫搜尋                                          [C5]
   → 三種找法：
     a. 自動推薦（根據當前案件類型）                          [C6-補充]
     b. 關鍵字搜尋（像 Google 一樣打字找）                    [C5]
     c. 按分類瀏覽（展覽/活動/教育 等樹狀結構）               [C5]
   → 找到卡片 → 看文字摘要 → 點開原檔手動複製
   → （第二刀才做自動搬入）
```

### Supabase Schema 設計

第一階段需要三張新表，加上現有的 `kb_entries` 表。

#### 表一：`intelligence_cache`（情報快取）

存放 PCC 拉取的情報數據，避免重複呼叫 API。

```sql
CREATE TABLE intelligence_cache (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  case_id TEXT NOT NULL,              -- 案件唯一碼（對應 Notion）
  intel_type TEXT NOT NULL,           -- 'agency_history' | 'competitor' | 'rfp_summary' | 'perplexity' | 'win_assessment'
  data JSONB NOT NULL,                -- 情報內容（結構依 intel_type 不同）
  source TEXT,                        -- 資料來源（'pcc' | 'perplexity' | 'rfp_upload' | 'manual'）
  pcc_unit_id TEXT,                   -- PCC 機關代碼（方便跨案件查詢同機關）
  expires_at TIMESTAMPTZ,             -- 快取過期時間（PCC 數據 7 天、Perplexity 30 天）
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- 索引
CREATE INDEX idx_intel_case ON intelligence_cache(case_id);
CREATE INDEX idx_intel_type ON intelligence_cache(case_id, intel_type);
CREATE INDEX idx_intel_unit ON intelligence_cache(pcc_unit_id);
CREATE INDEX idx_intel_expires ON intelligence_cache(expires_at);
```

`data` JSONB 結構依 `intel_type`：


#### 表二：`knowledge_cards`（知識庫卡片）

存放 Drive 檔案解析後的卡片索引。

```sql
CREATE TABLE knowledge_cards (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  source_file_id TEXT NOT NULL,       -- Google Drive file ID
  source_file_name TEXT NOT NULL,     -- 原始檔名
  source_folder_path TEXT,            -- Drive 中的資料夾路徑
  page_number INT,                    -- 第幾頁（簡報）或段落序號（文件）
  card_type TEXT NOT NULL,            -- 'slide' | 'document_section' | 'image' | 'table'
  title TEXT,                         -- 卡片標題（AI 生成）
  summary TEXT NOT NULL,              -- 文字摘要（AI 生成）
  content_text TEXT,                  -- 原始文字內容（全文搜尋用）
  tags TEXT[] DEFAULT '{}',           -- 分類標籤（AI 生成）
  category TEXT,                      -- 一級分類（'展覽' | '活動' | '教育' | '文化' | '設計' | ...）
  subcategory TEXT,                   -- 二級分類（'藝文' | '商業' | '社區' | ...）
  file_type TEXT,                     -- 'pptx' | 'pdf' | 'docx' | 'xlsx'
  mime_type TEXT,
  is_scannable BOOLEAN DEFAULT true,  -- false = 掃描 PDF，無法解析
  scan_error TEXT,                    -- 解析失敗的原因
  drive_url TEXT,                     -- 直接開啟原檔的 URL
  indexed_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- 索引
CREATE INDEX idx_cards_source ON knowledge_cards(source_file_id);
CREATE INDEX idx_cards_category ON knowledge_cards(category, subcategory);
CREATE INDEX idx_cards_tags ON knowledge_cards USING gin(tags);
CREATE INDEX idx_cards_search ON knowledge_cards
  USING gin(to_tsvector('chinese', COALESCE(title, '') || ' ' || summary || ' ' || COALESCE(content_text, '')));
CREATE INDEX idx_cards_type ON knowledge_cards(card_type);
```

#### 表三：`decisions`（投/不投決策記錄）

```sql
CREATE TABLE decisions (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  case_id TEXT NOT NULL,              -- 案件唯一碼
  notion_page_id TEXT,                -- Notion 頁面 ID
  decision TEXT NOT NULL,             -- 'bid' | 'no_bid' | 'conditional'
  reason TEXT,                        -- 決策理由
  win_assessment_id UUID REFERENCES intelligence_cache(id),  -- 關聯的勝算評估
  decided_by TEXT DEFAULT 'Jin',
  decided_at TIMESTAMPTZ DEFAULT now(),
  -- 決策後的行動記錄
  notion_created BOOLEAN DEFAULT false,
  drive_folder_id TEXT,
  drive_folder_url TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_decisions_case ON decisions(case_id);
CREATE INDEX idx_decisions_date ON decisions(decided_at);
```

#### B4：鷹架範本複製

**做什麼**：將 Drive 中「BZ. 製標鷹架」的範本結構複製到新案件資料夾。

**怎麼做**：

```typescript
// src/lib/case-setup/drive-setup.ts

/**
 * 遞迴複製 Drive 資料夾結構
 *
 * 1. 列出來源資料夾的所有子項目
 * 2. 資料夾 → 在目標建同名資料夾，遞迴
 * 3. 檔案 → 複製到目標資料夾
 */
async function copyFolderRecursive(
  sourceFolderId: string,
  targetFolderId: string,
  accessToken: string,
): Promise<{ copied: number; errors: string[] }> {
  const items = await listDriveFolder(sourceFolderId, accessToken);
  let copied = 0;
  const errors: string[] = [];

  for (const item of items) {
    try {
      if (item.mimeType === 'application/vnd.google-apps.folder') {
        // 建子資料夾，遞迴
        const newFolder = await createGoogleDriveFolder(item.name, targetFolderId, accessToken);
        const sub = await copyFolderRecursive(item.id, newFolder.id!, accessToken);
        copied += sub.copied;
        errors.push(...sub.errors);
      } else {
        // 複製檔案
        await copyDriveFile(item.id, targetFolderId, accessToken);
        copied++;
      }
    } catch (err) {
      errors.push(`${item.name}: ${err instanceof Error ? err.message : '未知錯誤'}`);
    }
  }

  return { copied, errors };
}

/** 列出 Drive 資料夾內容 */
async function listDriveFolder(folderId: string, accessToken: string) {
  const res = await fetch(
    `https://www.googleapis.com/drive/v3/files?q='${folderId}'+in+parents&supportsAllDrives=true&includeItemsFromAllDrives=true&fields=files(id,name,mimeType)`,
    { headers: { Authorization: `Bearer ${accessToken}` } }
  );
  const data = await res.json();
  return data.files ?? [];
}

/** 複製 Drive 檔案 */
async function copyDriveFile(fileId: string, targetFolderId: string, accessToken: string) {
  await fetch(
    `https://www.googleapis.com/drive/v3/files/${fileId}/copy?supportsAllDrives=true`,
    {
      method: 'POST',
      headers: {
        Authorization: `Bearer ${accessToken}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ parents: [targetFolderId] }),
    }
  );
}
```

```typescript
// src/lib/intelligence/pcc-intel.ts

import { pccApiFetch } from '@/lib/pcc/api';
import type { PCCSearchResponse, PCCTenderDetail } from '@/lib/pcc/types';

/**
 * 拉取機關歷史情報
 *
 * 1. 用機關代碼搜尋該機關過去 3 年的決標公告
 * 2. 整理得標者、金額、投標家數
 * 3. 找出最常得標廠商和連續得標年數
 * 4. 存入 intelligence_cache
 */
async function fetchAgencyHistory(unitId: string, unitName: string): Promise<AgencyHistoryData> {
  // 搜尋該機關的歷史標案
  const searchResult = await pccApiFetch<PCCSearchResponse>('listByUnit', {
    unit_id: unitId,
    page: 1,
  });

  const cases = [];
  for (const record of searchResult.records) {
    if (record.brief.type === '決標公告') {
      // 拉取詳細資料
      const detail = await pccApiFetch<PCCTenderDetail>('getTenderDetail', {
        url: record.tender_api_url,
      });
      // 解析得標者、金額等
      cases.push(parseTenderDetail(record, detail));
    }
  }

  // 統計最常得標廠商
  const topWinners = analyzeTopWinners(cases);

  return { unit_id: unitId, unit_name: unitName, total_cases: cases.length, cases, top_winners: topWinners };
}

/**
 * 拉取競爭者分析
 *
 * 1. 從機關歷史中找出所有得標者
 * 2. 對每個得標者搜尋其在其他機關的得標記錄
 * 3. 計算競爭者的專長領域和實績強度
 */
async function fetchCompetitorAnalysis(agencyHistory: AgencyHistoryData): Promise<CompetitorData> {
  const competitors = [];
  for (const winner of agencyHistory.top_winners) {
    // 用公司名稱搜尋其全部得標記錄
    const companyRecords = await pccApiFetch<PCCSearchResponse>('searchByCompany', {
      company: winner.name,
      page: 1,
    });
    competitors.push(analyzeCompetitor(winner, companyRecords));
  }
  return { competitors };
}
```

```typescript
/**
 * RFP 解析流程：
 * 1. 前端上傳 PDF/DOCX
 * 2. 用 pdf-parse 或 mammoth 提取文字
 * 3. 呼叫 Claude Code（同機，CLI 方式）解析
 * 4. 結果存入 intelligence_cache（intel_type = 'rfp_summary'）

import { exec } from 'child_process';
import { promisify } from 'util';

const execAsync = promisify(exec);

async function parseRFP(filePath: string, caseId: string): Promise<RFPSummaryData> {
  // Step 1: 提取文字
  const text = await extractText(filePath);

  // Step 2: 呼叫 Claude Code 解析
  // 同機呼叫，用現有訂閱登入，不需 API key
  const prompt = buildRFPAnalysisPrompt(text);

  // 使用 Claude CLI（已登入的 session）
  const { stdout } = await execAsync(
    `echo "${escapeForShell(prompt)}" | claude --print`,
    { maxBuffer: 1024 * 1024 * 10 }  // 10MB buffer
  );

  // Step 3: 解析 Claude 回傳的結構化結果
  const parsed = parseClaudeResponse(stdout);

  // Step 4: 存入快取
  await saveIntelCache(caseId, 'rfp_summary', parsed);

  return parsed;
}
```

#### C1：Drive 檔案掃描

**做什麼**：掃描 Google Drive 指定資料夾，列出所有檔案。

**怎麼做**：

```typescript
// src/lib/knowledge/drive-scanner.ts

interface DriveFile {
  id: string;
  name: string;
  mimeType: string;
  size: number;
  folderPath: string;
  modifiedTime: string;
  webViewLink: string;
}

/**
 * 遞迴掃描 Drive 資料夾
 *
 * 目標資料夾（環境變數）：
 * - GOOGLE_KB_FOLDER_IDS = 逗號分隔的 folder IDs
 *   包含：BA. 企劃認領、C. 執行期間、A. 配套資料
 */
async function scanDriveFolders(): Promise<DriveFile[]> {
  const folderIds = (process.env.GOOGLE_KB_FOLDER_IDS ?? '').split(',').filter(Boolean);
  const accessToken = await getGoogleAccessToken();
  const allFiles: DriveFile[] = [];

  for (const folderId of folderIds) {
    await scanFolderRecursive(folderId, '', accessToken, allFiles);
  }

  return allFiles;
}

async function scanFolderRecursive(
  folderId: string,
  parentPath: string,
  accessToken: string,
  results: DriveFile[],
): Promise<void> {
  const items = await listDriveFolder(folderId, accessToken);

  for (const item of items) {
    const currentPath = parentPath ? `${parentPath}/${item.name}` : item.name;

    if (item.mimeType === 'application/vnd.google-apps.folder') {
      await scanFolderRecursive(item.id, currentPath, accessToken, results);
    } else {
      results.push({
        id: item.id,
        name: item.name,
        mimeType: item.mimeType,
        size: item.size,
        folderPath: parentPath,
        modifiedTime: item.modifiedTime,
        webViewLink: item.webViewLink,
      });
    }
  }
}
```

**怎麼做**：

```typescript
// src/lib/knowledge/text-extractor.ts

/**
 * 支援的檔案類型：
 * - PPTX → 逐頁提取（每頁一張卡片）
 * - PDF → 文字型嘗試提取，掃描型標記「無法解析」
 * - DOCX → 逐段提取
 * - XLSX → 逐 sheet 提取
 *
 * 依賴套件：
 * - pdf-parse（PDF 文字提取）
 * - mammoth（DOCX → 文字）
 * - pptx-parser 或 officegen（PPTX 解析）
 * - xlsx（Excel 解析）
 */

interface ExtractedPage {
  page_number: number;
  text: string;
  card_type: 'slide' | 'document_section' | 'table';
  is_scannable: boolean;
  scan_error?: string;
}

async function extractFromFile(
  filePath: string,
  mimeType: string,
): Promise<ExtractedPage[]> {
  switch (mimeType) {
    case 'application/vnd.openxmlformats-officedocument.presentationml.presentation':
      return extractPPTX(filePath);
    case 'application/pdf':
      return extractPDF(filePath);
    case 'application/vnd.openxmlformats-officedocument.wordprocessingml.document':
      return extractDOCX(filePath);
    default:
      return [{ page_number: 1, text: '', card_type: 'document_section',
                is_scannable: false, scan_error: `不支援的格式：${mimeType}` }];
  }
}

async function extractPDF(filePath: string): Promise<ExtractedPage[]> {
  const pdfParse = require('pdf-parse');
  const buffer = await fs.readFile(filePath);
  const data = await pdfParse(buffer);

  // 判斷是否為掃描 PDF：文字量極少但頁數多
  if (data.text.trim().length < 100 && data.numpages > 1) {
    return [{
      page_number: 1,
      text: '',
      card_type: 'document_section',
      is_scannable: false,
      scan_error: '掃描型 PDF，無法提取文字',
    }];
  }

  // 按頁分割（pdf-parse 不直接支援，用 \f 分割）
  const pages = data.text.split('\f').filter(Boolean);
  return pages.map((text: string, i: number) => ({
    page_number: i + 1,
    text: text.trim(),
    card_type: 'document_section' as const,
    is_scannable: true,
  }));
}
```

```typescript
 * - 輸出：標題、摘要、分類標籤
 *
 * 批次處理：每次送 5-10 頁，減少呼叫次數
 */
async function generateCards(
  pages: ExtractedPage[],
  sourceFile: DriveFile,
): Promise<KnowledgeCard[]> {
  const cards: KnowledgeCard[] = [];

  // 批次處理（每批 5 頁）
  for (let i = 0; i < pages.length; i += 5) {
    const batch = pages.slice(i, i + 5);
    const prompt = buildCardGenerationPrompt(batch, sourceFile);

    const { stdout } = await execAsync(
      `echo "${escapeForShell(prompt)}" | claude --print`,
      { maxBuffer: 1024 * 1024 * 10 }
    );

    const batchCards = parseCardResponse(stdout, batch, sourceFile);
    cards.push(...batchCards);
  }

  return cards;
}

function buildCardGenerationPrompt(
  pages: ExtractedPage[],
  source: DriveFile,
): string {
  return `你是知識庫索引專家。請為以下頁面建立知識卡片。每張卡片包含：
- title: 簡短標題（10 字以內）
- summary: 文字摘要（50 字以內）
- tags: 分類標籤陣列（如 ["展覽", "文化", "策展"]）
- category: 一級分類（展覽/活動/教育/文化/設計/行政/其他）
- subcategory: 二級分類

來源檔案：${source.name}
路徑：${source.folderPath}

回傳 JSON 陣列。

${pages.map((p, i) => `=== 第 ${p.page_number} 頁 ===\n${p.text}`).join('\n\n')}`;
}
```

**依賴**：C2（文字提取結果）

```typescript
async function saveCards(cards: KnowledgeCard[]): Promise<{ saved: number; errors: number }> {
  const supabase = createSupabaseServerClient();
  let saved = 0;
  let errors = 0;

  // 批次插入（每批 50 張）
  for (let i = 0; i < cards.length; i += 50) {
    const batch = cards.slice(i, i + 50);
    const { error } = await supabase
      .from('knowledge_cards')
      .upsert(batch, { onConflict: 'source_file_id,page_number' });

    if (error) {
      errors += batch.length;
      logger.error('knowledge', 'Card batch save failed', { error });
    } else {
      saved += batch.length;
    }
  }

  return { saved, errors };
}
```

## 十、風險和備案

| 風險 | 影響 | 備案 |
|------|------|------|
| **Supabase 7 天暫停** | 知識庫搜尋和情報快取失效 | cron job 每 3 天 ping 一次；失效時手動進 console 喚醒，資料不會丟 |
| **電腦重開機/斷電** | 服務中斷 | Task Scheduler 設定開機自動啟動 cloudflared + Next.js dev server |
| **PCC API 不穩/限速** | 情報拉取失敗或很慢 | 快取機制（intelligence_cache 7 天有效）；失敗重試 3 次；顯示「部分情報拉取失敗」不阻塞其他功能 |
| **PPTX/PDF 解析品質不一** | 知識卡片內容不完整 | 掃描 PDF 標記「無法解析」並在狀態看板顯示；PPTX 特效多的頁面只提取文字不保證格式 |
| **鷹架範本需要更新** | 舊範本建出的案件結構過時 | 鷹架是設定不是程式碼——Jin 直接在 Drive 裡改，下一次建案自動用新版 |
| **Claude Code CLI 呼叫不穩** | RFP 解析和卡片生成失敗 | 重試機制 + 失敗時降級為不解析（保留原始檔案連結） |
| **Drive API 配額限制** | 初始化掃描太快被限速 | 每次 API 呼叫之間加 delay（200ms）；初始化分批跑 |
| **知識庫初始化耗時** | 第一次建索引需要數小時 | 背景執行 + 進度看板；可以邊建索引邊用已完成的部分搜尋 |
| **Cloudflare Tunnel 斷線** | 外部無法存取 | cloudflared 自動重連；可設定 watchdog 監控 tunnel 狀態 |

## Claude Code CLI 呼叫注意事項

手冊中 RFP 解析和知識庫分類使用 `echo "..." | claude --print` 方式呼叫 Claude Code CLI。

如果 `--print` flag 不可用或行為異常，fallback 方案：
1. 改用 `claude -p "..."` 或 `claude --message "..."`（視 CLI 版本）
2. 如果 CLI pipe 不穩定，改為寫入臨時檔案再用 `claude < tmpfile`
3. 最差情況：該功能降級為手動（使用者自己貼到 Claude 跑，結果貼回系統）

開發時先確認 `claude --print` 在本機可用，不可用就調整呼叫方式。
