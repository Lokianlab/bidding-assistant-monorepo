## 一、PCC 情報搜尋

### 進入方式

點左側欄「情報搜尋」，或直接開 `localhost:3000/intelligence`。

### 操作步驟（Tab 1：案件搜尋）

1. 頁面預設在「案件搜尋」分頁，輸入框游標在搜尋列上。
2. 在輸入框輸入：`食農教育`
3. 按 Enter 或點「搜尋」按鈕。
4. 等結果出現（約 1–3 秒）。
5. 結果清單每筆顯示：案名、機關、金額、決標日期。
6. 點任一筆結果右側的「廠商分析」連結（或類似按鈕）。

### 預期結果

- 搜尋後出現多筆含「食農教育」的標案清單。
- 每筆有機關名稱與金額。
- 點「廠商分析」後，頁面切換到「競爭分析」分頁，且廠商名稱已自動帶入。

### 操作步驟（Tab 2：競爭分析）

1. 手動點上方「競爭分析」分頁。
2. 在廠商名稱欄輸入：`大員洛川`
3. 點「查詢」按鈕。
4. 等結果出現。

### 預期結果

- 顯示大員洛川的歷史投標紀錄。
- 每筆顯示：案名、是否得標、決標金額。
- 右下角或卡片底部有「機關分析」按鈕，點擊後切換到「評委分析」分頁。

### 操作步驟（Tab 3：市場趨勢）

1. 點上方「市場趨勢」分頁。
2. 在輸入框輸入：`導覽規劃`
3. 點「搜尋趨勢」或按 Enter。

### 預期結果

- 出現近年「導覽規劃」相關標案的趨勢圖或統計摘要。
- 顯示案件數量、平均金額等指標。

### 驗收通過條件

- [ ] 以「食農教育」搜尋，至少出現 1 筆結果，每筆有案名與機關。
- [ ] 以「大員洛川」查詢競爭分析，有顯示歷史投標記錄。
- [ ] 各分頁可切換，切換後內容正確載入，不出現空白或錯誤訊息。

## 二、M03 戰略分析

### 進入方式

點左側欄「戰略分析」，或直接開 `localhost:3000/strategy`。

### 操作步驟

1. 在「案件名稱」欄輸入（必填）：
   ```
   114年臺灣文化節活動策展計畫
   ```
2. 在「機關名稱」欄輸入：
   ```
   文化部
   ```
3. 在「預算金額」欄輸入：
   ```
   5000000
   ```
4. 點「開始分析」按鈕。

### 預期結果

- 按鈕點下後頁面不跳轉，在下方出現「分析結果」區塊。
- 出現「適配度評分卡」，顯示五個維度的評分（如：領域熟悉度、競爭強度、機關熟悉度、規模適配、知識庫匹配）。
- 每個維度有分數（0–100 或類似比例）與簡短說明。
- 底部有總分與結論（如：「建議投標」、「謹慎評估」或「不建議投標」）。
- 頁面最下方有「開始撰寫（進入提示詞組裝）」按鈕。

### 驗收通過條件

- [ ] 輸入案件名稱後點「開始分析」，出現適配度評分卡。
- [ ] 五個維度都有顯示分數（允許某些因知識庫無資料而為 0 或顯示提示）。
- [ ] 若知識庫為空，頁面會顯示黃色提示「⚠️ 知識庫尚無資料」而非錯誤當掉。
- [ ] 「開始撰寫」按鈕存在且可點擊（點後導向 `/assembly` 頁面，帶上案件名稱）。

## 三、M04 品質閘門

### 進入方式

點左側欄「工具」展開後點「品質閘門」，或直接開 `localhost:3000/tools/quality-gate`。

### 操作步驟

1. 在文字區域（大輸入框）貼上以下測試段落：

   ```
   本公司擁有豐富的策展經驗，曾多次承辦類似活動，深受各界好評。
   我們的團隊由資深專業人員組成，能夠提供最優質的服務。
   預算規劃完善，執行時間充裕，必能順利完成本計畫目標。
   ```

2. 點「開始檢查」按鈕。
3. 等分析完成（約 1–2 秒）。

### 預期結果

- 頁面下方出現「品質報告」區塊。
- 顯示四道閘門的摘要：
  - **文字品質**：偵測空洞描述、重複詞
  - **事實查核**：偵測無來源宣稱
  - **需求對照**：（無需求輸入時跳過，顯示「未設定需求清單」）
  - **實務檢驗**：基本常識性檢查
- 總分（0–100）與總評標籤（「通過」、「有風險」或「不建議提交」）。
- 若有關鍵問題，顯示紅色「關鍵問題（必須處理）」區塊，列出具體問題項目。

### 驗收通過條件

- [ ] 貼上測試段落後點「開始檢查」，1–3 秒內出現品質報告。
- [ ] 總分有數字（0–100 範圍內），總評標籤顯示正確。
- [ ] 文字品質分頁有具體的檢查項目（如偵測到「豐富」、「最優質」等空洞詞）。
- [ ] 右下角有「匯出文件（進入文件生成）」按鈕。
- [ ] 點「清除結果」後，報告消失，輸入框回到空白狀態。

## 四、情報探索（新功能）

### 進入方式

點左側欄「情報探索」，或直接開 `localhost:3000/explore`。

### 操作步驟（完整鑽取流程）

1. 初始畫面顯示搜尋列，頂部無麵包屑（目前在根層）。
2. 在搜尋列輸入：`食農教育`
3. 按 Enter 或點搜尋。
4. 結果清單出現後，點任一標案的標題或「查看詳情」連結。
5. 頁面切換到「案件詳情」，頂部麵包屑更新（如：首頁 > 食農教育 > 某案名）。
6. 詳情頁中，找到「投標廠商」清單，點任一廠商名稱。
7. 頁面切換到「廠商頁」，顯示該廠商的所有投標紀錄。
8. 廠商頁中，點任一標案連結，再鑽進另一個案件詳情。
9. 點麵包屑中的第一層（搜尋結果）返回，確認畫面恢復到步驟 2 的結果清單。

### 預期結果

- 每次點擊鑽入，麵包屑增加一層。
- 每個層次的內容正確（搜尋結果 → 案件詳情 → 廠商投標紀錄）。
- 點麵包屑任一層，頁面跳回該層內容（不重新搜尋，直接展示已有資料）。
- 從廠商頁可繼續點進機關，看到該機關的標案清單（Agency View）。

### 驗收通過條件

- [ ] 搜尋「食農教育」後出現結果清單。
- [ ] 點標案標題進入案件詳情，麵包屑正確更新。
- [ ] 案件詳情中有廠商清單可繼續鑽入。
- [ ] 點廠商後進入廠商頁，顯示歷史投標紀錄。
- [ ] 點麵包屑返回上一層，畫面正確回到前一頁內容。
- [ ] 整個鑽取流程中不出現空白頁或錯誤訊息。
