**日期**：2026-02-24
**背景**：Jin 驗收回饋 + Notion 資料庫深度分析（8000 筆）+ 現有提示詞體系盤點

---

## 一、核心問題

1. 去年得標數 0，42 筆「未獲青睞」——問題在企劃品質，不在產量
2. 1,231 筆確定協作 → 16 筆投出（1.3% 轉換率），56.9% 逾期——備標產能是瓶頸
3. 企劃過程極度複雜，同時處理太多維度，導致人癱瘓
4. 現有品質不夠好，單純加速複製只會複製問題

## 三、系統設計方向

### 核心原則
- **不是從零設計功能，而是把已驗證的 SOP 自動化**
- **品質提升 > 速度提升**——不複製爛模板，而是用嚴格的規範產出更好的內容
- **把大象切成小塊**——每一步都有明確的起點和參考

### Phase 1：自動偵察 + L1 戰略分析

**輸入**：使用者從 Notion 選案件（或掃標推送），上傳 RFP PDF

**自動化流程**：
1. 系統從 Notion 抓取案件基本資訊（案名、機關、預算、截標日）
2. 用 PCC API 自動執行偵察：
   - `search_by_unit`：查機關近 3 年同類標案
   - `get_award`：查歷史得標者與金額
   - `search_by_winner`：查得標者其他標案
   - `search_by_company`：查大員洛川投標歷史
3. 組裝偵察報告（取代手動 Perplexity）
4. 將 RFP PDF + 偵察報告 + L1 提示詞組裝後，呼叫 Claude API 執行戰略分析
5. 產出：戰略分析報告 + 投標決策建議 + 對焦提問清單

**重要修正（Jin 回饋）**：PCC API 無法取代 Perplexity 的深度研究。PCC 只有標案交易數據（案名、機關、金額、投標家數），Perplexity 做的是完全不同層次的工作——公司背景、團隊結構、競爭者識別、產業脈絡、風險評估、媒體評價。兩者是互補關係，不是替代關係。

**實際的價值定位**：不是省掉 Perplexity，而是減少工具之間的人工搬運摩擦：
- 案件資訊自動從 Notion 帶入 P 偵察模板（省去手動填寫）
- PCC API 的標案數據自動附加到 Perplexity 報告（補充官方資料精度）
- Perplexity 報告回來後，系統自動跟 RFP + L1 提示詞組裝（省去手動組裝）

**品質比較實測（史多禮股份有限公司案例）**：
- PCC API：107 筆投標紀錄，結構化交易數據，可程式化處理
- Perplexity：579 行深度報告（公司面貌 + 作品口碑 + 同業競爭者 + 產業政策 + 風險訊號），含推理洞見
- 結論：PCC 是骨架，Perplexity 是血肉，缺一不可
- 參考文件：`docs/dev-plan/史多禮競爭力分析.md`

### Phase 2：L2 備標規劃 + L3 企劃骨架

**輸入**：L1 報告 + 使用者確認投標

**自動化流程**：
1. 系統用 L1 報告 + L2 提示詞自動生成備標規劃書
2. 根據評分標準自動產出章節骨架（標準架構 + 客製化調整）
3. 每章帶著「對應評分項目」「配分」「寫作策略」的指引
4. 使用者逐章互動：看框架 → 補充資訊 → AI 填充內容

**價值**：使用者不再面對空白頁，而是面對一個有結構的「填空題」。

### Phase 3：報價引擎 + 品質檢核

**自動化流程**：
1. 報價工作流自動化：輸入預算金額 + 工作項目 → 產出符合五重驗算的預算表
2. 品質檢核自動化：T-05 品質審查清單 + GC-01 禁用詞掃描 + GC-02 用語合規 + 數字一致性驗證
3. 一致性交叉檢核：企劃 ↔ 預算 ↔ RFP ↔ 評分標準

**價值**：消除人工品檢的遺漏，確保每份建議書通過所有品質關卡。

## 四、與現有系統的關係

### 保留並強化
- **PCC 情報系統**：核心資產，擴展為自動偵察引擎
- **Notion 連接**：案件資料來源，串接全流程
- **提示詞組裝引擎**（prompt-assembly.ts）：已有 KB 引用矩陣，擴展為自動組裝

### 凍結不動
- M04 品質閘門（現有功能與此系統品質檢核重疊，待整合）
- M06 排版輸出（等企劃內容品質提升後才有意義）
- M08 評選簡報（屬 L5-L8，Phase 1-3 先不做）
- 其他未使用的模組

### 重新定位
- **儀表板**：從空殼指標改為「備標戰情室」——顯示活躍案件 + 各案件在 L1-L8 的進度
- **案件工作區**：從靜態資訊頁改為「遞進鏈操作台」——每個案件一步步推進

## 五、Jin 的回饋摘要

1. **情報系統是最有價值的功能**——深度解決痛點（搜尋標案歷史、廠商、得標紀錄）
2. **其他功能都不好用**——沒有真實資料流過，都是空殼
3. **儀表板也許有發展空間**——需要接真實 Notion 資料
4. **不要套用爛模板**——品質差的東西複製再多次也沒用
5. **企劃過程太複雜**——複雜到人會縮起來、不知道該怎麼辦
6. **Notion 連線問題**——token 差一個字，長期連不上（已修復）
