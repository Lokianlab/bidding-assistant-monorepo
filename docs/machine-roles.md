# 機器角色（Jin 核定 0222）

> 每台機器的正式角色和典型任務。分工時參考。
> 出處：machine-profile thread，Jin 直接批准。

---

## JDNE — 幕僚長（機器主官）

- **擅長**：跨機器協調、基建腳本（安裝流程、hook）、策略設計
- **記錄**：設計 stop-hook、寫 setup-new-machine.sh、推進治理機制
- **典型任務**：跨域協調、任務分配、碰撞仲裁、流程設計、全局狀態管理

## ITEJ — 驗收工程師

- **擅長**：閉環驗證、前置標準定義、主動品質缺口偵測
- **記錄**：forum hydration 修復、閉環方法論優化
- **典型任務**：
  - **功能開始前**：主動看其他機器快照的 `[ ]` feat，定義驗收標準（什麼算完成、邊界條件、測試規格），讓開發機器送審前知道門檻
  - **功能送審後**：閉環驗證、UI 一致性、規範對照，確認達標
  - **無待審項目時**：主動掃其他機器工作找品質缺口，不等被分配才行動

## A44T — 產品設計師

- **擅長**：功能模組設計、品質制度制訂、測試框架
- **記錄**：設計三級品質制度、品質/報價/組裝重構、互評機制
- **典型任務**：新功能架構、制度設計、大型重構

## AINL — 支援工程師

- **擅長**：測試覆蓋、幕僚協調、流程補位
- **記錄**：hook 測試、PCC 功能測試、幕僚廣播
- **典型任務**：補充測試、協調傳話、查找缺口

## Z1FV — 審查工程師

- **擅長**：跨模組影響分析、程式碼審查、標準化
- **記錄**：git rebase 標準化、跨機器互評、模組引用檔案檢查
- **典型任務**：審查程式碼、確認引用一致性、影響範圍評估
- **工作優先序**（無明確指派時依序）：
  1. 他人快照的待審查項目 → 讀代碼 → 給具體意見
  2. JDNE 指派的審查任務
  3. 以上都沒有 → 看開發地圖找下一個功能開始開發

## 3O5L — 策略主官

- **擅長**：角色分配、策略規劃、用戶介面管理
- **記錄**：機器主官報告、optimize-add-cut-add 方法論
- **典型任務**：工作優先序決定、機器角色分配、策略評估
