## A. 議題列表

A1. 把三家 AI 的 CLI 都裝齊
  背景：用戶問怎麼跑 Gemini CLI，查完安裝方式和 Ultra 的額度後直接裝了。接著翻開發計畫，發現暫存檔當初寫 Gemini CLI 和 Codex CLI 都「不裝」
  經過：用戶說「全都裝」，兩個都裝上了（Gemini CLI v0.29.3、Codex CLI v0.104.0）。順便查了各級額度：免費每日 1,000 次，Pro 1,500 次，Ultra 2,000 次，每分鐘上限 Pro 和 Ultra 都是 120 次。暫存檔跟著改成「已安裝」
  現況：兩個 CLI 都裝好了，但還沒做首次登入。Gemini 要用 Ultra 帳號登入才有 2,000 次額度，Codex 要用 ChatGPT Pro 帳號登入

A2. 修正 NotebookLM MCP 被誤記為「不裝」
  背景：用戶問「nblm 不是裝了嗎？」，暫存檔寫的是「不裝」
  經過：打開 `.mcp.json` 一看，NotebookLM MCP 早就配好了（第 3-6 行，用 npx 自動拉），是暫存檔記錯了。改成「已配置」
  結論：檔案記錄跟實際狀況對不上的問題修掉了

A3. 讓 /更新 指令自動檢查工具有沒有裝齊
  背景：用戶指出「這不是在 /更新 裡就要檢查的嗎」——同步的時候應該順便確認工具齊不齊，不該靠人記
  經過：在 /更新 指令的 pull 和碰撞偵測之間插了一個新步驟：先跑 `gemini --version` 和 `codex --version`，缺的自動裝；再讀 `.mcp.json` 逐個檢查 MCP server，npx 的跳過，node 指向 dist 的確認檔案存在。後面的步驟重新編號
  結論：以後每次跑 /更新 都會自動檢查，不用人記得
