## 背景

目前 SaaS Phase 1 系統已上線（P1a-P1f 全覆蓋），但缺乏自動備份機制。隨著各模組進入生產階段，資料損失風險升高。

---

## 需求分析

### 備份對象

| 對象 | 優先級 | 頻率 | 目標RTO |
|-----|--------|------|--------|
| Supabase DB | 🔴 最高 | 每 6 小時 | <30分鐘 |
| 用戶上傳檔案（KB） | 🔴 最高 | 每天 1 次 | <1小時 |
| 知識庫配置 | 🟡 高 | 每週 1 次 | <4小時 |
| 系統設定檔 | 🟡 高 | 手動 | - |

### 備份方式

**方案 A：雲原生備份（推薦）**
- Supabase 內建備份 API（Point-in-time recovery）
- S3/GCS 存放上傳檔案備份
- 成本：月 $50-100
- 優點：自動化、高可靠性、符合企業級要求
- 缺點：需額外成本

**方案 B：應用層定時備份**
- 每 6 小時導出 DB snapshot（pg_dump）
- 定時上傳到 GitHub Release（免費）
- 成本：$0（利用 GitHub 儲存空間）
- 優點：無額外成本、易於恢復
- 缺點：手動恢復較複雜、RTO 較長

**方案 C：混合方案**
- Supabase 內建備份（日誌級恢復）+ GitHub Release 備份（週級全量）
- 成本：月 $30
- 優點：雙保險、平衡成本與安全
- 缺點：複雜度中等
