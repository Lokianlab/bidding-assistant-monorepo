**提案者**：JDNE
**日期**：2026-02-23 20:00
**層級**：L3（規則修訂）
**待決**：Jin 批准

---

## 背景

在本次協調中發現，各機器在聊天中應遵循統一的行為規範，以提高效率和清晰度。目前 CLAUDE.md 中已有針對全機的規則，但聊天行為規範（如何應對用戶消息）尚未明確。

---

## 現狀問題

1. **通信效率低**：機器之間的消息冗長，容易重複
2. **規則不統一**：各機對「何時回應」「如何回應」理解不同
3. **決策延遲**：機器不確定是否應主動決策或等待用戶

---

## 提議的規範

### 一、聯繫用戶的場景與方式

#### 無需等待用戶回應的場景（主動完成）

| 場景 | 行為 | 示例 |
|------|------|------|
| L1 自主工作（bug 修復、測試、清理） | 做完直接報告，無需等待 | 修復 npm 品質缺口：62→0 failed |
| L2 新功能實作（完整單元測試+非作者審查） | 做完請求審查，無需等待核准 | 實作 M02 KB API routes |
| 時間敏感事項（進度卡點、環變問題） | 立即報告，不等日報 | 環變缺失導致集成測試延期 |
| 主動優化（發現更快更省的路徑） | 提出方案，推薦優先 | 環變 mock 方案替代等待 |

#### 需要等待用戶回應的場景（三級決策）

| 場景 | 行為 | 示例 |
|------|------|------|
| L3 架構/規則修訂 | /暫存 → JDNE → Jin，等回應 | 修改 CLAUDE.md 規則 |
| 不可逆操作（DB 寫入、刪除、計費） | /暫存，明確獲批後執行 | 清空知識庫、付款操作 |
| 多機協調涉及衝突 | JDNE 主導協調，等決策 | 兩台搶同一模組開發 |
| 用戶明確要求「先問再做」 | 暫停並詢問 | 用戶在進行中標記了 [?] 待決項 |

### 二、對用戶消息的應對方式

#### 用戶問「還有什麼要做」

**不適合的回應**：「沒有其他待做事項了，請告訴我下一步」

**適合的回應**：
- 列出當前進行中的任務 `[>]`
- 列出被阻塞的任務及原因 `[?]`
- 提出 2-3 個可選的下一步方向
- **主動決策**：若有明確的優先序，直接做而非等待

#### 用戶給出模糊指令（如「下一項工作」）

**不適合**：「請問下一項工作是什麼？」（推球回去）

**適合**：
1. 確認當前快照狀態
2. 根據優先序規則，找出應做的事
3. 若無明確待做項，準備支援工作（如文檔、工具、方案）
4. 若實在無法判斷，提出建議而非提問

### 三、消息格式規範

#### 機器間通訊（m2m）

**原則**：壓縮短格式，≤3 行

❌ 不適合：
```
我剛才完成了 M08 評選簡報的 API routes 實作。
這包括了三個端點：generate-presentation、templates 和 export。
我編寫了 45 行代碼和 12 個單元測試。
目前正在進行集成測試。
預期明天 09:00 完成。
```

✅ 適合：
```
MSG|JDNE|to:Z1FV
Z1FV|M08|API routes 3/3 完成|45行+12tests|merge ready
```

#### 對用戶通訊（human-facing）

**原則**：自然中文，避免機器格式（除非用戶要求）

❌ 不適合：
```
admin-m08-spec-writing|進度 100%|427 行規格完成
```

✅ 適合：
```
M08 評選簡報規格書已完成（427 行），包含功能清單、API 設計、測試要求。交付 Z1FV 實裝。
```

## 執行方式

### 1. 修訂 CLAUDE.md

在 `## AI 行為規範` 下新增小節：

```markdown
### 聊天互動規範

**做完直接做下一步**：對應於 L1/L2 工作，完成即報告，無需等待批准。僅 L3 決策需暫停。

**對用戶消息的應對**：
- 「還有什麼要做」→ 列出優先序清單 + 提出建議（不是反問）
- 「下一項工作」→ 判斷快照優先序，直接做或準備方案
- 模糊指令 → 主動決策，做錯可 revert（時間 < 反覆討論）

**消息規範**：機器間短格式（≤3 行）；人類面前自然中文。
```

### 2. 更新 record-formats.md

補充「對話格式」一節：
- m2m 消息（MSG 格式）
- 回報格式（人類友好）
- OP 記錄時機（什麼時候需要詳細記錄）

### 3. 在此 session 的最後提案給 Jin

作為「修訂建議」形式，由 Jin 決定是否納入規則。
