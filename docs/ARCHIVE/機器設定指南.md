# 新機器安裝指南

> **專案不放在 OneDrive 裡**，放在 `C:\dev\cc程式`，用 GitHub 同步。

---

## 簡易版（給幫忙安裝的人看）

照著做就好。不懂的地方不用管，一路按下去就對了。

### 準備工作

安裝前跟 Jin 拿這些（用 LINE 或任何方式傳都行）：

| 名稱 | 長什麼樣子 | 一定要嗎？ |
|------|-----------|-----------|
| Notion Token | `secret_abc123...` 開頭的一長串文字 | 是 |
| Notion Database ID | 32 個字元的英文數字混合 | 是 |
| SmugMug Token | 四組文字（API Key、API Secret、Access Token、Token Secret） | 之後再補也行 |

### 安裝步驟

#### 第一步：拿到安裝檔

從 Jin 那裡拿到 `一鍵安裝.bat` 檔案，放到桌面。

（這個檔案在專案的 `scripts/` 資料夾裡。如果新機器還沒有專案，Jin 會用 LINE 或 USB 傳給你。）

#### 第二步：雙擊執行

1. 雙擊桌面上的 `一鍵安裝.bat`
2. 如果 Windows 問「要允許這個應用程式對您的裝置進行變更嗎？」→ 按「**是**」
3. 如果 Windows 問「Windows 已保護您的電腦」→ 按「**其他資訊**」→「**仍要執行**」
4. 看到黑色視窗，文字會自己跑

**可能會遇到的情況：**

- 它說「**請關掉視窗重新執行**」→ 關掉這個視窗，再雙擊 `.bat` 一次就好
- 它開了**第二個黑色視窗** → 切到那個新視窗繼續操作

#### 第三步：跟著新視窗的提示做

新的黑色視窗會一步步問你，照著做：

1. **按 Enter 開始** — 確認準備好了就按
2. **GitHub 登入** — 會自動開瀏覽器，用 Jin 給你的 GitHub 帳號登入
3. **Notion Token** — 把 Jin 給的 Token 貼上，按 Enter
4. **Notion Database ID** — 把 Jin 給的 ID 貼上，按 Enter
5. **SmugMug Token** — 有就貼，沒有直接按 Enter 跳過

> **超級重要：貼上的方法**
>
> 在這個黑色視窗裡，**Ctrl+V 是沒有用的**！
>
> 要貼東西，請用 **滑鼠右鍵** → 會直接貼上（或選「Paste」）。

#### 第四步：等它跑完

安裝和驗證大約需要 **5-10 分鐘**。中間會看到很多文字自己跑，這是正常的。

看到這個就是成功了：

```
  ╔══════════════════════════════════════╗
  ║         全部安裝成功！               ║
  ╚══════════════════════════════════════╝
```

如果看到黃色的警告文字也不用緊張，那些不影響使用，Jin 之後會處理。

#### 第五步：回報

把以下資訊傳給 Jin：
1. 最後畫面有沒有**紅色**的錯誤
2. 機器代號是什麼（畫面上會顯示 4 個字元，像 `A3B7`）

### 出問題了？

| 看到什麼 | 怎麼辦 |
|----------|--------|
| 「需要安裝 Git」然後失敗 | 到 https://git-scm.com/download/win 下載，一路按 Next 安裝。裝完重新雙擊 .bat |
| 「下載失敗」 | 確認 Wi-Fi 有連上，重新雙擊 .bat |
| 「GitHub 登入失敗」 | 確認瀏覽器有完成登入，重新雙擊 .bat |
| 畫面卡住不動超過 5 分鐘 | 關掉視窗，重新雙擊 .bat |
| 其他看不懂的錯誤 | **截圖**傳給 Jin |

**安心提示**：這個腳本可以重複執行，不會把東西搞壞。遇到問題就關掉重來。

## 完整版（技術人員參考）

### 手動安裝（逐步）

#### 第一步：安裝工具（已裝過的跳過）

```bash
# Node.js 22
winget install OpenJS.NodeJS.LTS

# Git
winget install Git.Git

# GitHub CLI
winget install GitHub.cli

# Claude Code CLI
npm install -g @anthropic-ai/claude-code

# Gemini CLI
npm install -g @google/gemini-cli

# Codex CLI
npm install -g @openai/codex

# ⚠ 全部裝完後，關閉並重開終端機
```

#### 第二步：登入 GitHub + 下載專案

```bash
# 登入 GitHub
gh auth login
# 選 GitHub.com → HTTPS → Yes → Login with a web browser

# 下載專案（不要放在 OneDrive 裡！）
mkdir -p /c/dev
git clone https://github.com/Lokianlab/bidding-assistant-monorepo.git /c/dev/cc程式
cd /c/dev/cc程式
```

#### 第三步：設定

```bash
# Git 使用者
git config user.name "Jin"
git config user.email "gasklath20312@gmail.com"

# 安裝主 Web App 依賴
cd bidding-assistant && npm install && cd ..

# 建置 MCP Server
cd pcc-api-mcp && npm install && npm run build && cd ..
cd smugmug-mcp && npm install && npm run build && cd ..

# 環境變數
cp bidding-assistant/.env.example bidding-assistant/.env.local
# ⚠ 手動編輯 .env.local，填入：
#   NOTION_TOKEN=你的key
#   NOTION_DATABASE_ID=你的資料庫ID
```

#### 第四步：設定 MCP Server

建立 `.mcp.json`（專案根目錄，已 gitignored）：

```json
{
  "mcpServers": {
    "notebooklm": {
      "command": "cmd",
      "args": ["/c", "npx", "-y", "notebooklm-mcp@latest"]
    },
    "pcc-api": {
      "command": "node",
      "args": ["C:\\dev\\cc程式\\pcc-api-mcp\\dist\\index.js"]
    },
    "smugmug": {
      "command": "node",
      "args": ["C:\\dev\\cc程式\\smugmug-mcp\\dist\\index.js"],
      "env": {
        "SMUGMUG_API_KEY": "你的 API Key",
        "SMUGMUG_API_SECRET": "你的 API Secret",
        "SMUGMUG_ACCESS_TOKEN": "你的 Access Token",
        "SMUGMUG_TOKEN_SECRET": "你的 Token Secret"
      }
    }
  }
}
```

#### 第五步：驗證

```bash
cd /c/dev/cc程式/bidding-assistant
npm test          # 預期：26 files, 560+ tests 全過
npm run build     # 預期：0 errors
```

#### 第六步：開 Claude Code

```bash
cd C:\dev\cc程式
claude
```

Claude Code 會自動讀取 CLAUDE.md，知道所有規範。首次啟動時 AI 會自動生成機器代號並建立快照。

不需要手動操作 — Claude Code 會自動在對話開始時 `git pull`，結束時 `git push`。

## 故障排除

### git pull 報衝突
```bash
git status                    # 看哪些檔案衝突
# 大部分情況讓 Claude Code 處理即可
```

### npm test 失敗
```bash
cd /c/dev/cc程式/bidding-assistant
rm -rf node_modules
npm install
npm test
```

### MCP Server 無法啟動
```bash
cd /c/dev/cc程式/pcc-api-mcp && npm install && npm run build
cd /c/dev/cc程式/smugmug-mcp && npm install && npm run build
```

## 機密清單

以下資訊不進 git，需要從現有機器取得：

| 檔案 | 內容 | 從哪來 |
|------|------|--------|
| `bidding-assistant/.env.local` | Notion Token + Database ID | 現有機器或 Notion 設定頁 |
| `.mcp.json` | MCP Server 設定 + SmugMug OAuth | 現有機器的 `.mcp.json` |
| `.machine-id` | 機器代號（4 字元） | 自動生成，不需複製 |
