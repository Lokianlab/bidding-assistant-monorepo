### 使用方式的演進路徑

```
目前（Phase 1 自用）：
  T1 prompt → 貼進 Claude Project → 手動對話 → 複製產出 → 手動貼入 .md 檔

近期可做（不需等系統）：
  T1 prompt → Claude Code Skill（/kb 指令）→ 對話式建庫 → 自動寫入本地 .md 檔

中期目標（需要儲存方案定案）：
  Web wizard 聊天介面 → 知識官 Agent → API 直接寫入 Notion/Supabase

遠期（Phase 2+）：
  多租戶 Web + Telegram Bot → 知識官 Agent → DB 寫入 + 版本控制
```

## 結論

1. **Prompt 已經夠用**：v2.0 涵蓋完整流程（A-E 五個 Phase），不需要重寫
2. **缺的是「跑起來」**：有 prompt 但沒有便利的執行環境，目前只能手動貼
3. **最小可行下一步**：做成 Claude Code Skill（`/kb` 指令），讓精靈在 CLI 裡直接跑，產出自動寫入本地檔案。不需要等儲存方案定案。
4. **應加回待辦追蹤**：作為 Layer 0 項目補列

## 待決事項

1. **要不要先做 Claude Code Skill 版本？**
   - 優點：零成本、1 天內可完成、立刻可用
   - 缺點：只能在 CLI 用，其他人（手機/非開發者）無法使用
   - 建議：先做，不影響後續改為 Web wizard

2. **知識庫儲存方案**（阻塞中期目標，不阻塞近期）
   - Notion：不加新服務、員工可直接編輯
   - Supabase：結構化查詢、pgvector 語意搜尋
   - 目前 Layer 3 討論中斷，需要繼續
