# 機器設定指南

> 讓每台機器加入開發的完整步驟。
>
> **重要：專案不放在 OneDrive 裡，放在 `C:\dev\cc程式`。用 GitHub 同步。**

---

## 每台機器都要做的事

不管是新機器還是既有機器，步驟都一樣：

### 第一步：安裝工具（已裝過的跳過）

```bash
# Node.js 22
winget install OpenJS.NodeJS.LTS

# Git
winget install Git.Git

# GitHub CLI
winget install GitHub.cli

# Claude Code CLI（需有 Anthropic 帳號）
npm install -g @anthropic-ai/claude-code

# ⚠ 全部裝完後，關閉並重開終端機
```

### 第二步：登入 GitHub + 下載專案

```bash
# 登入 GitHub
gh auth login
# 選 GitHub.com → HTTPS → Yes → Login with a web browser

# 下載專案到 C:\dev（不要放在 OneDrive 裡！）
mkdir C:\dev
gh repo clone Lokianlab/bidding-assistant-monorepo C:\dev\cc程式
cd C:\dev\cc程式
```

### 第三步：設定

```bash
# Git 使用者
git config user.name "Jin"
git config user.email "gasklath20312@gmail.com"

# 安裝依賴
cd bidding-assistant && npm install && cd ..

# 環境變數
cp bidding-assistant/.env.example bidding-assistant/.env.local
# ⚠ 手動編輯 .env.local，填入：
#   NOTION_TOKEN=你的key
#   NOTION_DATABASE_ID=你的資料庫ID
```

### 第四步：驗證

```bash
cd C:\dev\cc程式\bidding-assistant
npm test          # 預期：26 files, 560 tests 全過
npm run build     # 預期：0 errors
```

### 第五步：開 Claude Code

```bash
cd C:\dev\cc程式
claude
```

Claude Code 會自動讀取 CLAUDE.md，知道所有規範。

---

## 每次開工

不需要你手動操作 — Claude Code 會自動在對話開始時 `git pull`，結束時 `git push`。

如果你想手動操作：

```bash
cd C:\dev\cc程式

# 拉最新
git pull origin main

# ... 工作 ...

# 推上去
git add 修改的檔案
git commit -m "描述做了什麼"
git push origin main
```

---

## 故障排除

### git pull 報衝突
```bash
# 看哪些檔案衝突
git status

# 如果不確定怎麼處理，直接用遠端版本
git fetch origin
git reset --hard origin/main
# ⚠ 這會丟掉本地未推送的修改
```

### npm test 失敗
```bash
cd C:\dev\cc程式\bidding-assistant
rm -rf node_modules
npm install
npm test
```

### 想看其他機器做了什麼
```bash
git log --oneline -10
# 或看操作日誌
cat docs/operation-log.md
```
