# 機器設定指南

> 讓每台機器加入開發的完整步驟。
>
> **重要：專案不放在 OneDrive 裡，放在 `C:\dev\cc程式`。用 GitHub 同步。**

---

## 自動安裝（推薦）

如果已經有 Git Bash，一行搞定：

```bash
# 先手動下載腳本（或從已有的機器複製）
# 然後在 Git Bash 中執行：
bash /c/dev/cc程式/scripts/setup-new-machine.sh
```

腳本會自動：安裝工具 → 登入 GitHub → 下載專案 → Git 設定 → 安裝依賴 → 建置 MCP → 驗證測試。

完成後只剩兩件手動的事：
1. 編輯 `bidding-assistant/.env.local`，填入 Notion API Key
2. 編輯 `.mcp.json`，填入 SmugMug OAuth Token

---

## 手動安裝（逐步）

### 第一步：安裝工具（已裝過的跳過）

```bash
# Node.js 22
winget install OpenJS.NodeJS.LTS

# Git
winget install Git.Git

# GitHub CLI
winget install GitHub.cli

# Claude Code CLI（需有 Anthropic 帳號）
npm install -g @anthropic-ai/claude-code

# Gemini CLI（需有 Google 帳號）
npm install -g @google/gemini-cli

# Codex CLI（需有 OpenAI 帳號）
npm install -g @openai/codex

# ⚠ 全部裝完後，關閉並重開終端機
```

### 第二步：登入 GitHub + 下載專案

```bash
# 登入 GitHub
gh auth login
# 選 GitHub.com → HTTPS → Yes → Login with a web browser

# 下載專案到 C:\dev（不要放在 OneDrive 裡！）
mkdir -p /c/dev
git clone https://github.com/Lokianlab/bidding-assistant-monorepo.git /c/dev/cc程式
cd /c/dev/cc程式
```

### 第三步：設定

```bash
# Git 使用者
git config user.name "Jin"
git config user.email "gasklath20312@gmail.com"

# 安裝主 Web App 依賴
cd bidding-assistant && npm install && cd ..

# 建置 MCP Server
cd pcc-api-mcp && npm install && npm run build && cd ..
cd smugmug-mcp && npm install && npm run build && cd ..

# 環境變數
cp bidding-assistant/.env.example bidding-assistant/.env.local
# ⚠ 手動編輯 .env.local，填入：
#   NOTION_TOKEN=你的key
#   NOTION_DATABASE_ID=你的資料庫ID
```

### 第四步：設定 MCP Server

建立 `.mcp.json`（專案根目錄，已 gitignored）：

```json
{
  "mcpServers": {
    "notebooklm": {
      "command": "cmd",
      "args": ["/c", "npx", "-y", "notebooklm-mcp@latest"]
    },
    "pcc-api": {
      "command": "node",
      "args": ["C:\\dev\\cc程式\\pcc-api-mcp\\dist\\index.js"]
    },
    "smugmug": {
      "command": "node",
      "args": ["C:\\dev\\cc程式\\smugmug-mcp\\dist\\index.js"],
      "env": {
        "SMUGMUG_API_KEY": "你的 API Key",
        "SMUGMUG_API_SECRET": "你的 API Secret",
        "SMUGMUG_ACCESS_TOKEN": "你的 Access Token",
        "SMUGMUG_TOKEN_SECRET": "你的 Token Secret"
      }
    }
  }
}
```

SmugMug OAuth Token 可以從現有機器的 `.mcp.json` 複製。

### 第五步：驗證

```bash
cd /c/dev/cc程式/bidding-assistant
npm test          # 預期：31 files, 619 tests 全過
npm run build     # 預期：0 errors
```

### 第六步：開 Claude Code

```bash
cd C:\dev\cc程式
claude
```

Claude Code 會自動讀取 CLAUDE.md，知道所有規範。首次啟動時 AI 會自動生成機器代號（4 字元隨機碼）並建立快照。

## 每次開工

不需要你手動操作 — Claude Code 會自動在對話開始時 `git pull`，結束時 `git push`。

如果你想手動操作：

```bash
cd C:\dev\cc程式

# 拉最新
git pull origin main

# ... 工作 ...

# 推上去
git add 修改的檔案
git commit -m "描述做了什麼"
git push origin main
```

## 故障排除

### git pull 報衝突
```bash
# 看哪些檔案衝突
git status

# 如果不確定怎麼處理，直接用遠端版本
git fetch origin
git reset --hard origin/main
# ⚠ 這會丟掉本地未推送的修改
```

### npm test 失敗
```bash
cd /c/dev/cc程式/bidding-assistant
rm -rf node_modules
npm install
npm test
```

### MCP Server 無法啟動
```bash
# 重新建置
cd /c/dev/cc程式/pcc-api-mcp && npm install && npm run build
cd /c/dev/cc程式/smugmug-mcp && npm install && npm run build
```

### 想看其他機器做了什麼
```bash
git log --oneline -10
# 或看記錄層的主題索引
cat docs/records/_index.md
# 或看各機器的快照
cat docs/records/_snapshot-*.md
```

## 機密清單

以下資訊不進 git，需要從現有機器手動複製：

| 檔案 | 內容 | 從哪來 |
|------|------|--------|
| `bidding-assistant/.env.local` | Notion API Key、Google Drive OAuth | 現有機器或密碼管理器 |
| `.mcp.json` | SmugMug OAuth Token | 現有機器的 `.mcp.json` |
| `.machine-id` | 機器代號 | 自動生成，不需複製 |
