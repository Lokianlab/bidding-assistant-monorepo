# 機器設定指南

> 讓新機器或既有機器加入開發的完整步驟。

---

## 情境一：既有機器加入開發

適用於：機器上已透過 OneDrive 有 `cc程式/` 資料夾。

```bash
# 1. 進入專案目錄
cd ~/OneDrive/桌面/cc程式

# 2. 安裝 GitHub CLI（如尚未安裝）
winget install GitHub.cli
# ⚠ 安裝完必須關閉並重開終端機

# 3. 登入 GitHub
gh auth login
# 選 GitHub.com → HTTPS → Yes → Login with a web browser

# 4. 確認 OneDrive 同步完成（系統匣顯示 ✓）
#    然後檢查 git 狀態
git status

# → 如果正常顯示 "On branch main" → 跳到步驟 6
# → 如果報錯（unable to map index file 等）→ 執行步驟 5

# 5. 修復損壞的 git（只在報錯時需要）
git fetch origin
git reset --hard origin/main

# 6. 設定 Git 使用者
git config user.name "Jin"
git config user.email "gasklath20312@gmail.com"

# 7. 安裝 bidding-assistant 依賴
cd bidding-assistant && npm install && cd ..

# 8. 設定環境變數
cp bidding-assistant/.env.example bidding-assistant/.env.local
# ⚠ 手動編輯 .env.local，填入 NOTION_API_KEY 等

# 9. 驗證
cd bidding-assistant
npm test          # 預期：26 files, 560 tests 全過
npm run build     # 預期：0 errors
```

## 情境二：全新機器完整設定

適用於：全新電腦，什麼都沒裝。

### 第一步：安裝基礎工具

```bash
# Node.js 22
winget install OpenJS.NodeJS.LTS

# Git
winget install Git.Git

# GitHub CLI
winget install GitHub.cli

# Claude Code CLI（需有 Anthropic 帳號）
npm install -g @anthropic-ai/claude-code

# ⚠ 全部裝完後，關閉並重開終端機
```

### 第二步：Clone 專案

```bash
# 登入 GitHub
gh auth login

# Clone 到桌面（或你想要的位置）
cd ~/OneDrive/桌面
gh repo clone Lokianlab/bidding-assistant-monorepo cc程式
cd cc程式
```

### 第三步：環境設定

```bash
# Git 使用者
git config user.name "Jin"
git config user.email "gasklath20312@gmail.com"

# 安裝依賴
cd bidding-assistant && npm install && cd ..

# 環境變數
cp bidding-assistant/.env.example bidding-assistant/.env.local
# ⚠ 手動編輯 .env.local，填入：
#   NOTION_API_KEY=你的key
#   NOTION_DATABASE_ID=你的資料庫ID
```

### 第四步：驗證

```bash
cd bidding-assistant
npm test          # 預期：26 files, 560 tests 全過
npm run build     # 預期：0 errors
npm run dev       # 啟動 dev server (port 3003)
```

## 每日開工流程

```bash
cd ~/OneDrive/桌面/cc程式

# ⚠ 確認 OneDrive 同步完成（系統匣圖示顯示 ✓）

# 拉最新
git pull origin main

# 建自己的工作分支
git checkout -b 機器名/任務名
# 例：git checkout -b laptop/fix-dashboard

# ... 開始工作 ...

# 完成後
git add 修改的檔案
git commit -m "描述你做了什麼"
git push -u origin 機器名/任務名

# 合併到 main（確認測試通過後）
git checkout main
git pull origin main
git merge 機器名/任務名
git push origin main
```

## 故障排除

### git 報錯 "unable to map index file"
OneDrive 同步衝突。修復：
```bash
git fetch origin
git reset --hard origin/main
```

### npm test 失敗
可能是依賴沒裝好：
```bash
cd bidding-assistant
rm -rf node_modules
npm install
npm test
```

### 忘記建分支直接在 main 上改了
```bash
# 把修改搬到新分支
git stash
git checkout -b 機器名/任務名
git stash pop
```
