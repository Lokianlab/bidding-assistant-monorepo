### Stage 2b：智能工作流引擎（2026-04-03 ~ 2026-05-01）

**目標**：建立 Claude Code SDK 驅動的自動化工作流

**核心概念**：

```
使用者作業 → 自動觸發 Claude Agent → 執行任務 → 回報結果

範例：
1. 使用者上傳 RFQ → Agent 自動解析、提取需求
2. Agent 調用 Intelligence 模組→ 搜尋相似標案
3. Agent 調用 Strategy 模組 → 分析競爭對手
4. Agent 調用 Quality 模組 → 檢查提案品質
5. 自動匯總成仪表板卡片 → 使用者審閱
```

**工作流清單**：

| 工作流 ID | 名稱 | 觸發事件 | 執行步驟 | 優先序 |
|----------|------|---------|---------|--------|
| **WF-01** | 標案快速分析 | 上傳 RFQ | 解析 → 搜尋 → 分析競爭對手 | P0 |
| **WF-02** | 提案品質檢查 | 上傳提案草稿 | 檢查 → 評分 → 修改建議 | P0 |
| **WF-03** | 知識庫同步 | 定時（每日 2:00） | Notion → KB → 索引 | P1 |
| **WF-04** | 績效分析 | 月末自動觸發 | 統計 → 分析 → 報告 | P1 |
| **WF-05** | Prompt 自動優化 | 使用者 feedback | 收集 → 優化 → A/B 測試 | P2 |

**技術實裝**：
- Claude Code SDK 整合
- Webhook + 事件驅動架構
- 背景任務隊列（Supabase pg_cron）
- 實時進度通知（WebSocket / Server-Sent Events）

**交付物**：
- 5+ 核心工作流可在 Web App 中配置
- Claude Agent 能正確調用模組 API
- 所有工作流有 audit log
- 用戶可監控工作流執行進度

## 三、技術決策

### 3.1 Claude Code SDK 整合策略

**方案**：Claude Code SDK 作為智能工作流引擎

```
┌─────────────────────────────────────┐
│   SaaS Web App（Next.js）           │
│   ├─ User Interface                 │
│   ├─ API Routes                     │
│   └─ Webhook Handler                │
└────────────┬────────────────────────┘
             │
             ↓
┌─────────────────────────────────────┐
│   Claude Code SDK Agent             │
│   ├─ 工作流編排                     │
│   ├─ 模組調用                       │
│   └─ 錯誤恢復                       │
└────────────┬────────────────────────┘
             │
             ↓ 調用
┌─────────────────────────────────────┐
│   已有功能模組（16 個）             │
│   ├─ Knowledge Base API             │
│   ├─ Intelligence API               │
│   ├─ Strategy Engine                │
│   └─ Quality Gate                   │
└─────────────────────────────────────┘
```

**實裝位置**：
- `src/lib/workflow/` — 工作流定義
- `src/app/api/workflows/` — 工作流路由
- `src/lib/claude-sdk/` — SDK 封裝

### 3.2 資料架構升級

**Phase 1 到 Phase 2 的變化**：

| 層 | Phase 1 | Phase 2 | 目的 |
|---|---------|---------|------|
| 應用層 | 基礎 CRUD | 複雜工作流 | 自動化決策 |
| API 層 | 6 個 KB 端點 | 50+ 模組端點 | 模組互聯 |
| 資料層 | 4 張表 | 15+ 張表 | 支持複雜業務 |
| 快取層 | 無 | Redis | 提升性能 |
| 隊列層 | 無 | Bull/Agenda | 背景任務 |

**新增表結構**：

```sql
-- 工作流定義
CREATE TABLE workflows (
  id UUID PRIMARY KEY,
  tenant_id UUID NOT NULL,
  name TEXT NOT NULL,
  trigger_event TEXT,
  steps JSONB,
  enabled BOOLEAN DEFAULT true,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

-- 工作流執行日誌
CREATE TABLE workflow_executions (
  id UUID PRIMARY KEY,
  workflow_id UUID NOT NULL,
  status TEXT ('pending', 'running', 'success', 'failed'),
  input JSONB,
  output JSONB,
  error_message TEXT,
  started_at TIMESTAMP,
  completed_at TIMESTAMP
);

-- 團隊與權限
CREATE TABLE teams (
  id UUID PRIMARY KEY,
  tenant_id UUID NOT NULL,
  name TEXT NOT NULL,
  members JSONB
);

-- 審計日誌
CREATE TABLE audit_logs (
  id UUID PRIMARY KEY,
  tenant_id UUID NOT NULL,
  user_id UUID,
  action TEXT,
  resource_type TEXT,
  resource_id UUID,
  changes JSONB,
  timestamp TIMESTAMP
);
```

| 風險 | 概率 | 影響 | 應對 |
