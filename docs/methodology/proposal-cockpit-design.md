# 提案寫作駕駛艙（Proposal Writing Cockpit）系統設計

基於成功案例黃金模板 + 失敗案例診斷清單逆向推導的系統設計方案。

---

## 核心問題

用戶編寫標案提案時面臨：
1. **無結構化指引** → 不知道該寫什麼順序、寫多詳細
2. **無實績對標** → 看不到成功案例怎麼做，無法參考
3. **無即時驗證** → 寫完沒人檢查，直到投標失敗才知道缺陷
4. **無版本管理** → 多份提案混亂，難以追蹤改進

---

## 系統願景

**「結構化寫作嚮導 + 實時品質檢查 + 成功案例參考庫」**

用戶輸入：「案件類型」(e.g. 文化展覽) + 「基本資訊」(金主、預算、期程)
系統輸出：
- 逐欄結構化大綱（對應黃金模板 8 層）
- 每欄都有「成功案例怎麼寫」的範本
- 邊寫邊檢查：是否滿足 8 層各層的品質準則
- 一鍵匯出：Word 或 PDF 可投標的提案文件

---

## 系統架構

### 第 1 層：案件初始化

**用戶輸入**：
```
案件名稱：[xxx]
案件類型：□ 文化展覽  □ 青年課程  □ 農業推廣  □ 其他
金主：[機構名稱]
預算：[NT$ xxxx]
期程：[開始日期] ~ [結束日期]
目標受眾：[xxx]
```

**系統回應**：
- 根據「案件類型」推薦類似成功案例作為參考
- 生成「8 層結構化大綱」骨架

### 第 2 層：引導寫作（逐層遞進）

**層次 1：封面資訊**
```
[嚮導]
- 案號：[xxx]（來自金主公告）
- 投標廠商：[貴公司名稱]
- 主持人：[姓名] [電話] [Email]
- 日期：[yyyy/mm/dd]

[品質檢查]
✓ 案號符合金主公告格式嗎？
✓ 主持人電話和 Email 完整嗎？
```

**層次 2：目次**
```
[嚮導]
系統自動從其他層次的填寫情況生成目次結構

[品質檢查]
✓ 各章節名稱對應實際內容嗎？
```

**層次 3：計畫摘要**
```
[嚮導] + [成功案例範本]
「故宮校園大使」提案的摘要部分怎麼寫：
---
◆ 創新目標：
- 培育 30 位校園大使，覆蓋北台灣 20 所高中
- 累計達成 150 場校園講座

◆ 執行期程：
2024/1/15 ~ 2024/12/31（12 個月）
---

[品質檢查]
✓ 目標是否量化？（「推廣文化」vs「1000 人參與」）
✓ 期程是否精確到月日？
✓ 預算說明是否清晰？
```

**層次 4：核心活動計畫**（最複雜，需分項逐填）
```
[嚮導] 按活動類型引導
活動 1：[活動名稱]
  □ 課程 / 講座 / 工作坊 / 展覽 / 其他
  - 場次：[x] 場
  - 參與人數：[x] 人
  - 日期地點：[日期] @ [地點]
  - 師資：[姓名] ([背景])
  - 預期產物：[交付物] (e.g. 證書、手冊、影片)

[成功案例範本]
「黑蝙蝠中隊」文化講座怎麼寫：
---
◆ 講座主題：「黑蝙蝠中隊秘史」
◆ 場次：8 場（新竹、台中、高雄各市 2 場）
◆ 講者：戰爭史學家 X、退役飛行員 Y
◆ 預期參與人數：每場 80-100 人
◆ 預期產物：
  - 演講簽到冊（實體紀念冊）
  - 成果影片（編輯製作，用於後續推廣）
  - 參與者反饋問卷
---

[品質檢查]
✓ 活動數量是否合理？（vs 時程）
✓ 每個活動都有具體數字嗎？
✓ 師資背景是否能體現專業性？
✓ 交付物是否具體而非虛幻？
```

**層次 5：宣傳策略**
```
[嚮導] 分波段模版
預熱階段（活動前 6 週）：
  - 主視覺設計：[品牌色、LOGO 應用]
  - 社群文案：[3-5 個帖子主題]
  - 預期觸及：[目標粉絲/互動數]

開跑階段（活動期間）：
  - 實況報導：[直播/即時貼文計畫]
  - 互動玩法：[抽獎/投票等]

尾聲階段（活動後 2 週）：
  - 成果整理：[彙整照片、影片、反饋]
  - 感謝文案：[感謝金主、參與者]

[成功案例範本]
「青年情緒」課程的宣傳怎麼做：
---
◆ 主視覺：藍色系，youth 輪廓圖案
◆ 預熱（11 月）：
  - 知識帖：「情緒管理的 5 個小秘訣」
  - 名人見證：邀請知名心理諮商師分享故事
  - 預期：5000+ 粉絲接觸
◆ 進行中（12 月-1 月）：
  - 每週一學員分享（文字+照片）
  - 互動投票：「最有感觸的課程是？」
◆ 尾聲（2 月）：
  - 成果報告：「共有 450 人參與，92% 滿意度」
  - 結業式回顧影片（2 分鐘精華）
---

[品質檢查]
✓ 宣傳有分波段策略嗎？（vs 只是「在 FB 發文」）
✓ 有具體的投放數據嗎？（粉絲數、互動率）
✓ 主視覺或品牌識別清晰嗎？
```

**層次 6：時程表**
```
[嚮導] 自動從活動列表生成甘特圖視圖

[成功案例範本]
月份 | 工作項目 | 里程碑
-----|---------|-------
1月  | 企劃審查 |
2月  | 師資簽約、場地預訂 |
3月  | 宣傳素材製作 |
4月  | 正式推廣開始 | 第一波：預熱宣傳
5月  | 第 1、2 場講座 | 期中報告提交點
...

[品質檢查]
✓ 每個活動都有對應的月份嗎？
✓ 有標註「期中報告」、「請款點」等關鍵里程碑嗎？
✓ 時程是否現實？（活動數 vs 月份數）
```

**層次 7：團隊組織**
```
[嚮導] 按角色逐填
◆ 公司基本資料
  - 公司名稱：[xxx]
  - 統編：[xxx]
  - 成立年度：[xxxx]
  - 主要業務：[xxx]

◆ 主要成員（至少 3 人）
  - 姓名：[xxx]
  - 職位：[xxx]
  - 學歷：[xxx]
  - 相關經歷：[xxxx 年做過 yyy 項目，xxx 人參與，成果是...]

◆ 履約實績（最重要！）
  - 案件名稱：[xxx]
  - 執行期間：[yyyy/mm ~ yyyy/mm]
  - 經費規模：[NT$ xxxx]
  - 參與人數：[xxxx 人]
  - 成果數字：[完成率 xx%，滿意度 xx/5]

[成功案例範本]
故宮校園大使團隊怎麼寫：
---
主持人：王美芳（教育推廣主任）
- 學歷：文化大學藝術史所碩士
- 經歷：
  ✓ 2021-2023 擔任故宮教育推廣組組長
  ✓ 2020 主持「文化校園大使計畫」（30 人，8 校，完成率 98%）
  ✓ 2019 執行教育部青年課程補助（180 人，12 場，滿意度 4.8/5）

執行團隊：
- 講師 3 位（各自有博物館教學經驗 3-5 年）
- 行政 1 位（負責簽到、出席率統計、後勤）

組織圖：
[主持人 王美芳] → [講師群] → [校園大使]
                  → [行政人員]
---

[品質檢查]
❌❌ 最常見失敗點：沒有履約實績！
✓ 是否列舉過去類似案例？
✓ 履約實績是否包括「參與人數」「成果數據」？
✓ 團隊有責任分工嗎？
```

「故宮校園大使」預算怎麼編：
---
科目     | 數量 | 單價   | 小計     | 根據
---------|------|--------|----------|--------

### 第 3 層：品質自檢面板

邊寫邊檢查，系統在右側顯示進度：

```
8 層架構完成度
━━━━━━━━━━━
✓ 層 1：封面資訊         100% 完成
✓ 層 2：目次             100% 完成
✓ 層 3：計畫摘要         100% 完成
⚠️ 層 4：核心活動        75% 完成（5/8 個活動）
❌ 層 5：宣傳策略         0% 完成
⭕ 層 6：時程表          自動生成中
❓ 層 7：團隊組織         缺「履約實績」
❓ 層 8：預算表          單位成本未驗證

品質診斷
━━━━━━━━━━━
⚠️ 活動數量偏多（8 個），僅 5 個月期程，可能執行不力
   → 建議：減少到 5-6 個核心活動，或延長期程

⚠️ 宣傳策略尚未填寫
   → 根據類似案例「青年情緒」，應包含預熱/進行中/尾聲分波段

❌ 缺履約實績
   → 最常見失敗原因！建議補上 2-3 個類似過去案例
```

## 成功指標

1. **寫作效率**：5 小時 → 2 小時（提案初稿）
2. **品質一致性**：新用戶也能寫出符合 8 層標準的提案
3. **成功率提升**：對標成功案例，減少常見失敗模式
4. **用戶滿意度**：提案被金主評為「結構清晰、資料完整」

## 實現路徑

### MVP（最小可行版）
- [x] 8 層結構骨架生成
- [ ] 層 3-5 的結構化表單 + 範本
- [ ] 品質檢查清單（針對 8 層各層常見缺陷）
- [ ] 匯出為 Word / PDF

### Phase 2
- [ ] AI 輔助：「根據你的活動，自動建議時程和預算」
- [ ] 成功案例庫整合：「看看類似案例怎麼寫這層」
- [ ] 版本比較：前後版本差異高亮

### Phase 3
- [ ] 多人協作：團隊共編提案
- [ ] 金主評論模式：金主直接在提案裡加評論
- [ ] 自動化報告：投標後自動追蹤中選/未中原因

## 與現有系統的整合

```
現有「全能標案助理」：
├─ 儀表板（Dashboard）: 案件列表、進度追蹤
├─ 策略分析（Strategy）: 五維適配度評分
├─ 品質閘門（Quality Gate）: M04 檢查點
└─ 輸出模組（Output）: 排版匯出 (M06)

新增「提案寫作駕駛艙」（Proposal Cockpit）：
   用戶點選「新建提案」→ 進入駕駛艙
   ↓
   逐層填寫 + 實時品質檢查
   ↓
   系統預生成文檔骨架 → 發送給輸出模組（M06）排版
   ↓
   Final：可投標的 Word / PDF
```

## 必要的前置資料

1. **成功案例庫**（已開始收集）
   - 2-3 個各類型的完整提案文本
   - 含封面、目次、各層內容、預算表

2. **失敗案例對標**（進行中）
   - 同類型的失敗案例
   - 對比分析：成功 vs 失敗的具體差異

3. **金主公告範本**
   - 常見金主的評選標準
   - 各類型標案的預算通常範圍

4. **團隊實績庫**（內部）
   - 公司過往案例的結構化記錄
   - 便於新提案引用時快速查詢
