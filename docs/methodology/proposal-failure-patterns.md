# 標案失敗模式診斷清單

基於成功案例的 8 層「黃金模板」逆向推導。

---

## 診斷框架

### 第 1 層：封面資訊
**失敗跡象**：
- [ ] 案號或機構資訊錯誤/不完整
- [ ] 聯絡人資訊模糊（無電話、無郵件）
- [ ] 投標廠商名稱與內容團隊對應不清

### 第 2 層：目次
**失敗跡象**：
- [ ] 沒有目次或目次亂序
- [ ] 目次章節名與實際內容不符
- [ ] 結構邏輯不清（難以一眼看出proposal的論述順序）

### 第 3 層：計畫摘要
**失敗跡象**：
- [ ] 目標模糊（說「推廣文化」，沒有具體數字）
- [ ] 經費寫了總額，但沒有分配說明
- [ ] 期程寬泛（「一年內完成」，沒有月份里程碑）
- [ ] 窗口資訊缺（責任人不明確）

### 第 4 層：核心活動計畫
**失敗跡象**：
- [ ] 活動清單缺參與人數 / 場次 / 日期
- [ ] 活動名稱模糊（「舉辦講座」，沒說幾場、多少人、什麼題目）
- [ ] 師資背景不具體（「邀請專家」，沒有名字、學經歷、過往案例）
- [ ] 預期成果只有質化描述，無量化指標（「提升民眾認知」v.s. 「參與 500 人，滿意度 4.5/5」）
- [ ] 沒有「預期具體產物」（沒說交付什麼東西給金主）

### 第 5 層：宣傳策略
**失敗跡象**：
- [ ] 沒有宣傳計畫（假設「做好就有人來」）
- [ ] 有宣傳但無分波段策略（一樣的推文推全期間）
- [ ] 缺乏主視覺或品牌識別元素
- [ ] 社群媒體計畫不具體（「在 FB 發文」，沒說頻率、內容類型、目標粉絲數）
- [ ] 沒有針對不同受眾的差異化信息（只有一版文案）

### 第 6 層：時程表
**失敗跡象**：
- [ ] 時程表缺月份或日期（只說「上半年」）
- [ ] 沒有標註關鍵里程碑（期中報告、招生截止、期末成果發表）
- [ ] 沒有標示請款點（金主何時可以領款）
- [ ] 時程與活動數量邏輯不符（50 場活動但只有 3 個月）

### 第 7 層：團隊組織
**失敗跡象**：
- [ ] 公司資料不完整（沒有統編、成立年度、主要業務）
- [ ] 主要成員只有名字，沒有學經歷
- [ ] **沒有「履約實績」**（過往類似案例）
  - 這是最常見的失敗原因
  - 金主需要知道：「你們做過這種案子嗎？成績如何？」
- [ ] 履約實績寫得太簡略（「做過文化推廣」，沒說年度、金額、參與人數、成果數字）
- [ ] 團隊沒有責任分工清單（誰負責教學、誰負責宣傳、誰負責財務）

### 第 8 層：預算表
**失敗跡象**：
- [ ] 預算只有一行總數，沒有分項（人事、教材、場地、宣傳等）
- [ ] 分項不清（「活動費 50 萬」，什麼活動、為什麼要 50 萬）
- [ ] 成本分解不合理（師資費太低或場地費離譜）
- [ ] 沒有「單位成本」明細（如按參與人數、按場次分攤）
- [ ] 沒有說明預算來源或其他贊助

---

## 常見失敗模式組合（優先掃）

**模式 A：「想法好但準備不足」**
- ✗ 第 4 層：活動概念有趣，但沒有數字支撐
- ✗ 第 7 層：沒有履約實績
- ✗ 第 5 層：宣傳計畫缺乏

**模式 B：「亂槍打鳥」**
- ✗ 第 4 層：列出 20+ 個小活動，沒有邏輯關聯
- ✗ 第 6 層：時程嚴重超載
- ✗ 第 8 層：預算分散，單項成本無法驗證

**模式 C：「只有承諾，沒有證明」**
- ✗ 第 7 層：全是新公司或無相關經驗
- ✗ 第 3 層：承諾的數字（參與人數、成果）沒有根據
- ✗ 第 5 層：宣傳計畫空洞（沒有實際粉絲數、沒有投放預算）

**模式 D：「結構不完整」**
- ✗ 第 1-3 層：基本資訊跳過或不清
- ✗ 第 2 層：沒有清晰的導航
- 金主讀一半就放棄

---

## 使用方式

對失敗案例逐層檢查：

1. **快速掃** → 逐層看一遍，勾出現有的缺陷
2. **深入分析** → 對勾出的項目展開，說明為什麼這會導致失敗
3. **對標** → 比照成功案例同層內容，指出具體差距
4. **根因** → 推測失敗的主要原因（如：團隊經驗不足、概念創意不足、執行能力不足）

針對每份失敗案例：

```
案件名稱：[xxx]
預算：[xxx] / 金主：[xxx]

主要缺陷：
1. [第 X 層]：[具體問題]（vs 成功案例的做法）
2. [第 X 層]：[具體問題]

根本失敗原因：
- [主因 A]
- [次因 B]

對提案寫作駕駛艙的啟示：
- [建議 1]
- [建議 2]
```
