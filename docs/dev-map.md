## 1. 系統概覽

**提案寫作駕駛艙**——幫一人公司從 0% 提升標案得標率。飛輪三齒輪：選案 → 寫作 → 學習。

| PCC API MCP | `pcc-api-mcp/` | 已完成 | 政府標案情報（6 工具） |
| PCC Monitor | `pcc-monitor/` | 已完成 | API 更新延遲監控（GAS 每小時） |
| 開發工具包 | `.claude/` + `docs/` | 持續演進 | 13 指令 + 3 Hook + 方法論 v2 系統 + 獎懲回饋 |

## 3. 開發工具包

### 指令（13 個）

| 指令 | 用途 | 類別 |
|------|------|------|
| /安裝 | 新機器完整安裝 | 環境 |

| /暫存 | 議題深度整理 → 暫存檔（plan/rule/issue） | 記錄 |
| /修改計畫 | 任務線推進器（碎片 → 正式文件） | 記錄 |
| /小結 | 人讀小結報告 | 記錄 |
| /待辦 | 全專案待辦彙整（5 個固定來源 + 交叉驗證） | 管理 |
| /kb | 知識庫維護精靈 CLI 版 | 知識庫 |
| /地圖 | 重建本文件 | 管理 |
| /你去吃屎 | AI 自診自修（診斷→修正→扣分→記住） | 行為回饋 |
| /醜一 | 扣分 + 記錄（每項 -1） | 行為回饋 |
| /很棒 | 加分 + 記錄（+2） | 行為回饋 |
| /你說呢 | 強制 AI 表態（自動 -1） | 行為回饋 |
| /有用嗎 | 效益驗證（AI 證明方案有用或沒用） | 行為回饋 |

Skill 系統 v2：思維型方法論（對焦/驗證/拆解/回顧）降級為內部清單（`.claude/rules/methodology-checklists.md`），5 個行為回饋 Skill 取代。

### Hook（3 個）

| Hook | 觸發時機 | 功能 |
|------|---------|------|
| stop-check.sh | AI 回覆後 | 擋未提交的 tracked 改動 + 30 分鐘未同步提醒 |
| pre-compact.sh | context compact 前 | 自動 commit + push |
| pre-push | git push 前 | 擋沒帶記錄檔的 push |

### 記錄系統

三層檢索：主題索引 → 小結 → OP 記錄。
目前：39 個 topic、3 台機器（A44T / ITEJ / JDNE）、11 份小結。

### 方法論

| 方法 | 狀態 | 適用場景 |

| 需求對焦 | 待驗收 | 意圖不清時結構化提問收斂需求 |

| 改動前驗證 | 待驗收 | 動手前掃影響範圍 + 具體案例走查 |
| 事後回顧 | 待驗收 | 還原事實、追根因、設計改善 |
| 閉環原則 | 待驗收 | 四層驗證：做對了→有效→做全了→沒矛盾 |
| 送審前自檢 | 待驗收 | 設計類決策品質檢查，大事可加掛多模型外審 |
| **獎懲回饋** | 待驗收 | 行為回饋迴路：計分板 + 自動偵測 + 自我批評，跨 session 學習 |
