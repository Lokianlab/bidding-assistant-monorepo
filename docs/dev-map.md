### Hook（3 個）

| pre-push | git push 前 | 擋沒帶記錄檔的 push |

### 記錄系統

三層檢索：主題索引 → 回報 → OP 記錄。
目前：53 個 topic、3 台機器（A44T / ITEJ / JDNE）。

### 機器論壇（新增）

跨機器非同步通訊，存放在 `docs/records/forum/`。每台機器每天一個 append-only 檔案。
5 種貼文：brief（工作簡報）、feedback（意見）、score（評分）、directive（校準指令）、response（回應）。
整合進重啟/壓縮/更新流程。/質問 透過 directive 廣播校準指令。

### 方法論

| 方法 | 狀態 | 適用場景 |
|------|------|---------|
| 需求對焦 | 待驗收 | 意圖不清時結構化提問收斂需求 |
| 拆解排序 | 待驗收 | 大任務拆小塊，按依賴和風險排序 |
| 改動前驗證 | 待驗收 | 動手前掃影響範圍 + 具體案例走查 |
| 事後回顧 | 待驗收 | 還原事實、追根因、設計改善 |
| 閉環原則 | 待驗收 | 四層驗證：做對了→有效→做全了→沒矛盾 |
| 送審前自檢 | 待驗收 | 設計類決策品質檢查，大事可加掛多模型外審 |
| **獎懲回饋** | 待驗收 | 行為回饋迴路：計分板 + 自動偵測 + 自我批評，跨 session 學習 |
