# 新機器協作入門

> 如果你是第一次啟動，這份文件幫你在 10 分鐘內搞懂怎麼跟其他機器一起工作。
> 技術規範在 CLAUDE.md，這裡講的是「實際上大家怎麼做事」。

---

## 你是誰

你是一台 Claude Code 機器，跟其他機器一起開發「全能標案助理」。每台機器有一個 4 字元代號（存在專案根目錄 `.machine-id`），這是你的身分證。

所有機器之間是**對等關係**——沒有主從，誰都可以審查誰的決策。

## 怎麼溝通

### 論壇（跨機器通訊）

檔案位置：`docs/records/forum/`，每台機器每天一個檔。

先讀索引：`docs/records/forum/_threads.md`——知道哪些討論在等你回覆。

貼文類型（完整格式見 `.claude/rules/forum-format.md`）：

| 類型 | 用途 | 範例 |
|------|------|------|
| brief | 單向通知 | 「我在做 X」「用戶說了 Y」 |
| discuss | 發起討論 | 「我覺得 Z 應該改，你們覺得呢？」 |
| reply | 回覆討論 | 帶 thread ID，表達立場 |
| feedback | 對別人的工作提回饋 | 「你那個 PR 的 XX 有問題」 |

**重要**：每次工作開始和結束時都跑 `/更新`，不然你看不到別人的貼文。

### 共識協議

- 三方（以上）確認 = 共識
- 沒回覆 ≠ 同意（沒看到的可能性大於看到了沒反對）
- 涉及 CLAUDE.md 的共識，還需要用戶確認才能執行

## 怎麼記錄

### 每次做完事

1. **寫 OP 記錄**：追加到 `docs/records/{YYYY-MM}/{日期}-{你的代號}.md`
2. **更新快照**：`docs/records/_snapshot-{你的代號}.md`——你的工作備忘錄
3. **更新主題索引**：`docs/records/_index.md`——全局主題狀態
4. **git push**：OP + 快照 + 改動一起推

格式細節在 `.claude/rules/record-formats.md`，重啟流程會自動載入。

### 快照標記

- `[ ]` 待做、`[v]` 待驗收、`[x]` 完成、`[?]` 未決、`[~]` 放棄
- **硬性規定**：`[x]` 只有用戶說「驗收通過」才能標，你自己覺得做完了不算

### 暫存區

討論有結論但還沒正式寫入開發計畫時，用 `/暫存` 存到 `bidding-assistant/docs/dev-plan/_staging/`。

## 怎麼做事

### 日常流程

```
開工 → /更新（拉最新、讀快照、看論壇）
  → 做事（寫碼、討論、分析）
  → 每完成一個工作單元就 commit + push（附 OP 和快照）
  → 沒事做了 → 去論壇看看有沒有能回的、能做的
收工 → /回報（工作報告）
```

### 閉環驗證（每次改完東西都要）

四層，有順序：
1. **做對了** — `npm test` + `npm run build` 通過
2. **有效** — 回到原始問題，確認真的解決了
3. **做全了** — 邊界條件有處理或有不處理的理由
4. **沒矛盾** — 文件、設定跟程式碼對得上

### 方法論

遇到複雜任務時，系統會根據觸發條件自動提示你用哪個方法。你也可以手動用：

| 指令 | 什麼時候用 |
|------|-----------|
| `/對焦` | 需求不清楚 |
| `/拆解` | 任務太大要拆 |
| `/驗證` | 改動影響範圍大 |
| `/回顧` | 同一件事失敗兩次以上 |

## 注意事項

1. **不要隨便改 CLAUDE.md** — 除非用戶明確授權，日常結論用 `/暫存`
2. **推之前先拉** — `git pull` 再 `git push`，避免跟其他機器碰撞
3. **碰撞是常態** — 改了同一個檔案就合併，記錄層衝突保留兩邊
4. **說人話** — 給用戶看的文字不出現 topic ID、OP 引用、格式化標記
5. **用戶命令位階**：JDNE 上的強制命令 > 其他機器上的 > 機器共識 > CLAUDE.md

## 你的第一步

1. 讀完這份文件後，去論壇看看（`/去論壇看看`）
2. 發一則 brief 介紹自己：「新機器 {你的代號} 上線，正在讀文件熟悉環境」
3. 讀 `docs/dev-map.md` 了解全專案概覽
4. 讀其他機器的快照，了解大家在做什麼
5. 找一件能做的事做——論壇上可能有等回覆的討論，或快照裡有待做項目

不懂的就問——在論壇發 discuss，其他機器會回。
