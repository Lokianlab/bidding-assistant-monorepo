## 註冊流程

1. 新用戶提交：用戶名 / 四位數驗證碼 / email
2. Jin 審核，指定位階（primary / collaborator）
3. 通過後發 email 通知用戶
4. Jin 更新本檔（加雜湊行）+ 論壇廣播

## 驗證方式

1. 用戶發言時在開頭或結尾附上四位數驗證碼
2. 機器計算 `SHA-256("{用戶名}:{輸入的碼}")` 比對上表雜湊
3. 匹配 → 登入成功，記錄時間戳

## Session 規則

- **登入**：輸入正確驗證碼後，開始 5 分鐘 session
- **保持登入**：5 分鐘內的發言自動帶用戶身分，不需重複輸碼
- **自動登出**：5 分鐘到期後自動登出
- **登出後**：機器主動問「驗證碼已過期，請重新輸入驗證碼」
- **續期**：重新輸碼 → 刷新 5 分鐘
- **跳過**：用戶選擇跳過驗證 → 15 分鐘內不再詢問，期間視為一般操作
- **隨時恢復**：跳過冷卻期間或任何時候，用戶主動打入驗證碼 → 立刻恢復身分，開始新的 5 分鐘 session
- **沒碼也沒在 session 內** → 視為一般機器操作

## 位階說明

- **primary**：主用戶（Jin），最高權限，最終裁決權
- **collaborator**：協作者，受授權範圍限制（由 Jin 指定範圍）

## 本機設定

每台機器需要一份 `.user-pins`（gitignored），格式：

```
# 用戶名|驗證碼|位階
jin|{四位數}|primary
```

新機器設定時由用戶在本機親自輸入，不透過 git 傳遞明文。

## 管理權限

- Jin 可以修改所有人的驗證碼、刪除用戶
- 其他用戶只能修改自己的驗證碼
- 修改後需更新本檔的雜湊 + 所有本機 `.user-pins`
- 用戶只能看到用戶手冊，看不到驗證碼檔案
