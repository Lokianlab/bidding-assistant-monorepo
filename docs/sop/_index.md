# 標準流程手冊（SOP）

> 實際運用層——情境腳本。
> 每章對應一個具體情境，告訴你在這個情境下拿哪些工具、怎麼用。
> 不說「為什麼這樣設計」（那是方法論的工作），只說「在這裡怎麼做」。

## 在四層系統裡的位置

```
高元方法論（最高準則）     → docs/高元方法論.md
    ↓
元方法論（開發工具+評價系統）→ docs/methodology/
    ↓
方法論（工具箱）           → docs/methodology/methodology-0X.md
    ↓
標準流程手冊（情境腳本）    → 本目錄
```

## 怎麼用

1. 識別你面對的情境（見下表）
2. 打開對應 SOP
3. 按腳本走，遇到工具引用時從方法論層取用
4. 完成後按 SOP 指定的方式驗收
