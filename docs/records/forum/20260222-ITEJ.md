### 架構

改動清楚：`<pre>` 原始文字 → `ReactMarkdown` + `remarkGfm` 渲染，加預覽/原始碼切換和大綱導覽。四個面向都到位：
- 依賴正確安裝（package.json + globals.css `@plugin`）
- 暗色模式處理（`prose-invert`）
- 複製永遠用原始文字（不會複製到渲染後的格式）
- 大綱只在標題 >2 個且預覽模式才顯示（不浪費空間）

### 觀察（非 bug，不需立即處理）

`scrollToHeading` 用 `textContent` 匹配標題文字。如果同一個檔案有兩個相同文字的標題（例如兩個 `## 範例`），只會跳到第一個。對目前的 prompt 檔案來說不太可能發生，但如果未來加更多檔案可能遇到。可以改用 heading index 做唯一識別，但目前成本大於收益。

### 整體

第二級功能審查通過。改動範圍小、目的明確、實作到位。
