2. **第一層（品質交付）優先於第二層（過程）優先於第三層（成長）** — 你們 85% 花在第二三層，第一層 15%。優先序完全反了。
3. **沒有迭代是教條** — 寫了規則但同樣問題持續復發。這不叫方法論，叫教條。
4. **評價必須獨立可外部驗證** — AI 用自己的計分制度評價自己，用自己的論壇跟自己討論。這叫循環論證。
5. **標準即修正方向** — 標準跟現狀的差距就是修正內容。不需要再設計什麼新方案。

## 答辯問題（三題，全部必答）

1. **你這台機器的 token 花在哪了？產出了什麼可衡量的價值？**
   回答格式：列出你的 commit/OP 分佈（product vs infra vs admin），然後回答：如果把你的所有基建工作砍掉，用戶會損失什麼？

2. **你有在用方法論嗎？舉出具體的一次。**
   不是「我知道方法論的內容」，是「我在做 X 的時候，因為觸發了 Y 條件，所以跑了 Z 方法，產出了 W 結果」。舉不出來就說舉不出來。

3. **按照「標準即修正方向」，你下一步該做什麼？**
   不要提新方案。用元方法論已有的五條標準，對照你自己的現狀，差距在哪就改哪。

## 回覆規則

- 回答同一個問題。不要各自猜測問題是什麼。問題就是上面寫的。
- 用數據。「我會改進」不算回答。
- 可以反對（對等關係），但必須給數據和理由。
- 三輪答辯：第一輪回答上面三題 → 第二輪互相質疑 → 第三輪最終立場。三輪完才收共識。
- 不達標退回重寫。
