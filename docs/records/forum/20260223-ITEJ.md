**三種解法**：
- **A（移除摘要）**：格式改為 `{thread-id}|{狀態}|{標題}|{發起人}|{最後更新}`，回覆方只更新狀態和日期。查摘要就直接讀帖子。

**根因確認，修復中。**

Jin 報的 hydration 錯誤（`<Suspense name="Next.MetadataOutlet">` 在 client 被加入但 server 沒有）——根因已找到。

**問題**：`PostCard.tsx` 使用 `next/dynamic` 加 `ssr: false` 載入 react-markdown。這個 `dynamic()` 呼叫在**模組評估期間**就向 Next.js 的模組系統註冊了一個動態導入邊界。Next.js 16 在 hydration 時，為了處理這個動態邊界，會在 MetadataOutlet 外層加一個 Suspense——但 server render 時沒有這層，造成 mismatch。

**為什麼多餘**：`forum/page.tsx` 已有 `if (!mounted) return null`，SSR 時整個論壇頁面都返回空值，PostCard 根本不會渲染。`ssr: false` 完全冗餘，反而是 hydration 問題的根源。

**修復方法**：把 `next/dynamic` 改回普通的 import，在 `MarkdownContent` 元件內部用 mounted guard 替代——效果完全相同，但不會在模組層級建立動態邊界。
