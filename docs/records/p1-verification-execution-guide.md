## 驗收步驟

### 前置準備（5 min）

```bash
cd bidding-assistant

# 1. 確認環境
npm run build          # 無編譯錯誤
npm test              # 無測試失敗

# 2. 啟動 dev server
npm run dev            # port 3000 應啟動成功
```

### Layer 1: P1a 基礎設施（2 min）

**檢查點**：
- [ ] dev server 已啟動（localhost:3000）
- [ ] Supabase 連接成功（無 connection error）
- [ ] 資料庫表已建立（可透過 Supabase 控制台確認）

**驗收方式**：
```bash
# 檢查 Supabase 環境變數
grep SUPABASE .env.local

# 或直接訪問 dev server
# 預期：登入頁面正常載入
open http://localhost:3000
```

### Layer 2: P1b OAuth 流程（3 min）

**檢查點**：
- [ ] 登入頁面顯示「使用 Google 登入」按鈕
- [ ] 點擊後跳轉 Google OAuth 認證
- [ ] 認證完成後回到應用、顯示用戶名
- [ ] 側欄顯示用戶頭像 + 登出選項
- [ ] 登出後回到登入頁

**驗收方式**：
```
1. 訪問 http://localhost:3000
2. 點「使用 Google 登入」
3. 用 Google 帳號完成認證
4. 確認已登入狀態
5. 點「登出」確認登出功能
```

### Layer 3: P1c KB API MVP（5 min）

**檢查點**：
- [ ] KB 首頁可訪問（`/app/kb` 或主功能頁）
- [ ] KB 主題列表顯示（至少 1 個主題可見）
- [ ] 可新增知識庫項目（標題、內容輸入框）
- [ ] 可編輯項目（修改標題/內容）
- [ ] 可刪除項目（確認對話框 + 刪除成功）

**驗收方式**：
```
[在 dev server 中執行]
1. 瀏覽到 KB 主頁面
2. 新增一個測試項目：
   - 標題：「測試項目」
   - 內容：「測試內容」
3. 確認項目出現在列表
4. 編輯項目（改標題為「修改測試」）
5. 確認列表已更新
6. 刪除項目（確認彈窗 + 完成）
```

### Layer 4: P1d KB UI 完整功能（5 min）

**檢查點**：
- [ ] KB 搜尋框正常（輸入文字 → 篩選結果）
- [ ] 表格全選邏輯（勾選「全選」 → 所有項目勾選）
- [ ] 詳情頁面（點項目 → 展開詳細資訊 + 編輯按鈕）
- [ ] 分頁邏輯（≥20 項時，分頁按鈕出現可用）

**驗收方式**：
```
1. 搜尋框測試：
   - 在搜尋框輸入「測試」
   - 確認列表只顯示相關項目

2. 全選測試：
   - 勾選「全選」checkbox
   - 確認所有項目都被勾選

3. 詳情頁測試：
   - 點擊某個項目
   - 詳情頁應展開或導航到詳情頁面

4. 分頁測試（如有）：
   - 若列表 >20 項，分頁按鈕應可用
```

### Layer 5: P1e Notion 同步（3 min）

**檢查點**：
- [ ] Notion workspace 中的案件數據已連接
- [ ] 新增 KB 項目時，Notion 中相關 case record 可見「KB 連結」
- [ ] 修改 KB 項目，Notion 中反映更新

**驗證方式**：
```
1. 打開 Notion workspace
2. 找到一個 case record（已登入用戶的案件）
3. 新增一個 KB 項目（在 dev server 中）
4. 回到 Notion，刷新 case record
5. 確認新項目已關聯到該 case
6. 編輯 KB 項目（改名字）
7. 回到 Notion 確認同步已更新
```

### Layer 6: P1f 邊界中間件（2 min）

**檢查點**：
- [ ] multi-tenant 隔離生效（用戶 A 看不到用戶 B 的 KB）
- [ ] tenantId 正確傳遞（RLS policy 生效）
- [ ] 無跨租戶資料洩露（API 呼叫無錯誤）

**驗證方式**：
```
1. 用帳號 A 登入、新增 KB 項目 X
2. 記錄項目 ID
3. 登出，用帳號 B 登入
4. 嘗試在 KB 中搜尋項目 X
5. 確認項目不出現（隔離成功）
6. 開發者工具檢查 RLS error logs（無 permission denied）
```
