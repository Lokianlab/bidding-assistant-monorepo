## 主旨

CLAUDE.md「工作行為原則」補充「計畫工作的閱讀義務」

## 問題

本輪 Product Compass 對齁工作中發現：在執行計畫/規劃類任務時，如果沒有完整讀完清單就開始做，容易遺漏邊界條件（如「複雜說明應搬到 docs/methodology/」）。

根本原因是節奏太快導致「邊讀邊做」，而計畫文件（如 Manual-Adjustment-Checklist.md）對執行有法律約束力，不能選擇性讀取。

## 提案

在 CLAUDE.md「工作行為原則」中的「大事先拆」規則後添加：

```
- **計畫工作的閱讀義務**：計畫文件（如 Manual-Adjustment-Checklist.md、決策框架等）對執行有約束力。
  必須完整讀完後才開始做，邊讀邊做容易遺漏邊界條件。清單 ≥3 項要求時，逐項 grep 驗證。
  （出處：檢討 admin-product-compass-align 誤灌複雜內容的教訓）
```

## 位置

```
CLAUDE.md > 工作行為原則 > 「大事先拆」規則之後
```
