### Stage 2b：智能工作流引擎（2026-04-03 ~ 2026-05-01）

**目標**：Claude SDK 驅動的自動化工作流

**工作流核心流程**：
```
使用者上傳 RFQ
  ↓
Claude Agent 自動解析 + 提取需求
  ↓
Agent 調用 Intelligence → 搜尋相似標案
  ↓
Agent 調用 Strategy → 分析競爭對手
  ↓
Agent 調用 Quality → 檢查提案品質
  ↓
匯總成儀表板卡片 → 使用者審閱
```

**待決決策點（Jin 簽核）**：
- [ ] **Decision 1**：Claude SDK 整合方式
  - 選項 A：內嵌 SDK（Web App 內部執行 Agent）
  - 選項 B：外部服務（SaaS 呼叫獨立 Agent 服務）
  - 選項 C：混合模式（核心流程內部，複雜工作流外部）

- [ ] **Decision 2**：計費模型
  - 按 Agent 調用次數
  - 按儲存 Token
  - 按企業訂閱級別（Starter / Pro / Enterprise）

- [ ] **Decision 3**：Beta 發布時機
  - P2a 完成後（2026-04-02）即 Beta
  - P2b 完成後（2026-05-01）才正式上線
  - P2c 完成後（2026-05-22）才發布

- [ ] **Decision 4**：多租戶部署策略
  - 共用 Supabase（成本低，隔離需加強）
  - 租戶獨立 Supabase（隔離完美，成本高）
  - 分層策略（小租戶共用，大租戶獨立）
