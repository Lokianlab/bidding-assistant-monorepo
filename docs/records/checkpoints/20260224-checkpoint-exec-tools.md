```

```

## 三、決策樹

### 3.1 整體進度判斷

```
IF 所有機器 ≥20% 進度
  → 狀態：按時推進 ✅
  → 決策：繼續按 02-25/02-26 時程
  → 行動：鼓勵公告 + 提醒下一階段目標

ELSE IF 1-2 機器無進度
  → 狀態：部分卡點 🟡
  → 決策：JDNE 聯繫卡點機器診斷
  → 行動：如是規格問題 → 釐清後 push；如是技術 → 協助或調整時程

ELSE IF ≥3 機器無進度
  → 狀態：全局阻塞 ❌
  → 決策：暫停推進，升級 JDNE → Jin
  → 行動：分析根因、提出對策

ENDIF
```

### 3.2 環變判斷

```
IF 真實環變已到位
  → 更新 .env.production
  → 標記所有機器可進入集成測試
  → 取消 mock 環變

ELSE IF 環變仍在 mock
  → 允許單元測試繼續
  → 標記集成測試延期（預期 02-25 可開始）
  → 列入 checkpoint 風險

ENDIF
```

### 3.3 時程調整判斷

```
IF {某機進度} <20% AND {阻塞原因無法 2 小時解決}
  → 建議：延期該機 merge deadline 24~48 小時
  → 條件：必須 Jin 批准

ELSE IF {多機阻塞} AND {環變缺失}
  → 建議：全機延期 24 小時，等環變就位
  → 條件：必須 Jin 批准

ENDIF
```
