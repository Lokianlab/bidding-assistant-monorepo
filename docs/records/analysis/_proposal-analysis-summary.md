## 核心發現

### 問題陳述
標案投標市場在 2023-2025 年間經歷快速變化，失敗模式從「程序問題」演進為「內容品質」再到「成本競爭」。理解這些失敗模式對設計「提案寫作駕駛艙」系統至關重要。

### 三年失敗進化

| 年度 | 失敗案例數 | 主要失敗原因 | 市場信號 |
|------|-----------|-------------|---------|
| **112 年度** | 59 個 | 資格標不符（程序問題） | 「做好程序就能過」 |
| **113 年度** | 65 個 | 內容品質不足 | 「程序只是基本，內容決定成敗」 |
| **114 年度** | 60 個 | 競爭激烈（成本議價失敗） | 「程序+內容都好，成本還是會輸」 |

## 成功案例黃金模板（8 層架構）

已從用戶分享的 30+ 個成功結案案例中提取共同結構：

### 1. **封面資訊**
- 案號、機構、項目、投標廠商、聯絡人、日期

### 2. **目次**
- 清晰的結構導覽

### 3. **計畫摘要**
- 基本資訊（目標、經費、期程、窗口）
- **關鍵點**：目標必須量化（「培育 30 人」而非「培育人才」）

### 4. **核心活動計畫**
- 課程/活動清單（含人數、地點、日期）
- 師資背景及專業
- **預期成果**（量化+質化產物）
- **常見失敗**：模糊不清的活動內容

### 5. **宣傳策略**
- 主視覺、社群媒體計畫
- **分波段推廣**：預熱 → 進行中 → 尾聲
- **常見失敗**：只有傳統 DM，沒有整合行銷

### 6. **時程表**
- 精確到月日，標註關鍵里程碑（期中報告、請款點）

### 7. **團隊組織**
- 公司資料、主要成員簡歷
- **履約實績**（過往案例 + 成果數據）
- **常見失敗**：沒有清楚的過往案例展示

### 8. **預算表**
- 清晰的成本分解
- 單位成本合理性

## 詳細案例分析

### 案例 1：113年萬華故事館特展維護管理及教育推廣活動（臺北市文化局 113A119-01）
**預算**：125 萬 | **期程**：2024/4-12 | **結果**：🔴 未中選

**三大致命缺陷**：
1. **履約實績不清** — 無法建立「我們做過這種案子」的信任
2. **活動內容模糊** — 「教育推廣 2 場次」，完全看不清要做什麼
3. **宣傳策略零分** — 只有「DM 2000 份」，沒有社群媒體策略

**對比成功案例**：
- ✅ 成功：黑蝙蝠中隊「8 場講座，戰爭史學家和退役飛行員主講，預計 600-800 人參與，產出簽名冊+成果影片」
- ❌ 失敗：特展「教育推廣活動，2 場次」

**詳細分析文檔**：`docs/records/analysis/failed-case-analysis-113A119.md`

### 案例 2：112年度失敗案例（桃園市政府文化局 1120263-02）
**失敗原因**：「沒有放信用證明，資格標落標」
**特徵**：程序不符就直接淘汰，內容品質無關

**市場意涵**：
- 金主只看資格文件是否完整
- 優秀的內容也無法補救程序失誤

### 案例 3：114年度失敗案例（嘉義市文化基地）
**進度**：序位第 2 → 進議價 → 議價失敗 → 未獲選
**失敗原因**：成本競爭力不足

**市場意涵**：
- 程序 OK、內容好不夠
- 決勝點轉向成本（報價能否競爭）

## 系統設計建議

### 三層驗收架構（對應市場演進）

```
層級 1：程序合規檢查（112年教訓）
├─ 資格文件清單
├─ 法律聲明檢查
└─ 投標截止提醒

層級 2：內容品質檢查（113年教訓）
├─ 活動計畫：清晰度評分
├─ 履約實績：完整度評分
├─ 宣傳策略：創意度評分
└─ 預算透明：邏輯評分

層級 3：成本競爭力檢查（114年趨勢）
├─ 市場行情對標
├─ 成本分解合理性
├─ 議價空間預估
└─ 品質能否保障警示
```

### 提案寫作駕駛艙 UI 流程

1. **案件初始化** → 用戶選「案件類型」（文化展覽、青年課程等）
2. **逐層引導寫作** → 依序填寫 8 層，每層都有成功案例範本
3. **品質自檢面板** → 系統自動評分（內容評分 + 成本排名提示）
4. **導出與版本管理** → 產出可投標的 Word/PDF

**預期改進**：
- 減少提案撰寫時間：5 小時 → 2 小時
- 提升內容一致性：強制填寫 8 層結構
- 減少常見失敗：活動模糊、履約實績缺、宣傳無策

## 後續建議

### 短期（立即可行）
1. **驗證方案可行性** → 用歷史失敗案例走一遍系統流程
2. **提案者使用測試** → 邀請 3-5 個標案經驗者試用駕駛艙
3. **細化檢查清單** → 根據第 4 層（活動計畫）的常見錯誤設計具體問卷

### 中期（2-3 週）
1. **整合市場行情數據** → 建立按案件類型的「行情價資料庫」
2. **增加範本數量** → 提供更多按業別（文化、青年、農業等）的成功範本
3. **AB 測試** → 比較有/無駕駛艙提案的成功率

### 長期（1 個月+）
1. **用戶反饋迴圈** → 收集駕駛艙使用者的投標結果，優化檢查點
2. **跨年度追蹤** → 持續監控失敗模式演進（115年會是什麼？）
3. **評委評語分析** → 若能取得評委評語，進一步提升品質檢查準度
