# 失敗模式進化分析：112-114 年度跨年趨勢

三年份失敗案例對比，揭示標案市場的變化趨勢。

---

## 統計概況

| 年度 | 失敗案例數 | 主要失敗原因 | 演變趨勢 |
|------|----------|-----------|--------|
| **112 年度** | 59 個 | 資格標不符（程序問題） | 門檻守衛 |
| **113 年度** | 65 個 | 內容品質不足 | 品質篩選 |
| **114 年度** | 60 個 | 競爭激烈（序位靠後） | 優勝劣汰 |

---

## 詳細對比

### 112 年度：程序失敗時代

**特徵**：資格標卡關，內容沒機會被看

**代表案例**：桃園市政府文化局 1120263-02
- ❌ 失敗原因：「沒有放信用證明，資格標落標」
- 含義：程序不符就直接淘汰，內容品質無關

**金主評選重點**：
1. ✓ 資格文件是否完整？（信用證明、營登、履約保證）
2. ✓ 投標程序是否規範？
3. ✓ 法律責任聲明是否簽署？
（很少看內容）

**失敗者特徵**：
- 新團隊、不熟悉投標流程
- 以為內容好就能過，忽視程序細節
- 成本：前期資格標卡關，內容做得再好也白搭

---

### 113 年度：品質篩選時代

**特徵**：程序通過，但內容評分低，被淘汰

**代表案例**：臺北市政府文化局 113A119
- ✅ 程序：資格標通過（有所有文件）
- ❌ 失敗原因：內容評分低
  - 活動計畫模糊不清
  - 履約實績不足
  - 宣傳策略零分
  - 預算表不透明

**金主評選重點**：
1. ✓ 程序完整（baseline）
2. ✓ 內容清晰度（50%）
   - 活動是什麼？
   - 推廣怎麼做？
   - 過去做過沒有？
3. ✓ 成本合理性（30%）
4. ✓ 創意度和可行性（20%）

**失敗者特徵**：
- 程序沒問題，但內容表達能力弱
- 「我們想做這件事」，但不清楚怎麼做
- 沒有過往案例佐證
- 成本：資格通過，但品質評分被其他廠商超越

---

### 114 年度：競爭激烈時代（推斷）

**特徵**：程序 OK、內容好，但排名第 2

**代表案例**：嘉義市政府文化局「百大文化基地整合串聯推廣計畫」
- ✅ 程序：資格標通過
- ✅ 內容：進入議價階段（「序位第 2」）
- ❌ 失敗原因：**競爭對手更強** → 議價失敗 / 最後未獲選

**資料夾結構透露的信息**：
- 「評選通知」+ 「序位第 2」+ 「議價結果」 → 說明進階流程
- 「投標成本結算書」+ 「備標評估文件」 → 說明參與了議價
- 最終仍未獲選 → 可能原因：
  1. 報價被議下來，無法接受
  2. 其他廠商報價更低
  3. 其他廠商內容評分更高

**金主評選重點**：
1. ✓ 程序完整（baseline）
2. ✓ 內容品質（門檻高化）
3. ✓ **成本競爭力**（新重點）
   - 價格是否最優？
   - 成本結構是否合理？
   - 有無降價空間？

**失敗者特徵**：
- 內容不錯，但報價相對高
- 或被迫降價，但無法接受新的利潤空間
- 預算表不夠透明，無法有效議價
- 成本：進到 Top 3，但最後卡在成本競爭力

## 三年進化路徑圖

```
112年度              113年度              114年度
程序失敗            品質失敗            競爭失敗
   ↓                 ↓                   ↓
卡在               卡在               卡在
資格標             內容評分           成本議價

金主篩選條件演變：
程序完整度 ────→ 內容品質 ────→ 整體競爭力
(必須 100%)  (內容評分竞争) (程序+品質+成本均衡)

市場成熟度：
新市場           成熟市場           激烈市場
(卡程序)         (卡品質)          (卡價格和品質)
```

## 對提案寫作駕駛艙的多年啟示

### 1. **程序清單必備**（對應 112 年教訓）
系統應提醒用戶檢查：
```
□ 資格文件完整（統編、營登證、履約保證）
□ 信用證明已備
□ 所有法律聲明已簽署
□ 投標截止時間前提交
→ 缺一不可，否則直接落選
```

### 2. **內容品質檢查必備**（對應 113 年教訓）
系統應強制填寫 8 層結構，並檢查：
```
第 4 層（活動計畫）：活動名稱、場次、人數、師資、產物都清楚嗎？
第 5 層（宣傳策略）：有分波段嗎？有社群計畫嗎？
第 7 層（團隊組織）：有履約實績嗎？數據清楚嗎？
第 8 層（預算表）：分項清楚嗎？單位成本合理嗎？
→ 缺一項都會被扣分
```

### 3. **成本競爭力檢查必備**（對應 114 年趨勢）
系統應提醒用戶：
```
□ 預算表能否支撐議價？
□ 成本分解是否有下降空間？
□ 與市場行情比，報價是否偏高？
□ 降價後還能保證品質嗎？
→ 內容再好，價格高也會被超越
```

### 4. **跨年趨勢預測**
- **115 年預測**：競爭持續激烈 → 內容品質要求更高 + 成本更敏感
- **應對策略**：
  1. 強化「過往案例展示」（履約實績）
  2. 提高「成本透明度」（分項清楚、邏輯合理）
  3. 領先對手「1-2 個創意點」（宣傳策略、活動設計）

## 市場信號

### 對投標廠商的啟示

**112 年**：「做好程序，基本就過了」
- 競爭不激烈，只要不犯程序錯誤，內容中等也能中選

**113 年**：「程序只是基本，內容決定成敗」
- 競爭開始升溫，內容品質成為新門檻
- 履約實績、宣傳策略、活動細節成為評選重點

**114 年**：「程序+內容都好，成本還是會輸」
- 市場進入紅海
- 多家廠商都能做好內容
- 決勝點轉向：**誰的成本更有競爭力**

### 對金主的啟示

- **112 年心態**：「這些廠商靠譜嗎？」（卡程序）
- **113 年心態**：「這個廠商能做好嗎？」（卡內容）
- **114 年心態**：「這 3 家都不錯，誰的性價比最高？」（卡成本）

## 對「提案寫作駕駛艙」系統架構的影響

系統應分三層把關，對應三年演變：

```
層級 1：程序合規檢查
├─ 資格文件清單
├─ 法律聲明檢查
└─ 投標截止提醒

層級 2：內容品質檢查（8 層模板）
├─ 活動計畫：清晰度評分
├─ 履約實績：完整度評分
├─ 宣傳策略：創意度評分
└─ 預算透明：邏輯評分

層級 3：成本競爭力檢查
├─ 市場行情對標
├─ 成本分解合理性
├─ 議價空間預估
└─ 品質能否保障警示

用戶提交前：系統輸出「內容評分」+「成本排名」提示
→ 讓用戶知道自己在市場上的位置
```

---

## 結論

**112→113→114 年的三年變化，反映標案市場從「守衛程序」→「篩選品質」→「競爭成本」的自然演進。**

對策案助理系統的設計建議：
1. **不只要幫用戶寫好提案**，還要幫用戶了解**市場競爭位置**
2. **不只要檢查內容品質**，還要檢查**程序合規** + **成本合理性**
3. **不只要提供範本**，還要提供**市場情報**（行情價、競品信息）

這樣的系統才能真正提升用戶的中選率。
