針對特定議題的深度整理工具，產出具有下游指導性的詳細文件。

## 定位

| /暫存 | /小結 | OP 記錄 |
|-------|------|--------|
| 窄且深（單一議題） | 廣且淺（整場對話） | 單次操作事實 |
| 下游指導文件 | 歷史敘事 | 事實記錄 |
| 告訴下游「怎麼做」 | 告訴人「發生了什麼」 | 告訴系統「做了什麼」 |

## 步驟

1. **前置同步**：執行 /更新

2. **確定議題**
   - 用戶直接指定（如 `/暫存 MEMORY.md 維護規則`）→ 直接用
   - 用戶沒指定 → 從 context + 記錄層推斷，用人話列出候選：

     「這次聊了幾個主題，你想暫存哪一個？
      1. MEMORY.md 維護規則的改版
      2. 規範有效性的整體評估
      3. /暫存 指令的重新設計
      多個也可以，我一個一個來。」

3. **蒐集資料**（記錄層 + context）

   蒐集範圍取決於是否已有同 topic 的暫存檔：
   - 無既有暫存 → 該 topic 的全部 OP + context（從頭建檔）
   - 有既有暫存 → 既有暫存 + 暫存日期之後的 OP + context（增量更新）

   具體資料來源：
   - `_index.md` 中該 topic 的狀態和出處
   - OP 記錄（`grep "topic:{id}"` 撈出全部/增量）
   - 各機器 snapshot 中該 topic 的項目
   - 既有暫存檔（如果有）
   - Context window 中尚未寫入 OP 的討論

4. **判斷：已有同 topic 暫存檔？**
   - 有 → 讀取，合併新資訊，更新原檔
   - 無 → 新建

5. **產出暫存文件**（格式見下）

6. **更新 `_staging/_index.md`**

7. **git add → commit → push**（依 CLAUDE.md 同步規則）

## 暫存檔格式

```markdown
# {議題標題}

類別：{plan/rule}
目標：{target 檔案}
日期：{YYYY-MM-DD}
相關 OP：{@op:引用列表}

---

## 背景

{為什麼討論這個議題，來龍去脈}

## 結論

{最終決策和理由。待決時列出選項和各自優缺。}

## 寫入內容

{要寫進目標文件的具體文字，完整到可以直接貼上}

## 整合指南

{怎麼跟目標文件現有內容銜接：替換哪段、插入哪裡、刪除什麼}
```

## Type 分類

| type | 用途 | 下游動作 |
|------|------|---------|
| `plan` | 開發計畫內容 | `/修改計畫` 推入 |
| `rule` | 規範/規則變更 | 用戶確認後直接編輯 |

## 索引格式

`_staging/_index.md` 每行格式：

```
{filename}|{type}|{target}|{status}|{summary}
```

- type: `plan` / `rule`
- status: `待推`（結論明確）/ `待決`（結論未定）

## 注意事項

- 用戶面對的所有提示都用人話，不出現 topic ID、OP 引用或格式化標記
- 不修改任何正式文件（正式文件只有 `/修改計畫` 才能動）
- 完成後列出所有建立/更新的暫存檔清單
