## 步驟

1. 回顧本次對話中的所有決策和結論
2. 為每個結論建立一份暫存檔，檔名格式：`{目標文件編號}-{主題關鍵詞}.md`
   - 例：`03-Agent成本方案.md`、`M01-快取改IndexedDB.md`
3. 每份暫存檔必須包含以下結構：

```markdown
# 暫存：{主題}

- **目標文件**：{對應哪份 docs/dev-plan/ 正式文件}
- **操作**：新增 / 修改 / 刪除
- **來源對話**：{討論主題}（{日期}）

## 內容

{要寫入的具體文字}
```

4. 如果 `_staging/` 資料夾不存在，先建立
5. **更新暫存索引** `_staging/_index.md`：為每份新建的暫存檔追加一行
   - 格式：`{filename}|{target-doc}|{status}|{summary}`
   - status：結論明確 → `待推`；結論未定 → `待決`
6. **不修改任何正式文件**（正式文件只有 `/修改計畫` 才能動）
7. 完成後列出所有建立的暫存檔清單
8. git add → commit → push（依 CLAUDE.md 同步規則）
