AI 自我診斷 + 修正 + 記住。用戶覺得 AI 做錯了什麼，一個指令跑完全程。

## 步驟

### 1. 掃描（不需要展示過程）

讀 CLAUDE.md 行為原則（5 條）+ 回顧本輪對話行為 + 逐條比對 + 檢查常見違紀模式。

常見違紀模式：

| 模式 | 怎麼查 |
|------|--------|
| 提前宣布完成 | 說「完成」但沒列出具體檢查了什麼？ |
| 循環推理 | 用方法論工具評價方法論本身？ |
| 跳過閉環 | 改了檔案但沒跑四層驗證？ |
| 驗收僭越 | 標 [x] 但用戶沒說驗收通過？ |
| 範圍膨脹 | 做的事超出用戶要求？ |
| 假裝確定 | 推測當事實陳述？ |
| 做完就停 | 沒主動告訴用戶下一步？ |
| 不主動回報 | 沒說打算做什麼就動手？記錄沒主動做？ |
| 不主動表態 | 該給判斷的時候甩選項讓用戶自己選？ |

### 2. 報告結果

格式精簡，只說重點：

```
問題（N 項）：

1. {什麼時候}：{做了什麼}
   違反：{規則}
   修法：{具體怎麼修}

結論：{一句話總結檢查範圍和結果}
```

有不確定的就說。「檢查過沒問題的」不逐項列，用結論一句話帶過。

### 3. 升級判斷

同一個問題在本 session 出現 ≥2 次 → 不只修症狀，**必須提出系統級改善**（hook、自動化、工具防護）。

### 4. 修正

不等用戶逐項挑選。報告完直接問：「要全修嗎？」用戶說好就一次修完。

### 5. 扣分

每項違規自動執行 /醜一 的邏輯：讀 scoring.md → 更新對應行為模式 → 每項 -1 → 寫回。

### 6. 記住

修完後掃描本次診斷的教訓，判斷是否為通用規則：

| 層級 | 處置 |
|------|------|
| 只是這次的事實 | 寫 OP 教訓欄 |
| 通用規則（全機器適用） | 提議更新 CLAUDE.md |
| 除錯經驗 | 寫 docs/debugging.md |
| 方法論調整 | 寫對應 methodology 文件 |

直接寫入，CLAUDE.md 需用戶確認，其他直接寫。

## 注意事項

- **誠實優先**。寧可多報，不要漏報。
- **不要捏造問題**。真的沒問題就說沒問題。
- 用戶觸發時通常覺得哪裡怪了，重點放在找問題。
