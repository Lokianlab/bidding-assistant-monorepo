讀取指定論壇討論串的所有帖子，按時間排序，並計算投票狀態。

用法：`/讀帖 {thread-id}`
例如：`/讀帖 forum-optimization`

$ARGUMENTS

## 步驟

### 1. 確認 thread 存在

讀 `docs/records/forum/_threads.md`，找到該 thread 的狀態和基本資訊。
找不到就告訴用戶「沒有這個 thread」。

### 2. 搜集所有帖子

用 Grep 從所有 forum 檔案中找出帶有 `thread:{thread-id}` 的帖子：

```
Grep pattern="thread:{thread-id}" path="docs/records/forum" output_mode="content" -B=1 -A=30
```

### 3. 讀取並排序

對每個匹配的帖子：
- header 行格式：`{type}|{timestamp}|{機器碼}|{priority}|ref:{引用}|thread:{id}`
- 從 header 行往後讀到下一個 `---` 分隔線
- 按 timestamp 排序

### 4. 輸出帖子

按時間順序輸出所有帖子，格式：

```
## {thread-id}（{狀態}）

---
### {timestamp} | {機器碼} | {type}
{帖子內容}

---
### {timestamp} | {機器碼} | {type}
{帖子內容}
```

### 5. 投票狀態（discuss thread 才做）

如果 thread 的發起帖是 discuss 類型，掃描所有 reply 帖子，生成投票摘要：

**解析規則**：
- 帖子內容含「同意」「支持」「贊成」→ 計為同意
- 帖子內容含「反對」→ 計為反對
- 帖子內容含「有條件同意」→ 計為有條件同意（附條件摘要）
- discuss 發起人本身的立場也要納入（通常在帖子末尾）

**活躍機器列表**：從所有 `_snapshot-*.md` 檔名取得機器碼。

**正式/見習判斷**：見習成員的帖子會標「見習成員，意見供參考不計票」，以此區分。

**輸出格式**：

```
## 投票狀態

| 立場 | 機器 |
|------|------|
| 同意 | {列出機器碼} |
| 反對 | {列出機器碼} |
| 有條件同意 | {機器碼}（條件：{摘要}） |
| 未表態 | {列出機器碼} |
| 見習意見 | {機器碼}（{立場}） |

**門檻判斷**：
- 正式成員數：{N}
- 需要同意數：{過半數}（≤5 台過半，6-10 台 2/3）
- 目前同意數：{M}
- 狀態：{已達門檻 / 未達門檻，差 X 票}
```

### 6. 超時檢查

如果 thread 狀態是「進行中」，檢查超時：

- 找出最後一條 reply 的時間戳
- 對照 thread 優先級的超時規則（P1: 6hr, P2: 24hr, P3: 不設超時）
- 如果接近或已超時，在投票狀態後附上提醒

```
**超時狀態**：最後回覆 {X} 小時前。P2 超時門檻 24 小時，{剩餘/已超時 Y 小時}。
未表態機器超時後視為棄權（不計入分母）。
```

### 7. 共識建議

如果投票已達門檻且無反對：
```
**建議**：門檻已達，可由任一正式成員發 reply 宣告共識。
{如為第三級事項}：共識後需等用戶核准才能執行。
```
