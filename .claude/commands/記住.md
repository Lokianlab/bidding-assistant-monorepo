跨 session 學習：用戶糾正或指示時，把結論寫進對的持久位置，下次新對話能讀到。

## 參數

可選：要記住的內容（如 `/記住 永遠用 bun 不用 npm`）。沒帶參數就從最近對話中找用戶糾正或指示。

## 步驟

1. **辨識要記的內容**
   - 有參數 → 直接用
   - 沒參數 → 掃描最近對話，找用戶的糾正（「不要這樣做」「以後用這個」）或指示（「記住」「永遠」「不要再」）
   - 找不到 → 問用戶想記什麼

2. **分流到正確位置**

   先搜尋目標檔案，確認不重複已有內容。

   | 內容性質 | 判斷條件 | 寫到哪裡 |
   |----------|---------|---------|
   | 規則/慣例（全機器適用） | 所有機器都該遵守 | 提議更新 CLAUDE.md |
   | 除錯經驗（根因明確） | 能說清楚為什麼出錯 | docs/debugging.md |
   | 方法論調整 | 改變做事流程 | 對應的 methodology 文件 |
   | 本機限定偏好 | 只跟這台機器有關 | MEMORY.md |
   | 決策/結論（需推開發計畫） | 影響產品方向 | 建議用 /暫存 |

   不確定該寫哪裡 → 問用戶。

3. **寫入**
   - 讀取目標檔案，找合適的插入位置
   - 寫入，保持格式一致
   - **CLAUDE.md 需用戶確認**：先告訴用戶打算改什麼，用戶同意後才寫入。不能 AI 自行決定改規範。
   - CLAUDE.md 的修改最小化——只加必要的行，不重組結構

4. **回報**
   - 列出：寫了什麼、寫到哪個檔案的哪個位置
   - 如果寫到 CLAUDE.md → 提醒「這會影響所有機器，下次 git push 後生效」
   - 如果寫到 MEMORY.md → 提醒「這只在本機生效」

## 完成標準

用戶的指示已寫進持久位置。新 session 讀取該檔案時能看到。
