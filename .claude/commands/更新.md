對話中手動同步開發環境到最新狀態（不含 npm 驗證）。

## 步驟

### 1. 拉最新程式碼

```bash
cd /c/dev/cc程式
git pull origin main
```

如果報衝突，檢查衝突檔案並解決（記錄層衝突永遠保留兩邊記錄）。

### 2. 碰撞偵測

#### 2a. CLAUDE.md 一致性檢查

如果 pull 進來的改動包含 CLAUDE.md：
1. 讀取最近的 diff（`git log --oneline -5` 找到改動的 commit，`git diff` 看內容）
2. 檢查新增/修改的規則是否與現有規則矛盾
3. 有矛盾 → 告知用戶，列出矛盾點，等用戶裁決

#### 2b. Topic ID 重複掃描

讀取 `_index.md`，檢查有沒有不同 ID 但描述同一概念的 topic。判斷依據：
- 同一個 topic 前綴出現多次（如 `method-self-qa` 和 `doc-methodology-self-qa`）
- 一句話結論高度相似

有重複 → 告知用戶，建議合併成哪個 ID。

#### 2c. MEMORY.md 清理

如果 CLAUDE.md 有變更，讀取本機 MEMORY.md，刪除已被 CLAUDE.md 涵蓋的項目。

### 3. 恢復上下文

執行分層讀取：
- Layer 0: `_index.md` → 全局現況
- Layer 0.5: `_staging/_index.md` → 待推結論 + 待決討論
- Layer 1: `_snapshot-{自己}.md` → 目前進度
- Layer 2: 所有 `_snapshot-*.md` → 其他機器進度
- Layer 3: 近期 OP header（最多 30 筆）

產出差異摘要：其他機器最近做了什麼，同 topic 多台機器 → 標出。

### 4. 完成

告知使用者：
- 目前版本狀態（最新 commit）
- 是否有新增或變更的規範
- 是否偵測到碰撞（CLAUDE.md 矛盾、Topic ID 重複）
