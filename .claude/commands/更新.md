## 輕量版（預設）

### 1. 拉最新

```bash
git pull origin main
```

衝突 → 解決（記錄層保留兩邊）→ commit → push。

### 2. 看誰做了什麼

```bash
git log --oneline -15
```

用一句話摘要：「{機器碼} 推了 N 個 commit（關鍵字）」。只看 commit message，不讀檔案。

### 3. 掃論壇

讀 `docs/records/forum/_threads.md`（一個檔案掌握全局）：
- 有「進行中」的 thread 且自己沒回覆 → 記下待回覆

沒有新 forum 檔案 → 跳過。

### 4. 條件觸發（只在有變化時才做）

| 條件 | 動作 |
|------|------|
| pull diff 包含 CLAUDE.md | 讀 diff，檢查新規則是否與現有矛盾，有矛盾才報 |
| pull diff 包含 CLAUDE.md | 讀 MEMORY.md，刪已涵蓋項目 |
| 無變化 | 什麼都不做 |

### 5. 讀自己的快照

```
Read _snapshot-{自己}.md
```

只讀自己的，不讀其他機器的快照。接回上次工作。

### 6. 報告

一句話：版本狀態 + 有無規範變更 + 有無論壇待回覆。不展開分析。

## 完整版（`/更新 完整`）

在輕量版基礎上額外做：

- Topic ID 重複掃描（讀 `_index.md`）
- 讀所有 `_snapshot-*.md`（其他機器進度）
- 讀 `_staging/_index.md`（暫存區狀態）
- 讀 `docs/records/forum/_threads.md` + 進行中 thread 的相關帖子
- 產出差異摘要：其他機器做了什麼，同 topic 多台 → 標出，論壇意見 → 摘要
