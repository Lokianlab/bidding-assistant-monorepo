將既有機器的開發環境同步到最新狀態。

## 步驟

### 1. 拉最新程式碼

```bash
cd /c/dev/cc程式
git pull origin main
```

如果報衝突，檢查衝突檔案並解決（記錄層衝突永遠保留兩邊記錄）。

如果報錯 `unable to map index file` 或其他 git 損壞：
```bash
git fetch origin
git reset --hard origin/main
```
⚠ 告知使用者這會丟掉本地未推送的修改。

### 2. 更新依賴

```bash
cd /c/dev/cc程式/bidding-assistant
npm install
```

### 3. 驗證

```bash
cd /c/dev/cc程式/bidding-assistant
npm test          # 預期：全過
npm run build     # 預期：0 errors
```

如果測試失敗，嘗試清除重裝：
```bash
rm -rf node_modules
npm install
npm test
```

### 4. 載入最新規範

Pull 完成後，重新讀取以下兩份 CLAUDE.md，並在本次 session 中遵守其中的所有規則：

- `/c/dev/cc程式/CLAUDE.md`（根目錄共用規範）
- `/c/dev/cc程式/bidding-assistant/CLAUDE.md`（Web App 開發規範）

### 5. 碰撞偵測

#### 5a. CLAUDE.md 一致性檢查

如果 pull 進來的改動包含 CLAUDE.md：
1. 讀取最近的 diff（`git log --oneline -5` 找到改動的 commit，`git diff` 看內容）
2. 檢查新增/修改的規則是否與現有規則矛盾
3. 有矛盾 → 告知用戶，列出矛盾點，等用戶裁決

#### 5b. Topic ID 重複掃描

讀取 `_index.md`，檢查有沒有不同 ID 但描述同一概念的 topic。判斷依據：
- 同一個 topic 前綴出現多次（如 `method-self-qa` 和 `doc-methodology-self-qa`）
- 一句話結論高度相似

有重複 → 告知用戶，建議合併成哪個 ID。

#### 5c. MEMORY.md 清理

如果 CLAUDE.md 有變更，讀取本機 MEMORY.md，刪除已被 CLAUDE.md 涵蓋的項目。

### 6. 完成

告知使用者目前版本狀態（最新 commit、測試結果、build 結果），以及：
- 是否有新增或變更的規範
- 是否偵測到碰撞（CLAUDE.md 矛盾、Topic ID 重複）
