對話中手動同步開發環境到最新狀態（不含 npm 驗證）。

## 步驟

### 1. 拉最新程式碼

```bash
cd /c/dev/cc程式
git pull origin main
```

如果報衝突，檢查衝突檔案並解決（記錄層衝突永遠保留兩邊記錄）。

### 2. 工具檢查

#### 2a. CLI 工具

檢查以下指令是否存在（`which` 或 `--version`）：

| 工具 | 檢查指令 | 不存在時 |
|------|---------|---------|
| Gemini CLI | `gemini --version` | `npm install -g @google/gemini-cli` |
| Codex CLI | `codex --version` | `npm install -g @openai/codex` |

缺的直接裝，裝完告知用戶。

#### 2b. MCP Server

讀取專案根目錄 `.mcp.json`，對每個 server 做基本檢查：

- `command` 是 `npx` → 跳過（npx 會自動拉）
- `command` 是 `node` + 指向 `dist/index.js` → 檢查該檔案是否存在
  - 不存在 → 告知用戶：「{server} 的 {path} 不存在，可能需要 `npm install && npm run build`」
- 其他 → 檢查 command 是否可執行

列出檢查結果，有問題的標出。

### 3. 碰撞偵測

#### 3a. CLAUDE.md 一致性檢查

如果 pull 進來的改動包含 CLAUDE.md：
1. 讀取最近的 diff（`git log --oneline -5` 找到改動的 commit，`git diff` 看內容）
2. 檢查新增/修改的規則是否與現有規則矛盾
3. 有矛盾 → 告知用戶，列出矛盾點，等用戶裁決

#### 3b. Topic ID 重複掃描

讀取 `_index.md`，檢查有沒有不同 ID 但描述同一概念的 topic。判斷依據：
- 同一個 topic 前綴出現多次（如 `method-self-qa` 和 `doc-methodology-self-qa`）
- 一句話結論高度相似

有重複 → 告知用戶，建議合併成哪個 ID。

#### 3c. MEMORY.md 清理

如果 CLAUDE.md 有變更，讀取本機 MEMORY.md，刪除已被 CLAUDE.md 涵蓋的項目。

### 4. 恢復上下文

執行分層讀取：
- Layer 0: `_index.md` → 全局現況
- Layer 0.5: `_staging/_index.md` → 待推結論 + 待決討論
- Layer 1: `_snapshot-{自己}.md` → 目前進度
- Layer 2: 所有 `_snapshot-*.md` → 其他機器進度
- Layer 3: 近期 OP header（最多 30 筆）

產出差異摘要：其他機器最近做了什麼，同 topic 多台機器 → 標出。

### 5. 完成

告知使用者：
- 目前版本狀態（最新 commit）
- 是否有新增或變更的規範
- 是否偵測到碰撞（CLAUDE.md 矛盾、Topic ID 重複）
- 工具檢查結果（缺的已自動安裝 / MCP 有問題的列出）
