## 輕量版（預設）

### 1. 同步

```bash
git fetch origin && git rebase origin/main
```

衝突 → 解決（記錄層保留兩邊）→ commit → push。

### 2. 掃描（並行讀取）

同時讀兩個檔案：

| 檔案 | 看什麼 |
|------|--------|
| `_threads.md` | 進行中 thread 有沒有自己沒回的（P0/P1 優先）|
| `_snapshot-{自己}.md` | 接回上次工作，確認下一步 |

### 3. 條件觸發（有變化才做）

| 條件 | 動作 |
|------|------|
| pull diff 包含 CLAUDE.md | 讀 diff 檢查新規則有無矛盾；讀 MEMORY.md 刪已涵蓋項目 |
| 無 CLAUDE.md 變化 | 跳過 |

### 4. 報告

一句話涵蓋四項：同步結果 + 規範有無變更 + 論壇有無待回覆 + 快照下一步是什麼。

## 完整版（`/更新 完整`）

在輕量版基礎上額外做：

- 讀所有 `_snapshot-*.md`，同 topic 多台 → 標出碰撞風險
- 讀 `_index.md` 掃描 Topic ID 重複
- 讀 `_staging/_index.md`（暫存區狀態）
- 產出差異摘要：其他機器做了什麼 + 碰撞 + 論壇意見
