將既有機器的開發環境同步到最新狀態。

## 步驟

### 1. 拉最新程式碼

```bash
cd /c/dev/cc程式
git pull origin main
```

如果報衝突，檢查衝突檔案並解決（操作日誌衝突永遠保留兩邊記錄）。

如果報錯 `unable to map index file` 或其他 git 損壞：
```bash
git fetch origin
git reset --hard origin/main
```
⚠ 告知使用者這會丟掉本地未推送的修改。

### 2. 更新依賴

```bash
cd /c/dev/cc程式/bidding-assistant
npm install
```

### 3. 驗證

```bash
cd /c/dev/cc程式/bidding-assistant
npm test          # 預期：全過
npm run build     # 預期：0 errors
```

如果測試失敗，嘗試清除重裝：
```bash
rm -rf node_modules
npm install
npm test
```

### 4. 載入最新規範

Pull 完成後，重新讀取以下兩份 CLAUDE.md，並在本次 session 中遵守其中的所有規則：

- `/c/dev/cc程式/CLAUDE.md`（根目錄共用規範）
- `/c/dev/cc程式/bidding-assistant/CLAUDE.md`（Web App 開發規範）

### 5. 完成

告知使用者目前版本狀態（最新 commit、測試結果、build 結果），以及是否有新增或變更的規範。
