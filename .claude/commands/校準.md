全面校準：診斷偏離，發校準指令讓所有機器重新對齊用戶終極需求。

## 觸發

`/校準` 或 `/校準 {議題}`

## 流程

### 1. 同步 + 輕量掃描

```bash
git fetch origin && git rebase origin/main
```

**必讀（4 個來源，~300 行，覆蓋 90% 資訊）：**

| 來源 | 提供什麼 |
|------|---------|
| `docs/records/_index.md` | 所有 topic 的一句話現況 |
| 所有 `docs/records/_snapshot-*.md` | 各機器進度和狀態 |
| `docs/records/forum/_threads.md` | 論壇討論全局 |
| `.claude/rules/scoring.md` | 行為模式（用腳本統計，不手動數） |

**計分統計腳本**（一行出結果）：
```bash
grep "^[+-][0-9]" .claude/rules/scoring.md | awk -F'|' '{split($3,a,":"); m=a[1]; gsub(/^ +| +$/,"",m); t[m]+=$1+0; c[m]++} END {for(i in t) printf "%s|淨分:%d|筆數:%d\n",i,t[i],c[i]}' | sort -t'|' -k2 -n
```

### 2. 選擇性深入（只在 Pass 1 不夠時）

| 需要什麼 | 才讀什麼 |
|---------|--------|
| 某 thread 的完整討論 | 該 thread 相關的 forum 帖 |
| 某 topic 的操作細節 | 該 topic 的 OP 記錄 |
| 整體產品方向評估 | `docs/dev-map.md` |
| 特定行為模式的細節 | scoring.md 中該模式的行（grep 篩選） |

**有議題** → 只深入議題相關的來源。
**無議題** → 也只在 Pass 1 發現具體問題時才深入，不全撈。

### 3. 診斷 + 校準指令 + OP + 推送

**診斷三維度**（有議題時只做相關的）：

| 維度 | 核心問題 |
|------|---------|
| 方向 | 在做的事有沒有推進用戶終極需求？基建/產品比例？ |
| 效率 | 產出價值 vs 消耗成本？重複犯的行為模式？ |
| 盲區 | 重要的事沒人做？待決太久（[?] >3 天）？過時計畫？ |

**一次性產出並推送**（不分步）：

a. 校準指令寫入論壇（`docs/records/forum/{YYYYMMDD}-{本機碼}.md`）：
```
directive|{YYYYMMDD-HHMM}|{本機碼}|ref:{用戶指示 or none}
觸發：/校準 {議題}

## 診斷（2-3 段，數據佐證，用人話）
## 各機器：停止 / 繼續 / 開始
## 全局決策（如有）
---
```

b. OP 記錄（topic: infra-forum）+ 快照更新
c. `git add → commit -m "校準：{一句話}" → push`

### 4. 給用戶摘要

- 根因（1-3 句）
- 計分表
- 驗收堆積 → 附批量驗收清單
- 幕僚自己的改善行動

## 原則

- **讀最少做最準**——4 個檔能回答就不深入。上次讀 9 個來源吃掉整個 context 是反面教材
- **數據驅動**——計分統計用腳本、topic 分佈看 _index.md、快照看標記分佈。不靠感覺
- **不美化**——哪台效率低就說，哪個方向偏了就指出
- **有議題就聚焦**——不做全面掃描
- **校準指令是建議**——機器可在 reply 中異議（對等關係）
