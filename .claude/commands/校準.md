緊急立法會議：快速建構問題框架，發 P0 指令要求全員停工討論，得出結論，形成規章。

## 觸發

`/校準 {問題}`

問題描述必填。用戶已經知道問題在哪——AI 不需要重新發現。

## 流程

### 1. 同步

```bash
git fetch origin && git rebase origin/main
```

### 2. 建構問題框架（5 分鐘內完成）

把用戶描述的問題，翻譯成效率公式的語言：

> **效率 = 產出價值 / 消耗成本**（時間 + 溝通 + 返工 + 認知負擔 + 維護成本）

三步：

**a. 分子**：產出價值哪裡受損？
  品質低？方向偏？速度慢？該做的沒人做？

**b. 分母**：消耗成本哪裡膨脹？
  時間浪費？溝通往返？反覆返工？認知負擔？維護開銷？

**c. 指標**：對應元方法論的哪些操作化指標？

| 類別 | 指標 |
|------|------|
| A 成果 | A1 首次接受率、A2 返工次數、A3 事後發現率 |
| B 過程 | B1 判斷可見率、B2 問題定位速度、B3 意外率 |
| C 成本 | C1 總輪次、C2 用戶介入頻率、C3 同類效率趨勢 |
| D 成長 | D1 問題復發率、D2 介入層級趨勢、D3 新情境處理 |

指標回答「怎麼量修好了沒」。

需要佐證時快速掃（不必全讀）：
- `scoring.md`（腳本統計行為模式）
- `_snapshot-*.md`（各機器在做什麼）
- `_index.md`（topic 分佈）

### 2.5. 向用戶確認問題框架（不可跳過）

把框架結果用一句話說回給用戶：「你的問題是 {X}，對嗎？」

**為什麼這步是硬性的**：AI 會把用戶的問題翻譯成自己理解的版本，然後整場討論都在回答 AI 理解的問題，不是用戶的問題。六台機器各自猜測問題是什麼，等於六場平行的自說自話。

- 用戶說「對」→ 繼續步驟 3
- 用戶修正 → 重做步驟 2
- 用戶說「不是」→ 請用戶重新描述問題

**P0 討論帖中必須包含用戶確認過的問題原文**，讓所有機器回答同一個問題。

### 3. 擬定討論議程

把問題框架轉化為 2-3 個具體問題。每個問題：
- 明確到能回答「我認為 X，因為 Y」
- 不能是開放式的「大家怎麼看」
- 預設回答格式

同時定義期望結論的形式：一條規則？一個流程？一個決策？

## 回覆要求（全部必須滿足，不達標退回重寫）

### 一、深度自我檢討（用事後回顧方法論）

不接受表面描述。「我會加個檢查流程」不夠——為什麼之前沒有？制度哪裡沒接住？

必須：
- 用根因追問逐層往下追（至少 3 層 why）
- 每一層有查證（OP 記錄、commit 歷史、scoring 模式），不靠推理
- 追到結構性根因：為什麼這個問題會反覆發生？

### 二、效率公式自我分析

用「產出價值 / 消耗成本」拆解本機的時間和 token 分佈：
- 花在什麼上？（產品開發 / 基建 / 同步 / 衝突解決 / 論壇 / 行政）
- 哪些消耗可以砍？哪些產出可以加？
- 用數據佐證（commit 計數、OP 筆數、topic 類別分佈）

## 後續流程（等回覆後執行）

### 三輪答辯（最低門檻，不可跳過）

共識必須經過至少三輪答辯才能收。「大家都同意」不夠——要經得起反覆攻防。

**第一輪：各機器初始回覆**
- 滿足三項回覆要求（深度自我檢討 + 效率公式分析 + 對全局方案表態）
- 不達標的退回重寫，不算一輪

**第二輪：交叉質疑**
- 每台機器必須針對其他機器第一輪的回覆逐一回應
- 重點：挑戰對方的根因分析、質疑數據、指出邏輯漏洞、提出替代解讀
- 不能只說「同意」——要補充新的論據或指出對方沒看到的面向

**第三輪：最終立場**
- 看完所有第二輪的交叉質疑後，每台機器表明最終立場
- 如果立場改變，說明是什麼論據說服了自己
- 如果立場不變，說明為什麼第二輪的質疑沒有動搖

三輪完成後才進入綜合階段。用戶可以要求加開更多輪。

### 綜合 → 結論

三輪答辯完成後：

1. 歸納共識和分歧（標出各輪立場變化）
2. 分歧用效率公式判斷——哪個選擇讓效率更高
3. 提出結論草案，發到同 thread
4. 無異議或用戶裁決後，結論定案

## 原則

- **用戶知道問題**——AI 結構化它，不重新發現它
- **框架要快**——5 分鐘，不是研究報告
- **P0 不濫用**——只有用戶觸發 /校準 才發
- **對齊終極目標**——問題框架永遠連結到 效率 = 產出價值 / 消耗成本
- **結論必須落地**——寫進文件或設定，「大家注意一下」不算結論
- **機器可異議**——P0 要求回覆，回覆可以是反對（但必須附理由和替代方案）
