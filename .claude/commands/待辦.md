彙整全專案的待辦事項，從固定來源掃描，確保結果穩定完整。

## 流程

1. **同步**：`git pull origin main`

2. **掃描來源**（全部必掃，順序固定）
   a. 所有 `docs/records/_snapshot-*.md` → 提取 `[ ]`（待做）和 `[?]`（未決）項目
   b. `docs/records/_index.md` → 提取狀態為「進行中」的 topic
   c. `bidding-assistant/docs/dev-plan/_staging/_index.md` → 列出所有暫存項目
   d. 暫存檔內容 → 逐檔讀取，提取明確命名的交付物或任務（如「建 X」「實作 Y」「T1 知識庫精靈」）
   e. `docs/summaries/_index.md` → 找近期小結 → 讀 C 章待決策項

3. **去重合併**：同一概念出現在多個來源 → 合併為一項，標註所有來源

4. **分類排序**
   - 待做：明確要建或實作的東西
   - 待決：需要討論或選擇的東西
   - 等待中：依賴外部條件（如等資料）
   - 排序：待做 → 待決 → 等待中

5. **輸出表格 + 摘要**

## 輸出格式

```
| # | 項目 | 狀態 | 來源 |
|---|------|------|------|
| 1 | ... | 待做 | ... |

共 N 項（X 待做、Y 待決、Z 等待中）
```

## 規則

- 純讀取，不修改任何檔案
- 暫存檔內容中，只提取明確命名的交付物或任務，不提取模糊的未來構想
- 用人話呈現，不出現 topic ID 或 OP 引用
