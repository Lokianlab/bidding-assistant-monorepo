將暫存區的討論結論正式寫入開發計畫文件。

## 步驟

1. 掃描 `bidding-assistant/docs/dev-plan/_staging/` 所有暫存檔（排除 `_done/`）
2. 如果暫存區是空的，告知使用者「沒有待寫入的暫存檔」並結束
3. 列出所有暫存檔清單，讓使用者確認要寫入哪些（可選擇部分）
4. 若多份暫存檔指向同一目標文件 → 先整合，標出衝突點讓使用者裁決
5. 逐筆處理：
   a. 顯示「舊內容 → 新內容」diff
   b. 使用者確認後才寫入
   c. 寫入後更新該文件的 YAML frontmatter：`version`（+0.1）、`updated`、`changelog`
6. 全部完成後，同步更新 `bidding-assistant/docs/dev-plan/01-索引.md` 的文件清單
7. 已寫入的暫存檔 → 移到 `_staging/_done/`
8. 未寫入的暫存檔 → 保留在 `_staging/`
9. git add → commit → push（依 CLAUDE.md 同步規則）
