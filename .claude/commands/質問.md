全面校準：讀取所有機器的工作狀態，診斷偏離，發出校準指令讓所有機器重新對齊用戶終極需求。

## 觸發方式

`/質問` 或 `/質問：{具體議題}`

## 步驟

### 1. 同步

```bash
git pull origin main
```

### 2. 全面資料蒐集

**必須逐項讀取，不能靠記憶。**

a. `docs/dev-map.md` — 系統定位和開發路線
b. `docs/methodology/_index.md` — 方法論索引
c. `bidding-assistant/docs/dev-plan/_staging/元方法論框架.md`（如存在）— 元方法論
d. 所有 `docs/records/_snapshot-*.md` — 各機器快照
e. `docs/records/_index.md` — 主題索引
f. `docs/records/forum/` 近 3 天所有檔案 — 論壇貼文
g. 近期 OP header（最新 50 筆，所有機器）
h. `.claude/rules/scoring.md` — 計分板
i. Context window 中的對話（如有）

### 3. 多維度診斷

#### 3a. 終極需求對齊

- 用戶要的是什麼？現在離那個目標有多近？
- 各機器在做什麼？哪些直接推進目標、哪些在維持系統本身？
- 有沒有機器在做已經不需要的事？

#### 3b. 效率診斷

效率 = 產出價值 / 消耗成本

- 各機器的效率比健康嗎？
- 基建已夠用的部分還在加東西嗎？
- 有沒有花時間在維護流程而不是產出成果？

#### 3c. 方法論合規

- 各機器有沒有遵循方法論？
- 計分板顯示的模式——哪些問題反覆出現？
- 獎懲回饋有沒有在運作？

#### 3d. 跨機器協調

- 重複工作（同 topic 多台機器）？
- 互相矛盾的決策？
- 機器 A 建的東西機器 B 可以用但不知道？
- 機器 A 在等機器 B 的產出但 B 不知道？

#### 3e. 盲區掃描

- 重要的事所有機器都沒在做？
- 待決太久的議題（[?] 超過 3 天）？
- 過時但還在執行的計畫？

### 4. 產出校準指令

寫入 `docs/records/forum/{YYYYMMDD}-{本機碼}.md`：

```
directive|{YYYYMMDD-HHMM}|{本機碼}|ref:none
觸發：/質問 {議題（如有）}

## 診斷摘要

{2-3 段，整體狀況和主要問題——用人話寫}

## 各機器調整項目

### {機器碼 A}
- 停止：{應該停止做的事}
- 繼續：{方向正確應該繼續的事}
- 開始：{應該開始做但還沒做的事}

### {機器碼 B}
...

## 全局決策

{需要所有機器統一執行的決定（如有）}
---
```

### 5. 寫 OP + 推送

a. 寫 OP 記錄（topic: `infra-forum`）
b. 更新本機快照
c. `git add` → `git commit -m "質問：{一句話摘要}"` → `git push`

### 6. 告知用戶

- 校準指令已發出並推上 git
- 其他機器會在下次啟動或 /更新 時讀到
- 各機器的回應會出現在論壇中

## 注意

- 必須基於數據診斷（OP 筆數、commit 內容、快照狀態、計分模式），不靠感覺
- 校準指令是建議——機器可以在 reply 中提出異議（對等關係）
- 如果用戶指定議題，聚焦該議題，不做全面掃描
- 不要美化：哪台機器效率低就說，哪個方向偏了就指出
