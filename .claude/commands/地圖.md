重新生成全專案開發地圖（`docs/dev-map.md`）。

## 步驟

### 1. 同步

```bash
git pull origin main
```

### 2. 掃描來源

讀取以下檔案，提取所需資訊：

| 來源 | 提取什麼 |
|------|---------|
| `bidding-assistant/src/lib/modules/feature-registry.ts` | FEATURE_REGISTRY 陣列的所有 id, name, section |
| `bidding-assistant/docs/dev-plan/01-索引.md` | 全域 + 模組 + 附錄文件表格的版號和狀態 |
| `bidding-assistant/docs/dev-plan/_staging/_index.md` | 按 target 分組統計 plan/rule/issue 數量 |
| `docs/records/_index.md` | topic 總數、進行中和已放棄的項目 |
| `.claude/commands/*.md` | 檔名 → 指令列表，每個讀第一行作為用途描述 |
| `.claude/hooks/*.sh` | 檔名列表 |
| `docs/methodology/_index.md` | 方法論列表 |
| `docs/summaries/_index.md` | 回報數量 |
| `smugmug-mcp/`、`pcc-api-mcp/`、`pcc-monitor/` | 目錄是否存在 + package.json（如有） |
| `docs/records/_snapshot-*.md` | 所有 `[ ]` 和 `[?]` 項目 → 彙整為待做/未決清單 |

### 3. 合成地圖

按以下 6 章結構合成 `docs/dev-map.md`：

```
## 1. 系統概覽
一句話定位 + 組件清單表格（路徑、狀態、說明）

## 2. 主程式模組地圖
- Feature Registry 表格（從程式碼讀）
- 已規劃但未實作的模組（從 dev-plan 讀）
- 開發計畫文件進度表格（從 01-索引讀）

## 3. 開發工具包
- 指令表格 + Hook 表格
- 記錄系統統計（topic 數 + 機器數 + 回報數）
- 方法論表格

## 4. MCP 生態系
表格：已完成 + 待建

## 5. 開發路線
- 已完成里程碑（checklist）
- 當前階段（一段話）
- 下一步優先序（3-5 項，附理由）
- 待決事項表格（從 snapshot [?] 和 staging 待決提取）
- 已放棄表格（從 records _index 中已放棄/已被取代的 topic 提取）

## 6. 暫存區概況
碎片統計表格 + 碎片最多的目標
```

### 重點規則

- **只寫事實**：從來源讀到什麼寫什麼，不推測或展望
- **已放棄必列**：records _index 中標為「已放棄」「已被取代」的都要出現在第 5 章
- **不列碎片內容**：第 6 章只統計數量和分組，不列暫存檔的具體內容
- **用人話**：地圖中盡量用中文標題，不用檔名路徑（第 1 章的路徑欄除外）

### 4. 覆寫地圖

更新 `docs/dev-map.md` 頂部的 header：

```markdown
> 最後更新：{YYYY-MM-DD HH:MM}（UTC+8）｜更新者：{機器碼}
```

### 5. 收尾

```bash
git add docs/dev-map.md
git commit -m "更新開發地圖"
git push origin main
```
