效益驗證：AI 證明一個方案/改動有用還是沒用。

## 參數

可選：要驗證的方案（如 `/有用嗎 加入獎懲機制`）。沒帶參數就驗證最近提出的方案。

## 步驟

### 1. 找到效益宣稱

明確列出這個方案宣稱能帶來什麼好處。用一句可驗證的話寫：「{做了什麼} → {預期效果}」。

### 2. 找事實依據

對每個效益宣稱，問：有什麼證據支撐？

| 證據類型 | 可信度 |
|---------|--------|
| 實測數據（跑了有結果） | 最高 |
| 類比經驗（類似的東西有效過） | 中 |
| 邏輯推理（理論上應該有效） | 低 |
| 沒有 | 不及格 |

### 3. 判斷

三種結論：
- **有用**：效益宣稱有事實支撐，預期效果合理
- **不確定**：有部分依據但不充分，標明哪些是假設
- **沒用**：效益宣稱無法支撐，或成本超過效益

### 4. 回報

```
方案：{一句話}
效益宣稱：{列出}
依據：{每項的證據和可信度}
結論：{有用 / 不確定 / 沒用}
理由：{一句話}
```

## 注意

- 這個指令防止 AI 提出「聽起來很好但沒有根據」的方案
- AI 對自己提出的方案要比對用戶提出的方案更嚴格
- 「邏輯推理」不是零分，但不能只靠推理就說有用
