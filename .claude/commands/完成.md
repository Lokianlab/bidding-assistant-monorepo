任務收尾：commit + push → 更新快照 → 輸出摘要 → 停在這裡等 /clear。

觸發：用戶說「完成了」「done」「這個任務結束」，或直接呼叫 /完成。

---

## 步驟

### 1. 確認任務名稱

從對話推導，或直接問：「這個任務叫什麼？」（例：「修復登入 bug」「新增標案列表頁」）

### 2. 確認沒有未提交改動

⚠️ 待批准：`git status`

有改動 → 先 commit + push。格式：`[type] 任務摘要（機器碼）`
沒改動 → 跳過。

### 3. 更新快照

讀 `docs/records/_snapshot-{自己}.md`，找對應任務：
- 已在快照中 → 改為 `[x]`（完成）
- 不在快照中 → 新增 `[x] {topic-id}|{任務描述}|{完成摘要}`

更新後 commit + push：`[admin] 快照更新：{任務名稱}收尾（{機器碼}）`

### 4. 輸出收尾摘要

```
✅ {任務名稱} 完成

做了什麼：{一到三行具體描述}
產出：{測試數 / 改動檔案數 / 主要功能}
驗收方式：{怎麼確認可用，例：npm run dev → 點某頁面}

請 /clear 開新對話。
```

### 5. 停在這裡

**不要繼續找下一個任務**。不問「還有什麼要做」。
等用戶 /clear 後，在新對話繼續。
