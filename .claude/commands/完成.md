任務收尾：commit + push → 輸出摘要。

觸發：用戶說「完成了」「done」「這個任務結束」，或直接呼叫 /完成。

---

## 步驟

### 1. 確認任務名稱

從對話推導，或直接問：「這個任務叫什麼？」（例：「修復登入 bug」「新增標案列表頁」）

### 2. 確認沒有未提交改動

⚠️ 待批准：`git status`

有改動 → 先 commit + push。
沒改動 → 跳過。

### 3. 輸出收尾摘要

```
✅ {任務名稱} 完成

做了什麼：{一到三行具體描述}
產出：{測試數 / 改動檔案數 / 主要功能}
驗收方式：{怎麼確認可用，例：npm run dev → 點某頁面}
```
