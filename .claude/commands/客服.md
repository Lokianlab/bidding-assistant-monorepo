用戶回報問題或許願。接收、分類、建票、通知機器主官處理。

## 你的角色

你是**收到這個指令的機器**。不管你是誰，你現在負責接收這張票，建立記錄，通知機器主官（JDNE）跟進。

用戶說了什麼就接什麼。你不需要是 JDNE 才能收。

## 步驟

### 1. 讀懂用戶說了什麼

可能模糊、可能情緒化、可能缺細節。確認：
- 哪個功能
- 是**壞掉了** / **想要新的** / **體驗不好**

不清楚就問一個問題（只能問一個）。

### 2. 分類

| 類型 | 標準 | 標籤 |
|------|------|------|
| 🐛 Bug | 原本可以用，現在壞了 | `bug` |
| ✨ 許願 | 想要現在沒有的功能 | `wish` |
| 😤 體驗差 | 能用但不舒服 | `ux` |

### 3. 建票

讀 `docs/records/tickets.md`，在「待處理」區加一行：

```
- [ ] #{下一個編號} | {標籤} | {一句話描述} | 未分配 | {YYYYMMDD}
```

編號從 001 開始，每次 +1，讀現有最大號再 +1。

### 4. 通知機器主官

先讀 `docs/records/_roles.md`，找「機器主官」現在是誰。

在機器主官的快照備註或 commit message 中標記：

```
用戶回報：#{編號} | {標籤} | {描述} — 請機器主官分配處理
```

### 5. 回報用戶

**Bug：**
> 🐛 收到。已記錄為 #{編號}，我們來處理，修好後回報你。

**許願：**
> ✨ 記下來了。#{編號}——{一句話確認你理解的需求}。機器主官評估後告訴你什麼時候可以做。

**體驗差：**
> 😤 了解。#{編號} 已記錄，我們看看怎麼改。

---

## 機器主官收到通知後做什麼

1. 讀票，確認分類正確
2. 決定分配給哪台機器
3. 更新 `tickets.md`：把「未分配」改成機器碼
4. 透過 commit message 或快照備註通知負責機器
5. 追蹤到好，好了回報用戶

## 票的生命週期

```
用戶說 → 任一機器建票 → JDNE 分配 → 負責機器處理
→ A44T 驗證 → JDNE 確認 → 關票 → 告訴用戶「好了」
```

## 不說的事

- 不說誰去做（用戶不在乎）
- 不叫用戶去測試
- 不問超過一個問題
