在全新機器上安裝完整開發環境。根據對象選擇路徑。

## 判斷路徑

先問一句：「這台機器是你自己在操作，還是要給不會電腦的人裝？」

### 路徑 A：新手自己裝（沒有技術背景）

1. **準備 .bat 檔**：從 `scripts/一鍵安裝.bat` 取得，用 LINE/USB/雲端硬碟傳給對方
2. **準備 Token**：把以下三樣用文字訊息傳給對方（一次傳齊，讓他可以直接貼）
   - Notion Token（`secret_` 開頭那串）
   - Notion Database ID（32 字元）
   - SmugMug Token × 4 組（沒有可以跳過）
3. **告訴對方**：「把 .bat 放桌面，雙擊，跟著它的提示做。遇到問題截圖給我。」
4. **完成後對方回報機器代號**（4 個字元），記下來

AI 不需要做任何技術操作。這條路的重點是「幫人準備好材料」。

### 路徑 B：技術人員自己裝（或 AI 在目標機器上執行）

#### 1. 執行安裝腳本

如果專案已 clone：

```bash
bash /c/dev/cc程式/scripts/setup-new-machine.sh
```

如果是全新機器（連專案都沒有）：

```bash
mkdir -p /c/dev
git clone https://github.com/Lokianlab/bidding-assistant-monorepo.git /c/dev/cc程式
cd /c/dev/cc程式
bash scripts/setup-new-machine.sh
```

腳本會自動搞定一切（裝工具 → 登入 GitHub → 下載專案 → Git 設定 → 裝依賴 → 建 MCP → 設定環境變數 → 驗證測試），過程中只需要貼幾個 Token。

#### 2. 腳本跑完後驗證

```bash
cd /c/dev/cc程式/bidding-assistant
npm test          # 預期：全部通過
npm run build     # 預期：0 errors
```

#### 3. 完成

以後開工：`cd C:\dev\cc程式` → `claude`

## 機密清單

| 需要的東西 | 長什麼樣 | 從哪來 |
|-----------|---------|--------|
| Notion Token | `secret_abc123...` | Notion 設定 → Integrations → Internal Integration |
| Notion Database ID | 32 字元英數混合 | Notion 資料庫 URL 裡的那段 ID |
| SmugMug Token × 4 | API Key / Secret / Access Token / Token Secret | 現有機器的 `.mcp.json` |

## 故障排除

| 問題 | 解法 |
|------|------|
| 腳本說要關掉重開 | 有新裝的工具需要重啟終端。關掉再跑一次就好 |
| GitHub 登入失敗 | 確認瀏覽器完成登入，重跑腳本 |
| npm install 失敗 | 確認網路連線。`rm -rf node_modules && npm install` |
| 測試失敗 | 不影響安裝。跑 `rm -rf node_modules && npm install && npm test` |
| 其他 | 截圖傳 Jin |

**安心提示**：腳本可以重複跑，已完成的步驟會跳過，不會把東西搞壞。
