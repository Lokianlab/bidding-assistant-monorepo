將討論內容整理成人讀的小結報告，存到 `docs/summaries/`。

## 完整流程

### 1. 前置同步

執行 `/更新` 的完整流程（pull + 碰撞偵測 + 恢復上下文）。

### 2. 確定範圍與編號

1. 讀 `docs/summaries/_index.md`，找本機（依 `.machine-id`）最新一筆
   - 有 → 範圍起點 = 上次小結之後，編號 = 上次 base-36 +1
   - 無 → 範圍起點 = session 開始，編號 = 0001
2. Base-36 遞增規則：字元序 `0-9 A-Z`，大小寫不區分，顯示統一大寫
   - 例：`000Z` → `0010`，`00ZZ` → `0100`

### 3. 蒐集資料

**必須逐項讀取，不能靠記憶。** 按以下順序全部執行：

1. `git log --oneline` 列出範圍內所有 commit（確認期間發生了什麼）
2. `docs/records/_index.md`（主題現況）
3. `docs/records/_snapshot-{自己}.md`（本機進度）
4. 所有 `_snapshot-*.md`（其他機器進度，跨機器 topic 標出）
5. 範圍內的 OP 記錄（依 _index.md 出處欄位定位，逐檔讀取）
6. `bidding-assistant/docs/dev-plan/_staging/_index.md`（暫存區變化）
7. Context window 中尚未寫入 OP 的對話內容
8. 本機前兩份小結的 C 章（用於連續未決檢查）

蒐集完成後，比對 git log 和 OP 記錄，找出有 commit 但沒有 OP 的改動——這些是潛在遺漏，必須納入小結或補寫 OP。

### 4. 產出報告

#### 章節結構（沒有內容的章不列出）

| 章 | 含義 |
|----|------|
| A | 議題列表 |
| B | 已定案 |
| C | 待決策 |
| D | 檔案變更 |

#### 編號規則

- 節：數字（A1, A2, B1, ...）
- C 章選項：小寫字母（C1a, C1b, ...）
- 引用格式：`{機器碼}-{流水號}:{章}{節}`
  - 例：`JDNE-0001:B2` = JDNE 第 1 份小結，B 章第 2 節

#### 報告模板

```markdown
# 小結 {機器碼}-{流水號}

日期: {YYYY-MM-DD HH:MM}
機器: {機器碼}
範圍: 自 {上一份編碼 或「起始」} 至本次
資料來源: {OP 引用列表, context window}

## A. 議題列表

A1. {帶動詞的議題標題}
  背景：{為什麼會討論這個}
  經過：{做了什麼、討論了什麼}
  現況：{目前狀態}（或 結論：{已定案的結果}）

## B. 已定案

B1. {一句話結論 + 理由}

## C. 待決策

**C1. {帶動詞的決策標題}**
背景：{來龍去脈}
  C1a. {選項一}
  C1b. {選項二}

**C2. {帶動詞的決策標題}**
背景：{來龍去脈}
  現況：{尚無明確選項時，描述目前狀態}

## D. 檔案變更

D1. {檔案路徑} — {變更摘要}
```

#### 撰寫指南

**通則**
- 用人話寫，不用機器格式簡稱
- 客觀描述事實，不加 AI 的評論或建議
- 主謂賓語齊備，要有跨時間參考性（隔一週回頭看仍能讀懂）

**A 章**
- 標題帶動詞（如「設計並建置」「評估」「處理」）
- 三行子項用標籤（背景/經過/現況或結論），不加子編號
- 按時間順序排列，待辦盤點永遠排最後
- 已定案的議題第三行用「結論：」，進行中用「現況：」

**B 章**
- 每項一句話結論 + 理由
- 只放本次確實關閉的議題

**C 章**
- 標題加粗，帶動詞
- 「背景：」標籤開頭
- 有明確選項時：用編號（C1a/C1b/...）列出各選項
- 尚無明確選項時：用「現況：」標籤描述目前狀態
- 不寫行動指令，寫待決定的事和可選方向
- 順序盡量對應 A 章

**完備性規則**
- C 每一項必須能在 A 找到出處
- A 中提到的未決事項必須出現在 C
- **連續未決檢查**：同一待決項在本機連續 ≥3 份小結出現 → 在 C 章標記「連續 N 份未決，需重新定義問題或關閉」

#### 限制

- 每份 ≤ 200 行
- 沒有內容的章不列出

### 5. 修訂迴圈

報告輸出後進入**修訂模式**，只接受對報告內容的補充或修正。

#### 接受（修改報告）

- 指名編號 + 補充事實（如「B2 少了 xxx」）
- 修正事實錯誤（如「A3 不是 ITEJ 做的」）
- 補充遺漏（如「C 章漏了 xxx 議題」）
- 格式或呈現意見（如「C 章加上負責人」）

#### 擋住（存檔後再聊）

- 對報告中某個決策提出不同意見，要重新討論
- 跟報告內容完全無關的新話題

擋住時的回覆：

> 「{話題摘要}我記下來，存檔後再聊。針對小結報告還有其他修訂意見嗎？沒有的話就先存檔。」

同時更新 snapshot 記錄被擋住的話題（`[?]` 標記）。

### 6. 存檔

用戶確認無修訂後：

1. 建目錄（如不存在）：`docs/summaries/{YYYY-MM}/{機器碼}/`
2. 寫入報告：`docs/summaries/{YYYY-MM}/{機器碼}/{流水號}.md`
3. 追加索引 `docs/summaries/_index.md`：
   - 格式：`{編碼}|{日期}|{關鍵詞摘要}|{路徑}`
4. **不執行 git 操作**（由下一步 /更新 處理）

### 7. 收尾同步

執行 `/更新`（會偵測到未提交的新檔案，先 commit + push 再完成同步流程）。
